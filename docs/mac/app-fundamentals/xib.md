# <a name="xib-files"></a>файлы .xib

_В этой статье рассматривается работа с файлами .xib, созданные в построителе Xcode интерфейс для создания и поддержания пользовательские интерфейсы для приложений Xamarin.Mac._

> [!NOTE]
> Предпочтительный способ создания пользовательского интерфейса для приложения Xamarin.Mac — с помощью раскадровки. Исторически сложилось так и для работы с проектами более ранних версий Xamarin.Mac была оставлена в этой документации на месте. Дополнительные сведения см. в разделе нашей [введение в раскадровки](~/mac/platform/storyboards/index.md) документации.

## <a name="overview"></a>Обзор

При работе с C# и .NET в приложении Xamarin.Mac, имеют доступ к одной элементы пользовательского интерфейса и средств, предоставляемых разработчик, работающий *Objective-C* и *Xcode* does. Поскольку Xamarin.Mac напрямую интегрируется с Xcode, можно использовать в Xcode _интерфейс построителя_ для создания и поддержания интерфейсов пользователя (или их при необходимости создать непосредственно в код C#).

Файл .xib macOS используется для определения элементов интерфейса пользователя приложения (например, меню, Windows, представления, метки, текстовые поля), создаются и поддерживаются графически в Xcode интерфейс построителя.

[![Пример выполняемого приложения](xib-images/intro01.png "пример выполняемого приложения")](xib-images/intro01-large.png#lightbox)

В этой статье мы обсудим основы работы с файлами .xib в приложении Xamarin.Mac. Настоятельно рекомендуется работать через [Hello, Mac](~/mac/get-started/hello-mac.md) статью во-первых, как он охватывает основные принципы и методы, которые будут использоваться в этой статье.

Может потребоваться рассмотреть [предоставление C# классы и методы для Objective-C](~/mac/internals/how-it-works.md) раздел [внутренние компоненты Xamarin.Mac](~/mac/internals/how-it-works.md) документов также объясняется, какие `Register` и `Export` атрибуты используется для подключения классов C# для Objective-C-объекты и пользовательского интерфейса элементов.


## <a name="introduction-to-xcode-and-interface-builder"></a>Общие сведения об Xcode и конструкторе интерфейсов Interface Builder

В составе Xcode Apple создала средство, вызывается конструктор интерфейс, который позволяет визуально создавать пользовательский интерфейс в конструкторе. Xamarin.Mac fluently интегрируется с помощью построителя интерфейс, что позволяет создавать пользовательский Интерфейс с помощью тех же средств, которые выполняют пользователи Objective-C.


### <a name="components-of-xcode"></a>Компоненты Xcode

При открытии файла .xib в Xcode из Visual Studio для Mac, он открывается с **навигаторе проекта** с левой стороны экрана **иерархии интерфейса** и **интерфейс редактора** в середине и **свойства и служебные программы** раздел справа:

[![Компоненты пользовательского интерфейса Xcode](xib-images/xcode03.png "компоненты пользовательского интерфейса Xcode")](xib-images/xcode03-large.png#lightbox)

Давайте посмотрим на содержимого каждого из этих разделов Xcode выполняет и как будет использовать их для создания интерфейса Xamarin.Mac приложения.


#### <a name="project-navigation"></a>Проект навигации

При открытии файла для редактирования в Xcode .xib Visual Studio для Mac создает файл проекта Xcode в фоновом режиме для передачи изменений между собой и Xcode. Позже при переключении обратно в Visual Studio для Mac с Xcode, любые изменения, внесенные в этот проект синхронизируются с Xamarin.Mac проекта в Visual Studio для Mac.

**Навигации проекта** раздел позволяет перемещаться между все файлы, составляющие это _прокладка_ проекту Xcode. Как правило, можно, только интерес в этом списке, такие как файлы .xib **MainMenu.xib** и **MainWindow.xib**.


#### <a name="interface-hierarchy"></a>Интерфейс иерархии

**Иерархии интерфейса** раздел позволяет легко получить несколько ключевых свойств пользовательского интерфейса, такие как его **заполнители** и main **окна**. В этом разделе также можно получить доступ к отдельные элементы (представления), которые обеспечивают пользовательский интерфейс и учесть способом, что они являются вложенными, перетаскивая их в иерархии.


#### <a name="interface-editor"></a>Интерфейс редактора

**Интерфейс редактора** раздел содержит области, на котором вы графически макета в пользовательском интерфейсе. Перетащите элементы из **библиотеки** раздел **свойства и служебные программы** раздела, чтобы создать проект. Как добавить элементы пользовательского интерфейса (представления) в область конструктора, они будут добавлены к **иерархии интерфейса** раздел в том порядке, в котором они появляются в **редактор интерфейса**.


#### <a name="properties--utilities"></a>Свойства и служебные программы

**Свойства и служебные программы** раздел состоит из двух основных разделов, которые мы будем работать с, **свойства** (также называемые инспекторы) и **библиотеки**:

![Инспектор свойств](xib-images/xcode04.png "инспектор свойств")

Изначально этот раздел пуст почти, однако при выборе элемента в **интерфейс редактора** или **иерархии интерфейса**, **свойства** раздел будет заполнен сведения о данного элемента и свойства, которые можно настраивать.

В разделе **Свойства** находится восемь разных *вкладок инспекторов*, как показано на следующем рисунке:

[![Общие сведения о всех инспекторах](xib-images/xcode05.png "Обзор всех инспекторах")](xib-images/xcode05-large.png#lightbox)

Слева направо располагаются следующие вкладки:

-   **Инспектор файлов.** Инспектор файлов отображает сведения о файле, такие как имя файла и расположение редактируемого XIB-файла.
-   **Экспресс-справка.** На вкладке "Экспресс-справка" содержится контекстная справка по компоненту или функции, выбранной в Xcode.
-   **Инспектор удостоверений.** Инспектор удостоверений предоставляет сведения о выбранном элементе управления или представлении.
-   **Атрибуты инспектора** — инспектора атрибутов позволяет настраивать различные атрибуты в выбранный элемент управления или представлении.
-   **Размер инспектора** — размер инспектора позволяет управлять размером и размеров в выбранный элемент управления или представлении.
-   **Инспектор подключений** — подключений инспектора показаны розетки и действие подключения элементов управления. Мы рассмотрим выходов и действия в сразу.
-   **Инспектор привязок** — инспектора привязки позволяет настроить элементы управления, чтобы их значения автоматически привязываются к модели данных.
-   **Просмотр эффектов инспектора** — инспектора эффекты представление позволяет указать влияние на элементах управления, таких как анимации.

В **библиотеки** можно найти элементы управления и объекта, чтобы поместить в конструктор для графического построения пользовательского интерфейса:

![Пример библиотеки инспектора](xib-images/xcode06.png "пример инспектора библиотеки")

Теперь, когда вы знакомы с Xcode интегрированной среды разработки и интерфейс построителя, давайте взглянем на ее использованием для создания пользовательского интерфейса.


## <a name="creating-and-maintaining-windows-in-xcode"></a>Создание и обслуживание windows в Xcode

Для создания пользовательского интерфейса на Xamarin.Mac приложение рекомендуется с помощью раскадровки (см. в разделе нашей [введение в раскадровки](~/mac/platform/storyboards/index.md) Дополнительные сведения) и в результате работы любого нового проекта в Xamarin.Mac будут по умолчанию используется раскадровки.

Переключиться на использование .xib на базе пользовательского интерфейса, выполните следующие действия.

1. Откройте Visual Studio для Mac и начать новый проект Xamarin.Mac.
2. В **Pad решения**, щелкните правой кнопкой мыши проект и выберите **добавить** > **новый файл...**
3. Выберите **Mac** > **Windows контроллера**:

    ![Добавление нового окна контроллера](xib-images/setup00.png "добавлении нового контроллера окна")
4. Введите `MainWindow` имя и нажмите кнопку **New** кнопки:

    ![Добавление нового окна Main](xib-images/setup01.png "Добавление нового главного окна")
5. Щелкните правой кнопкой мыши проект еще раз и выберите **добавить** > **новый файл...**
6. Выберите **Mac** > **главного меню**:

    ![Добавление нового меню Main](xib-images/setup02.png "Добавление нового главного меню")
7. Оставьте имя, `MainMenu` и нажмите кнопку **New** кнопки.
8. В **Pad решения** выберите **Main.storyboard** файл, щелкните правой кнопкой мыши и выберите **удалить**:

    ![При выборе основных раскадровки](xib-images/setup03.png "выбора основного раскадровки")
9. В диалоговом окне «Удалить» щелкните **удалить** кнопки:

    ![Подтверждение удаления](xib-images/setup04.png "подтверждения удаления")
10. В **Pad решения**, дважды щелкните **Info.plist** файл, чтобы открыть его для редактирования.
11. Выберите `MainMenu` из **главного интерфейса** раскрывающегося списка:

    [![В главном меню](xib-images/setup05.png "параметр главного меню")](xib-images/setup05-large.png#lightbox)
12. В **Pad решения**, дважды щелкните **MainMenu.xib** файл, чтобы открыть его для редактирования в построителе интерфейса в Xcode.
13. В **инспектора библиотеки**, тип `object` в поле поиска затем перетащите новую **объекта** на поверхность разработки:

    [![Изменения в главном меню](xib-images/setup06.png "изменения в главном меню")](xib-images/setup06-large.png#lightbox)
14. В **удостоверение инспектора**, введите `AppDelegate` для **класса**:

    [![При выборе делегат приложения](xib-images/setup07.png "при выборе делегат приложения")](xib-images/setup07-large.png#lightbox)
15. Выберите **владелец файла** из **иерархии интерфейса**, переключитесь в **инспектора подключения** и перетащите линию из делегата для `AppDelegate` **Объект** только что добавлен в проект:

    [![Подключение приложения делегат](xib-images/setup08.png "подключение делегат приложения")](xib-images/setup08-large.png#lightbox)
16. Сохранить изменения и вернуться в Visual Studio для Mac.

Все эти изменения в месте, изменить **AppDelegate.cs** файла и сделать его выглядеть следующим образом:

```csharp
using AppKit;
using Foundation;

namespace MacXib
{
    [Register ("AppDelegate")]
    public class AppDelegate : NSApplicationDelegate
    {
        public MainWindowController mainWindowController { get; set; }

        public AppDelegate ()
        {
        }

        public override void DidFinishLaunching (NSNotification notification)
        {
            // Insert code here to initialize your application
            mainWindowController = new MainWindowController ();
            mainWindowController.Window.MakeKeyAndOrderFront (this);
        }

        public override void WillTerminate (NSNotification notification)
        {
            // Insert code here to tear down your application
        }
    }
}
```

Теперь окно основного приложения, определенные в **.xib** файл, включенный в проект автоматически при добавлении контроллера окна. Для изменения проекта windows в **Pad решения**, дважды щелкните **MainWindow.xib** файла:

![При выборе файла MainWindow.xib](xib-images/edit01.png "выбора MainWindow.xib файла")

Откроется окно конструктора в построителе Xcode элемента интерфейса:

[![Редактирование MainWindow.xib](xib-images/edit02.png "редактирование MainWindow.xib")](xib-images/edit02-large.png#lightbox)


### <a name="standard-window-workflow"></a>Стандартное окно рабочего процесса

Для любого периода, создания и работы с Xamarin.Mac приложения процесса в основном аналогичен:

1. Для новых окон, которые не используются по умолчанию, автоматически добавляется в проект добавьте в проект новое определение окна.
2. Дважды щелкните файл .xib, чтобы открыть окно конструктора для редактирования в Xcode интерфейс построителя.
3. Задайте свойства требуется окно **инспектора атрибут** и **инспектора размер**.
4. Перетащите элементы управления, необходимые для построения интерфейса и настройки их в **инспектора атрибут**.
5. Используйте **инспектора размер** обрабатывать изменения размера для элементов пользовательского интерфейса.
6. Предоставляет элементы пользовательского интерфейса окна в код C# с помощью выходов и действия.
7. Сохранить изменения и вернуться в Visual Studio для Mac для синхронизации с Xcode.


### <a name="designing-a-window-layout"></a>Проектирование макета окна

1. Найдите нужный элемент управления в **инспектора библиотеки** и перетащите его в **редактор интерфейса** и разместите его.
2. Задайте свойства требуется окно **инспектора атрибут**.
3. Используйте **инспектора размер** обрабатывать изменения размера для элементов пользовательского интерфейса.
4. При использовании пользовательского класса, установите его в **удостоверение инспектора**.
5. Предоставляет элементы пользовательского интерфейса в код C# с помощью выходов и действия.
6. Сохранить изменения и вернуться в Visual Studio для Mac для синхронизации с Xcode.

## <a name="exposing-ui-elements-to-c-code"></a>Предоставление пользовательского интерфейса элементов кода на языке C#

После завершения компоновки внешний вид пользовательского интерфейса в интерфейс построителя необходимо для предоставления элементов пользовательского интерфейса, чтобы они могли быть доступны из кода C#. Чтобы сделать это, вы будете использовать действия и выходов.


### <a name="setting-a-custom-main-window-controller"></a>Установка контроллера пользовательских главное окно

Чтобы иметь возможность создать выходов и действия, чтобы предоставить элементы пользовательского интерфейса для кода C#, Xamarin.Mac приложению потребуется использовать настраиваемый контроллер окна.

Выполните следующие действия:

1. Открытие раскадровки приложения в построителе интерфейса в Xcode.
2. Выберите `NSWindowController` в рабочей области конструирования.
3. Переключитесь в **удостоверение инспектора** Просмотр и ввод `WindowController` как **имя класса**:

### <a name="outlets-and-actions"></a>Выходы и действия

Так что такое выходов и действия В традиционном программировании пользовательского интерфейса .NET добавляемый элемент управления в пользовательском интерфейсе автоматически предоставляется как свойство. В Mac это работает по-другому — простое добавление элемента управления в представление не делает его доступным для кода. Необходимо явно предоставить элемент пользовательского интерфейса в код. В порядке это сделать, Apple дает два варианта:

-  **Переменные экземпляров.** Они аналогичны свойствам. Если связать элемент управления к розетке, она предоставляется коду через свойство, что позволяет выполнять такие действия, как присоединить обработчики событий, вызывать методы для его и т. д.
-  **Действия.** Они аналогичны командам в WPF. Например при выполнении действия в элементе управления, например нажатие кнопки, элемент управления автоматически вызывает метод в коде. Действия — удобный и эффективный, так как вы можете подключать множество элементов управления, в то же действие.

В Xcode, выходов и действия добавляются непосредственно в код с помощью *перетаскивание элемента управления*. В частности, это означает, что создание розетки или действие, выбрать какой элемент управления вы хотите добавить розетки или действие, удерживая нажатой **управления** кнопку на клавиатуре, а затем перетащите этот элемент управления непосредственно в код.

Для разработчиков, Xamarin.Mac это означает, перетащите в файлы заглушки Objective-C, которые соответствуют файл C#, где нужно создать розетки или действия. Visual Studio для Mac создан файл с именем **MainWindow.h** как часть проекта Xcode прокладка он создан с помощью построителя интерфейса:

[![Примером h-файл в Xcode](xib-images/xcode16.png "примером h-файл в Xcode")](xib-images/xcode16-large.png#lightbox)

Этот файл заглушки .h отражает **MainWindow.designer.cs** , автоматически добавляется в проект Xamarin.Mac при создании нового `NSWindow` создается. Этот файл будет использоваться для синхронизации изменений, внесенных интерфейс построителя и является, где мы создадим выходов и действия, чтобы элементы пользовательского интерфейса, предоставляются кода на языке C#.

### <a name="writing-the-code"></a>Написание кода

С помощью пользовательского интерфейса создан и открыт для кода через выходов и действия, элементы пользовательского интерфейса можно приступать написать код для воплощать программы. Например, откройте **MainWindow.cs** для редактирования, дважды щелкнув его в файл **Pad решение**:

[![Файл MainWindow.cs](xib-images/code01.png "MainWindow.cs файла")](xib-images/code01-large.png#lightbox)

И добавьте следующий код в `MainWindow` класс для работы с розетки образца, созданной выше:

```csharp
private int numberOfTimesClicked = 0;
...

public override void AwakeFromNib ()
{
    base.AwakeFromNib ();

    // Set the initial value for the label
    ClickedLabel.StringValue = "Button has not been clicked yet.";
}
```

Обратите внимание, что `NSLabel` осуществляется в C# прямое имя, назначенное его в Xcode, при создании его выхода в Xcode, в этом случае вызывается `ClickedLabel`. Так же, как любой обычного класса C# можно открыть любой метод или свойство, предоставляемого объекта.

> [!IMPORTANT]
> Необходимо использовать `AwakeFromNib`, вместо другой метод, такой как `Initialize`, так как `AwakeFromNib` называется _после_ загружается и создан пользовательский интерфейс из файла .xib операционной системы. При попытке получить доступ к метке, прежде чем файл .xib полной загрузки и создания экземпляра, получаемому `NullReferenceException` ошибка, так как элемент управления label может быть еще не создан.

Добавьте следующий разделяемый класс, чтобы `MainWindow` класса:

```csharp
partial void ClickedButton (Foundation.NSObject sender) {

    // Update counter and label
    ClickedLabel.StringValue = string.Format("The button has been clicked {0} time{1}.",++numberOfTimesClicked, (numberOfTimesClicked < 2) ? "" : "s");
}
```

Этот код присоединяет к действию, созданные в Xcode и интерфейс построителя и будет вызываться при каждом нажатии кнопки.

Некоторые элементы пользовательского интерфейса автоматически имеют встроенные действия, например, элементы в строке меню по умолчанию такие как **открыть...**  элемента меню (`openDocument:`). В **Pad решения**, дважды щелкните **AppDelegate.cs** файл, чтобы открыть его для редактирования и добавьте следующий код ниже `DidFinishLaunching` метод:

```csharp
[Export ("openDocument:")]
void OpenDialog (NSObject sender)
{
    var dlg = NSOpenPanel.OpenPanel;
    dlg.CanChooseFiles = false;
    dlg.CanChooseDirectories = true;

    if (dlg.RunModal () == 1) {
        var alert = new NSAlert () {
            AlertStyle = NSAlertStyle.Informational,
            InformativeText = "At this point we should do something with the folder that the user just selected in the Open File Dialog box...",
            MessageText = "Folder Selected"
        };
        alert.RunModal ();
    }
}
```

Строки ключа является `[Export ("openDocument:")]`, он сообщает `NSMenu` , **AppDelegate** имеет метод `void OpenDialog (NSObject sender)` , отвечающей на запросы `openDocument:` действие.

Дополнительные сведения о работе с меню см. в разделе нашей [меню](~/mac/user-interface/menu.md) документации.

Теперь измените CS-файле окна, которое будет применяться для отображения нового представления (например, **MainWindow.cs**) и добавьте следующий код:

```csharp
private SubviewType ViewType = SubviewType.None;
private NSViewController SubviewController = null;
private NSView Subview = null;
...

private void DisplaySubview(NSViewController controller, SubviewType type) {

    // Is this view already displayed?
    if (ViewType == type) return;

    // Is there a view already being displayed?
    if (Subview != null) {
        // Yes, remove it from the view
        Subview.RemoveFromSuperview ();

        // Release memory
        Subview = null;
        SubviewController = null;
    }

    // Save values
    ViewType = type;
    SubviewController = controller;
    Subview = controller.View;

    // Define frame and display
    Subview.Frame = new CGRect (0, 0, ViewContainer.Frame.Width, ViewContainer.Frame.Height);
    ViewContainer.AddSubview (Subview);
}
```

Если необходимо показать новое представление, загруженная из файла .xib в контейнере окна ( **пользовательское представление** добавленных выше), этот код обрабатывает удаления любого существующего представления и замена его для создания нового. Он проверяет его уже имеется представление отображается, поэтому она удаляется из экрана. Затем необходимое представление, которая была передана в (как загружаются из View Controller) размер до помещается в область содержимого и добавляет его содержимое для отображения.

Чтобы отобразить новое представление, используйте следующий код:

```csharp

```
