# <a name="introduction-to-macos-sierra"></a>Общие сведения о macOS Сьерра

С помощью новый macOS Сьерра разработчик может использовать преимущества новых интерфейсов API, позволяющих конечным пользователям взаимодействовать с их приложений и веб-сайтов ранее недоступным способами. Например Apple теперь позволяет веб-сайтов предоставить клиентам возможность безопасно оплаты через Apple Pay и улучшения для повышения исходного framework графики приложения и вычислительных потенциальных. 

Дополнительные сведения о macOS Сьерра см. в разделе Apple [macOS + приложений](https://developer.apple.com/macos/) документации.

<a name="Whats-New-in-macOS-Sierra" />

## <a name="whats-new-in-macos-sierra"></a>Новые возможности macOS Сьерра

Apple были добавлены несколько новых API-интерфейсов и служб в macOS Сьерра вместе с усовершенствованиях существующих компонентов, включая:

<a name="Apple-File-System" />

### <a name="apple-file-system"></a>Apple файловой системы

С macOS Сьерра Apple выпущена новая файловая система Apple как современных файловая система iOS, macOS, tvOS и watchOS. В файловой системе Apple был оптимизирован для хранения флэш-памяти и SSD и предоставляет следующие возможности: пространства стойкое шифрование, метаданные копирования при записи управления доступом, клонирование для файлов и каталогов, моментальные снимки, изменения размера быстрого каталога и atomic примитивы safe сохранить.

Дополнительные сведения см. в разделе Apple [руководство по системе Apple файл](https://developer.apple.com/library/prerelease/content/documentation/FileManagement/Conceptual/APFS_Guide/Introduction/Introduction.html#//apple_ref/doc/uid/TP40016999).

<a name="Apple-Pay-Enhancements" />

### <a name="apple-pay-enhancements"></a>Усовершенствования оплаты Apple

Apple предприняла ряд усовершенствований для оплаты Apple macOS Сьерра разрешить пользователю безопасный выплатах с веб-сайтов.

С macOS Сьерра ряд новых интерфейсов API были добавлены, работающие с macOS Сьерра, iOS и watchOS для поддержки динамического оплаты сетей и среде тестирования новой "песочницы".

macOS Сьерра включает новую платформу ApplePay Javascript, разработчик может включать Apple Pay непосредственно в iOS и macOS веб-сайтов на основе Safari. Для веб-сайтов, поддерживающих Apple Pay пользователь может авторизовать с помощью iPhone или Apple Watch оплаты.

Дополнительные сведения см. в разделе Apple [ApplePay JS Framework](https://developer.apple.com/reference/applepayjs) ссылки.

<a name="Building-Modern-macOS-Apps" />

### <a name="building-modern-macos-apps"></a>Построение macOS современных приложений

MacOS современных приложений, таких как Apple веб-браузера Safari, страниц word процессора и номера разворота лист открыть много новых технологий для унифицированной, зависящие от контекста пользовательского интерфейса, выполняющий с традиционные элементы пользовательского интерфейса, такие как плавающие панели и нескольких Windows.

Наш [macOS создание современных приложений](~/mac/platform/introduction-to-macos-sierra/modern-cocoa-apps.md) руководство охватывает несколько советов, функции и методы разработчик может использовать для выполнения сборки приложения в Xamarin.Mac современных macOS.

<a name="CloudKit-Data-Sharing" />

### <a name="cloudkit-data-sharing"></a>Совместное использование данных CloudKit

Платформа CloudKit была расширена в macOS Сьерра, чтобы разрешить пользователю быстро и легко совместное использование записей или наборов записей из базы данных — закрытый iCloud.

CloudKit предоставляет полный пользовательский Интерфейс для отправки и принимать общих записей приглашения и у пользователя есть полный чтение и запись контроль над людей, имеющих доступ к записи.

Дополнительные сведения см. в разделе Apple [ссылке платформы CloudKit](https://developer.apple.com/reference/clockkit) и [CloudKit JS Framework ссылка](https://developer.apple.com/reference/cloudkitjs).

> [!IMPORTANT]
> Apple [предоставляет средства](https://developer.apple.com/support/allowing-users-to-manage-data/) для помощи разработчикам правильно обрабатывать Европейского Союза общие данные защиты стабилизации (GDPR).

<a name="Safari-App-Extensions-Support" />

### <a name="safari-app-extensions-support"></a>Поддержка расширений приложений Safari

Расширения приложения Safari Позвольте приложению при тесно интегрированы macOS Сьерра для расширения поведения класса веб-браузера Safari. Поскольку расширения приложения Safari macOS работают аналогично iOS Safari расширения приложения, они просты для порта из одной системы в другую.

Дополнительные сведения см. в разделе Apple [руководство по программированию Safari приложения расширения](https://developer.apple.com/library/prerelease/content/documentation/NetworkingInternetWeb/Conceptual/SafariAppExtension_PG/index.html#//apple_ref/doc/uid/TP40017319).

<a name="Security-and-Privacy-Enhancements" />

### <a name="security-and-privacy-enhancements"></a>Безопасность и конфиденциальность усовершенствования

Apple внесены некоторые улучшения безопасности и конфиденциальности в macOS Сьерра, которая поможет повысить безопасность приложения и обеспечить конфиденциальность пользователя, включая следующие приложения:

- Новый `NSAllowsArbitraryLoadsInWebContent` ключ можно добавить в приложение `Info.plist` файл и позволит использовать веб-страницы правильно загрузить во время защиты безопасности транспорта Apple (ATS) все еще включена для остальной части приложения.
- Общие данные безопасности архитектуры (CDSA) интерфейс API рекомендуется к использованию и должны быть заменены с помощью SecKey API для создания асимметричных ключей.
- Для всех подключений SSL/TLS симметричного шифрования RC4 теперь отключено по умолчанию. Кроме того API защиты транспорта больше не поддерживает SSLv3 и рекомендуется, приложение остановить использование шифрования SHA-1 и 3DES как можно быстрее.
- Так как новый буфер обмена в iOS 10 и macOS Сьерра позволяет пользователю скопировать и вставить между устройствами, API была увеличена, чтобы разрешить буфера не должна превышать к определенному устройству и применит автоматически отключен в заданной точке. Кроме того именованные монтажных столах, более не сохраняются и должны быть заменены с общим контейнерам монтажном столе.
- Если приложение обращается к защищенным данным (например, календарь пользователя), он _должен_ объявления этого намерения с ключом цели строковое значение в его `Info.plist` файла (`NSCalendarUsageDescription` в случае календарь).
- Приложения подписано разработчика, которые не были доставлены через Mac App Store теперь могут получить преимущество CloudKit, iCloud цепочки ключей, iCloud диск, удаленный push-уведомлений, VPN и MapKit прав.
- macOS Сьерра больше не поддерживает доставки внешнего кода или данных вместе с приложением подписи кода в ZIP-архиве или образа диска без знака, как среда выполнения путь не известен до времени выполнения.

Кроме того, приложения, работающие на macOS Сьерра (или более поздней версии) необходимо объявлять статически их назначение для доступа к определенных функций или сведения о пользователе, указав один или несколько конфиденциальности определенные ключи в их `Info.plist` файлы, в которых объясняется, почему приложению необходимо получить пользователю доступ.

Поскольку macOS Сьерра совместно использует эти изменения с iOS 10, см. в разделе iOS 10 [безопасности и конфиденциальности улучшения](~/ios/app-fundamentals/security-privacy.md) для дополнительных сведений.

<a name="Smart-Card-Driver-Extension-Support" />

### <a name="smart-card-driver-extension-support"></a>Поддержка расширения драйвер смарт-карт

С помощью macOS Сьерра, можно создать приложение `NSExtension` основе драйверы смарт-карт, которые разрешает доступ только для чтения к содержимому из определенного типа смарт-карт. Затем эти сведения представлены в цепочке ключей системы (замены устаревший метод общую архитектуру безопасности данных).

Дополнительные сведения, Pleas разделе Apple [CryptoTokenKit Framework ссылка](https://developer.apple.com/reference/cryptotokenkit).

<a name="Unified-Logging" />

### <a name="unified-logging"></a>Унифицированный ведения журнала

Унифицированный ведения журнала предоставляет приложению один API-Интерфейс для эффективного обмена сообщениями на всех уровнях системы. При выполнении единого входа приложение имеет возможность точного управления несколько уровней ведения журнала, включая конфиденциальностью и отслеживания, чтобы упростить отладку действий. 

Ведение журнала предоставляет корреляция автоматического сообщений при совместном использовании действия ведения журнала и отслеживания.

macOS Сьерра включает новое консольное приложение (в приложения и служебные программы), могут отображать данные журнала из нескольких источников, включая подключенные устройства. Также поддерживает токенами и сохраненный поиск и отображает соединения между связанных сообщений в нескольких процессах.

Кроме того сообщения журнала можно просматривать и поддерживаются с использованием средства командной строки.

Дополнительные сведения см. в разделе Apple [ведения](https://developer.apple.com/reference/os/1891852-logging).

<a name="Wide-Color" />

### <a name="wide-color"></a>Широкую цветовую

macOS Сьерра расширяет поддержку форматов пикселей расширенный диапазон и широким пробелов во всей системе, включая платформы, например Core графики, образ Core, системы и AVFoundation. Поддержка устройства с дисплеями широкую цветовую дальнейшей упрощают, предоставляя это поведение во всем стеке всей графики.

Кроме того `AppKit` был изменен для работы в новом расширенных **sRGB** цветовое пространство, что упрощает смешивания цветов в широкую цветовую охваты без потери значительного увеличения производительности.

Apple предоставляет следующие рекомендации при работе с шириной цветов:

- `NSColor` Теперь использует sRGB цветовое пространство и больше не будут выполните приведение значения к `0.0` для `1.0` диапазона. Если приложение зависит от предыдущего поведения приведения, будет необходимо изменить для macOS Сьерра.
- При использовании низкого уровня API, например двухмерной графики или исходного состояния системы для обеспечения обработки изображений, приложение следует использовать расширенный диапазон цвет пространства и пикселей формате, который поддерживает 16-разрядных значений с плавающей запятой. При необходимости, приложению придется вручную выполните приведение значений компонентов цвета.
- Основные графики, образ Core и шейдеров производительности системы предоставляют новые методы для преобразования между двумя цветовых схем.

Чтобы узнать больше, см. в разделе нашей [введение в широкую цветовую](~/ios/platform/wide-color.md) руководства.
