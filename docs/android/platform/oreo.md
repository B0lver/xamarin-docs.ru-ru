# <a name="oreo-features"></a>Функции Oreo

_Как приступить к работе с Xamarin. Android для разработки приложений для последней версии Android._

[Android 8,0 Oreo](https://developer.android.com/index.html) — это последняя версия Android, доступная в Google. Android Oreo предлагает множество новых функций, представляющих интерес для разработчиков Xamarin. Android. К таким функциям относятся каналы уведомлений, эмблемы уведомлений, пользовательские шрифты в формате XML, загружаемые шрифты, автозаполнение и изображение на рисунке (PIP). Android Oreo включает новые API для этих новых возможностей, и эти API доступны для приложений Xamarin. Android при использовании Xamarin. Android 8,0 и более поздних версий.

[![Образ Android Oreo Hero](oreo-images/01-android-o-logo-sml.png)](oreo-images/01-android-o-logo.png#lightbox)

Эта статья структурирована, которая поможет вам приступить к разработке приложений Xamarin. Android для Android 8,0 Oreo. В нем объясняется, как установить необходимые обновления, настроить пакет SDK и создать эмулятор (или устройство) для тестирования. Здесь также приводится описание новых функций в Android 8,0 Oreo со ссылками на примеры приложений, демонстрирующих использование функций Oreo Android в приложениях Xamarin. Android.


## <a name="requirements"></a>Требования

Для использования функций Android Oreo в приложениях на основе Xamarin необходимо следующее:

-   **Visual Studio** &ndash; Если вы используете Windows, требуется версия Visual Studio 15,5 или более поздняя.  Если вы используете Mac, требуется Visual Studio для Mac версии 7.2.0.

-   **Xamarin. Android** &ndash; Xamarin. Android 8,0 или более поздней версии должен быть установлен и настроен с помощью Visual Studio.

-   **Пакет SDK для Android** &ndash; Пакет SDK для Android 8,0 (API 26) или более поздней версии необходимо установить с помощью диспетчера пакет SDK для Android.



## <a name="getting-started"></a>Начало работы

Чтобы приступить к работе с Android Oreo с Xamarin. Android, перед созданием проекта Oreo для Android необходимо загрузить и установить последние версии средств и пакетов SDK:

1. Обновление до последней версии Visual Studio.

2. Установите пакеты и средства **Android 8.0.0 (API 26)** или более поздней версии с помощью диспетчера пакетов SDK.

3. Создайте новый проект Xamarin. Android, предназначенный для Android Oreo (API 26).

4. Настройте эмулятор или устройство для тестирования приложений Oreo Android.

Каждое из этих действий описано в следующих разделах:

#### <a name="creating-virtual-devices-using-avdmanager"></a>Создание виртуальных устройств с помощью авдманажер

Чтобы использовать **авдманажер** для создания нового виртуального устройства, выполните следующие действия.

# <a name="visual-studiotabwindows"></a>[Visual Studio](#tab/windows)

1.  Откройте окно командной строки и укажите `JAVA_HOME` расположение пакета SDK для Java на компьютере. Для типичной установки Xamarin можно использовать следующую команду:

    ```cmd
    setx JAVA_HOME "C:\Program Files\Java\jdk1.8.0_131"
    ```

2.  Добавьте расположение папки пакет SDK для Android `bin` `PATH`в.
    Для типичной установки Xamarin можно использовать следующую команду:

    ```cmd
    setx PATH "%PATH%;C:\Program Files (x86)\Android\android-sdk\tools\bin"
    ```

3.  Закройте окно командной строки и откройте новое окно командной строки. Создайте новое виртуальное устройство с помощью команды [авдманажер](https://developer.android.com/studio/command-line/avdmanager.html) .    
    ```cmd
    avdmanager create avd -n AVD-Oreo-8.0 -k "system-images;android-26;google_apis;x86"
    ```

4.  При появлении запроса на **Создание настраиваемого профиля оборудования [No]** можно ввести **No** и принять профиль оборудования по умолчанию. Если вы скажите **Да**, **авдманажер** предложит получить список вопросов по настройке профиля оборудования.

## <a name="new-features"></a>Новые функции

В Android Oreo введено множество новых функций и возможностей, таких как каналы уведомлений, эмблемы уведомлений, пользовательские шрифты в XML, загружаемые шрифты, автозаполнение и изображение в картинке. В следующих разделах описаны эти функции и приведены ссылки, которые помогут приступить к их использованию в приложении.



### <a name="notification-channels"></a>Каналы уведомления

*Каналы уведомления* — это категории, определяемые приложением для уведомлений.
Вы можете создать канал уведомления для каждого типа уведомлений, которые необходимо отправить, а также создать каналы уведомлений для отображения вариантов, сделанных пользователями приложения. Новая функция каналов уведомлений позволяет пользователям точно контролировать различные виды уведомлений, которые позволяют им управлять. Например, при реализации приложения для обмена сообщениями можно создать отдельные каналы уведомлений для каждой группы сообщений, созданной пользователем.

[Каналы уведомлений](~/android/app-fundamentals/notifications/local-notifications.md#notif-chan) поясняют, как создать канал уведомлений и использовать его для отправки локальных уведомлений. Пример кода реального мира см. в примере [каналы уведомлений](https://developer.xamarin.com/samples/monodroid/android-o/NotificationChannels) . Этот пример приложения управляет двумя каналами и задает дополнительные параметры уведомлений.



### <a name="notification-badges"></a>Эмблемы уведомлений

Значки уведомлений — это небольшие точки, отображаемые над значками приложений, как показано на следующем снимке экрана:

[![Пример эмблем уведомления на значках приложения](oreo-images/02-badges-sml.png)](oreo-images/02-badges.png#lightbox)

Эти точки указывают на наличие новых уведомлений для одного или нескольких каналов уведомления в приложении, связанном с этим значком &ndash; приложения. это уведомления, которые пользователь еще не отменял или не выполнил действия. Пользователи могут быстро нажать на значок, чтобы просмотреть уведомления, связанные с эмблемой уведомления, открыв или обрабатывая уведомления из меню длительного нажатия аппеаарс.

Дополнительные сведения о индикаторах уведомлений см. в разделе [уведомления](https://developer.android.com/guide/topics/ui/notifiers/notifications.html#Badges) разработчика Android.



### <a name="custom-fonts-in-xml"></a>Пользовательские шрифты в XML

Android Oreo вводит *шрифты в формате XML*, что позволяет включать пользовательские шрифты в качестве ресурсов. Поддерживаются форматы шрифтов OpenType ( **. ОТФ**) и TrueType ( **. ttf**). Чтобы добавить шрифты в качестве ресурсов, выполните следующие действия.

1. Создайте папку **ресурсов или шрифтов** .

2. Скопируйте файлы шрифтов (например, **. ttf** и **. ОТФ** ) в файл Resources **/Font**. 

3. При необходимости переименуйте каждый файл шрифта, чтобы он соответствовал соглашениям об именовании файлов Android (т. е. Используйте только строчные буквы *a – z*, *0-9*и символы подчеркивания в именах файлов). Например, файл `Pacifico-Regular.ttf` шрифта может быть переименован в нечто вроде `pacifico.ttf`.

4. Примените пользовательский шрифт с помощью нового `android:fontFamily` атрибута в XML-макете. Например, следующее `TextView` объявление использует добавленный ресурс Font **паЦифико. ttf** :

   ```xml
   <TextView
     android:text="Example Text in Pacifico Regular"
     android:layout_width="wrap_content"
     android:layout_height="wrap_content"
     android:fontFamily="@font/pacifico" />
   ```

Можно также создать XML-файл семейства шрифтов, описывающий несколько шрифтов, а также сведения о стиле и весе. Дополнительные сведения см. в разделе шрифты разработчика Android [в XML](https://developer.android.com/guide/topics/ui/look-and-feel/fonts-in-xml.html) .


### <a name="downloadable-fonts"></a>Загружаемые шрифты

Начиная с Android Oreo, приложения могут запрашивать шрифты от поставщика, а не объединять их в APK. Шрифты загружаются из сети только по мере необходимости. Эта функция сокращает размер APK, сохраняя память телефона и использование сотовых данных. Эту функцию также можно использовать в версиях API для Android 14 и выше, установив пакет библиотеки поддержки Android 26.

Если приложению требуется шрифт, создайте `FontsRequest` объект (указав шрифт для загрузки) и передайте его `FontsContract` методу для загрузки шрифта. Следующие шаги подробно описывают процесс загрузки шрифтов:

1.  Создайте экземпляр объекта [фонтрекуест](https://developer.android.com/reference/android/provider/FontRequest.html) . 

2.  Подкласс и создайте экземпляр [фонтсконтракт. фонтрекуесткаллбакк](https://developer.android.com/reference/android/provider/FontsContract.FontRequestCallback.html).

3.  Реализуйте метод [фонтрекуесткаллбакк. онтипефацеретриевед](https://developer.android.com/reference/android/provider/FontsContract.FontRequestCallback.html#onTypefaceRetrieved%28android.graphics.Typeface%29) , который используется для завершения запроса шрифта.

4.  Реализуйте метод [фонтрекуесткаллбакк. онтипефацерекуестфаилед](https://developer.android.com/reference/android/provider/FontsContract.FontRequestCallback.html#onTypefaceRequestFailed%28int%29) , который используется для информирования приложения о любых ошибках, происходящих во время обработки запроса шрифта.

5.  Вызовите метод [фонтсконтракт. рекуестфонтс](https://developer.android.com/reference/android/provider/FontsContract.html#requestFonts(android.content.Context,%20android.provider.FontRequest,%20android.os.Handler,%20android.os.CancellationSignal,%20android.provider.FontsContract.FontRequestCallback)) , чтобы получить шрифт от поставщика шрифтов. 

При вызове `RequestFonts` метода сначала проверяется локально кэшированный шрифт (из предыдущего `RequestFont`вызова). Если он не кэширован, он вызывает поставщик шрифтов, получает шрифт асинхронно, а затем передает результаты обратно в приложение, вызывая `OnTypeFaceRetrieved` метод.

Образец [загружаемых шрифтов](https://developer.xamarin.com/samples/monodroid/android-o/DownloadableFonts) демонстрирует использование функции загрузки шрифтов, представленной в Android Oreo. 

Дополнительные сведения о загрузке шрифтов см. в разделе загружаемые [шрифты](https://developer.android.com/guide/topics/ui/look-and-feel/downloadable-fonts.html) для разработчиков Android.



### <a name="autofill"></a>Им

Новая платформа _автозаполнения_ в Android Oreo упрощает для пользователей обработку повторяющихся задач, таких как вход, создание учетной записи и транзакции кредитной карты. Пользователи тратят меньше времени на повторное ввод информации (что может привести к ошибкам ввода). Прежде чем приложение сможет работать с платформой автозаполнения, в параметрах системы необходимо включить службу автозаполнения (пользователи могут включать или отключать автозаполнение).

В примере [аутофиллфрамеворк](https://developer.xamarin.com/samples/monodroid/android-o/AutoFillFramework/) демонстрируется использование платформы автозаполнения. Он включает реализации клиентских действий с представлениями, которые должны быть автоматически заполнены, и службу, которая может предоставлять данные автозаполнения для действий клиента.

Дополнительные сведения о новой функции автозаполнения и о том, как оптимизировать приложение для автозаполнения, см. в разделе Android Developer Автозаполнение [платформы](https://developer.android.com/guide/topics/text/autofill.html) .



### <a name="picture-in-picture-pip"></a>Изображение на рисунке (PIP)

Android Oreo делает возможным запуск действия в режиме изображения на рисунке (PIP) с наложением экрана другого действия. Эта функция предназначена для воспроизведения видео.

Чтобы указать, что действие приложения может использовать режим PIP, установите для следующего флага значение true в манифесте Android:

```xml
android:supportsPictureInPicture
```

Чтобы указать, как действие должно вести себя в режиме PIP, используйте новый объект [пиктуреинпиктурепарамс](https://developer.android.com/reference/android/app/PictureInPictureParams.html) . `PictureInPictureParams`представляет набор параметров, которые используются для инициализации и обновления действия в режиме PIP (например, предпочтительное пропорции действия). В Android Oreo добавлены `Activity` следующие новые методы PIP:

-   [Ентерпиктуреинпиктуремоде](https://developer.android.com/reference/android/app/Activity.html#enterPictureInPictureMode%28android.app.PictureInPictureParams%29) &ndash; помещает действие в режиме PIP. Действие помещается в углу экрана, а остальная часть экрана заполняется предыдущим действием, которое было на экране.

-   [Сетпиктуреинпиктурепарамс](https://developer.android.com/reference/android/app/Activity.html#setPictureInPictureParams%28android.app.PictureInPictureParams%29) &ndash; Обновляет параметры конфигурации PIP действия (например, изменение пропорций).

Пример [пиктуреинпиктуре](https://developer.xamarin.com/samples/monodroid/android-o/PictureInPicture) демонстрирует базовое использование режима "изображение в картинке" (PIP) для карманных устройств, представленных в Oreo. Пример воспроизводит видео, которое остается непрерывным при переключении между режимами экрана и другими действиями.

## <a name="behavior-changes"></a>Изменения в поведении

Android Oreo включает различные изменения в работе системы и API, которые могут оказать влияние на функциональность существующих приложений. Эти изменения описаны ниже.


### <a name="background-execution-limits"></a>Ограничения фонового выполнения

Чтобы улучшить взаимодействие с пользователем, Android Oreo накладывает ограничения на то, какие приложения могут выполняться в фоновом режиме. Например, если пользователь наблюдает за видео или играет игру, то приложение, работающее в фоновом режиме, может нарушать производительность приложения, интенсивно использующего видео на переднем плане. В результате Android Oreo накладывает следующие ограничения на приложения, которые не взаимодействуют напрямую с пользователем:

1.  **Ограничения фоновых служб** &ndash; Когда приложение выполняется в фоновом режиме, оно имеет окно в течение нескольких минут, в течение которого ему по-прежнему разрешено создавать и использовать службы. В конце этого окна Android останавливает фоновую службу приложения и обрабатывает ее как _бездействие_.

2.  **Ограничения вещания** &ndash; В Android 7,0 (API 25) накладываются ограничения на широковещательные передачи, которые приложение регистрирует для получения. Android Oreo делает эти ограничения более строгими. Например, приложения Android Oreo больше не могут регистрировать широковещательные приемники для неявных широковещательных рассылок в своих манифестах.

Дополнительные сведения о новых ограничениях на выполнение в фоновом режиме см. в статье [ограничения фонового выполнения](https://developer.android.com/about/versions/oreo/background.html) для Android Developer.


### <a name="breaking-changes"></a>Критические изменения

Приложения, предназначенные для Android Oreo или более поздней версии, должны изменять свои приложения для поддержки следующих изменений, где это применимо.

- Android Oreo не позволяет устанавливать приоритет отдельных уведомлений. Вместо этого при создании канала уведомления задается рекомендуемый уровень важности. Уровень важности, назначаемый каналу уведомлений, применяется ко всем сообщениям уведомлений, отправляемым в него.

- Для приложений, предназначенных для Android `PendingIntent.GetService()` Oreo, не работает из-за новых ограничений, помещаемых в службы, запущенные в фоновом режиме. Если вы используете Android Oreo, вместо него следует использовать [pendingintents. Broadcast](xref:Android.App.PendingIntent.GetBroadcast*) .
