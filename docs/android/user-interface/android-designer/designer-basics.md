# <a name="xamarinandroid-designer-basics"></a>Основные сведения о конструкторе Xamarin.Android

_В этом разделе рассматриваются возможности конструктора Xamarin.Android, содержатся сведения о запуске конструктора, описание области конструктора и подробно описывается использование панели «Свойства» для изменения свойств мини-приложения._


# <a name="visual-studiotabwindows"></a>[Visual Studio](#tab/windows)

## <a name="launching-the-designer"></a>Запуск конструктора

Конструктор вызывается автоматически при запуске создания макета, или его можно запустить, дважды щелкнув файл макета. Например, дважды щелкнув **activity_main.axml** в **ресурсы > макет** папку загрузит конструктора, как показано на следующем снимке экрана:

[![Экран конструктора в Visual Studio](designer-basics-images/vs/01-open-designer-sml.png)](designer-basics-images/vs/01-open-designer.png#lightbox)

Аналогичным образом, вы можете добавить новый макет, щелкнув правой кнопкой мыши **макета** папку в **обозревателе решений** и выбрав **Добавить > новый элемент... > макет Android**:

[![Диалоговое окно добавления нового элемента](designer-basics-images/vs/02-add-new-layout-sml.png)](designer-basics-images/vs/02-add-new-layout.png#lightbox)

При этом создается новый **.axml** файл макета и загружает их в конструкторе.

## <a name="designer-features"></a>Функции конструктора

Конструктор состоит из нескольких разделов, которые поддерживают его функциональных возможностей, как показано на следующем снимке экрана:

[![Схема панелей конструктора](designer-basics-images/vs/03-designer-features-sml.png)](designer-basics-images/vs/03-designer-features.png#lightbox)

При изменении макета в конструкторе, можно использовать следующие функции для создания и формирование макета:

-   **Область конструктора** &ndash; облегчает визуального создания пользовательского интерфейса, предоставляя изменяемое представление как макет будет отображаться на устройстве. **Конструктора** отображается внутри **панель конструктора** (с **панель инструментов конструктора** расположен над ним).

-   **Панель источника** &ndash; обеспечивает представление базового источника XML, соответствующий проектирования, представленные в **конструктора**.

-   **Панель инструментов конструктора** &ndash; отображает список селекторов: **устройства**, **версии**, **темы**, макет конфигурации и параметры панели действий. **Панель инструментов конструктора** также содержит значки для запуска редактора темы, а также для на сетку конструктора материал.

-   **Панель элементов** &ndash; предоставляет список мини-приложений и макеты, вы можете перетащить в **конструктора**.

-   **Окно "Свойства"** &ndash; список свойств выбранного мини-приложения для просмотра и редактирования.

-   **Структура документа** &ndash; показывает дерево мини-приложений, которые составляют макета. Можно щелкнуть элемент в дереве, чтобы вызвать его, чтобы выбрать на **конструктора**. Кроме того, если щелкнуть элемент в дереве загружает свойства элемента в **свойства** окна.

## <a name="design-surface"></a>Область конструктора

Конструктор позволяет вам перетащите мини-приложения из панели элементов на **конструктора**. При взаимодействии с мини-приложения в конструкторе (путем, либо добавляя новые мини-приложения, либо изменение положения существующие), для пометки точек доступных вставки отображаются вертикальные и горизонтальные линии. В следующем примере новый `Button` перетаскиваемого мини-приложение в **конструктора**:

[![Пример вставки строк в рабочей области конструктора](designer-basics-images/vs/05-insertion-points-sml.png)](designer-basics-images/vs/05-insertion-points.png#lightbox)

Кроме того, можно скопировать мини-приложения: можно использовать копирования и вставки для копирования мини-приложение, или можно перетаскивать существующие мини-приложение, удерживая нажатой клавишу <kbd>CTRL</kbd> ключ.

### <a name="designer-toolbar"></a>Панель инструментов конструктора

**Панель инструментов конструктора** (разместить над **конструктора**) представляет селекторы конфигурации и меню инструментов:

[![Схема панель инструментов конструктора](designer-basics-images/vs/04-toolbar-sml.png)](designer-basics-images/vs/04-toolbar.png#lightbox)

**Панель инструментов конструктора** предоставляет доступ на следующие компоненты:

-   **Альтернативные Селектор макета** &ndash; можно выбрать другой макет версий.

-   **Селектор устройства** &ndash; определяет набор квалификаторов (например, размер экрана, разрешение и доступности клавиатуры), связанные с определенным устройством. Кроме того, можно также добавлять и удалять новые устройства.

-   **Android выбора версий** &ndash; Android версии, которую макета. Конструктор подготавливается к просмотру макет в соответствии с выбранной версии Android.

-   **Выбор темы** &ndash; выбирает тему пользовательского интерфейса для макета.

-   **Выбор конфигурации** &ndash; выбирает конфигурации устройства, такие как *книжной* или *Альбомная*.

-   **Параметры квалификатора ресурса** &ndash; открывает диалоговое окно, в которой представлены раскрывающиеся меню для выбора *языка*, *режим пользовательского интерфейса*, *ночной режим*, и *Круглый экран* параметры.

-   **Параметры панели действий** &ndash; настраивает параметры панели действий для макета.

-   **Редактор тем** &ndash; открывает *редактор тем*, что позволяет настраивать элементы выбранной темы.

-   **Материала бланк** &ndash; включает или отключает *материал бланк*. Элемент раскрывающееся меню рядом на сетку конструктора материал открывает диалоговое окно, можно настроить в сетке.

Каждый из этих компонентов является более подробно в следующих разделах:

-   [Квалификаторы ресурсов и параметры визуализации](~/android/user-interface/android-designer/resource-qualifiers.md) предоставляет подробные сведения о **селектора устройства**, **Android выбора версий**, **темы селектора**, **Выбора конфигурации**, **параметры квалификации ресурса**, и **параметры панели действий**.

-   [Альтернативные представления с макетом](~/android/user-interface/android-designer/alternative-layout-views.md) содержит сведения об использовании **Селектор макета альтернатива**.

-   [Функции разработки Xamarin.Android конструктор материал](~/android/user-interface/android-designer/material-design-features.md) предоставляет исчерпывающий обзор **редактор тем** и **материал бланк**.

### <a name="context-menu-commands"></a>Команды контекстного меню

Доступно контекстное меню в **конструктора** и в **Структура документа**. Это меню отображаются команды, которые доступны для выбранного мини-приложения и его контейнером, облегчая процесс для выполнения операций в контейнерах (который не всегда легко выбирать на **конструктора**). Вот пример контекстного меню:

[![Пример контекстное меню, если щелкнуть правой кнопкой мыши область конструктора](designer-basics-images/vs/06-context-menu-sml.png)](designer-basics-images/vs/06-context-menu.png#lightbox)

В этом примере щелкните правой кнопкой мыши `TextView` открывается контекстное меню, которое предоставляет несколько вариантов:

-   **LinearLayout** &ndash; открывается подменю для редактирования `LinearLayout` предком `TextView`.

-   **Удалить**, **копирования**, и **Вырезать** &ndash; операций, которые применяются к правой кнопкой мыши, `TextView`.


### <a name="zoom-controls"></a>Элементы управления масштабом

**Конструктора** поддерживает масштабирование с помощью нескольких элементов управления, как показано:

[![Схема конструктора элементы управления масштабом](designer-basics-images/vs/07-zoom-controls-sml.png)](designer-basics-images/vs/07-zoom-controls.png#lightbox)

Эти элементы управления облегчают определенных областей пользовательского интерфейса в конструкторе см. в разделе:

-   **Выделение контейнеров** &ndash; отражения контейнеры от **конструктора** таким образом, чтобы их можно было легко найти при увеличение и уменьшение.

-   **Обычный** &ndash; отрисовывает макета для размера x таким образом, чтобы можно было увидеть, как будет выглядеть макет в разрешении выбранного устройства.

-   **По размеру окна** &ndash; задает масштаб, так что всего макета становится видимым в области конструктора.

-   **Увеличить масштаб** &ndash; добавочное увеличение каждого щелчком мыши увеличение макета.

-   **Уменьшить масштаб** &ndash; уменьшает масштаб постепенно каждого щелчка, делая макет казаться меньше, чем в рабочей области конструирования.

Обратите внимание, что выбранный масштаб не влияет на пользовательском интерфейсе приложения во время выполнения.

## <a name="switching-between-design-and-source-panes"></a>Переключение между панелями конструирования и источника

Center полосу между **разработки** и **источника** панелей, есть несколько кнопок, которые используются для изменения как **разработки** и **источника**панели отображаются:

[![Расположения кнопок панели отображения](designer-basics-images/vs/25-pane-buttons-sml.png)](designer-basics-images/vs/25-pane-buttons.png#lightbox)

Эти кнопки сделайте следующее:

-   **Проектирование** &ndash; эту кнопку, верхний, **разработки**, выбирает **разработки** области. При нажатии этой кнопки, **элементов** и **свойства** области, включены и **панель инструментов редактора текстов** не отображается. При **свернуть** кнопки (см. ниже), **разработки** отдельно открыть область без **источника** области.

-   **Поменять панели местами** &ndash; кнопки (который имеет вид двух противоположными стрелок) меняет местами **разработки** и **источника** области, чтобы **источника** области находится на слева и **разработки** области — справа. Щелкнув ее еще раз переключения этих панелей обратно в исходное расположение.

-   **Источник** &ndash; выбирает эту кнопку (которая напоминает двумя противоположными угловые скобки) **источника** области. При нажатии этой кнопки, **элементов** и **свойства** панели будут отключены и **панель инструментов редактора текстов** является видимым в верхней части Visual Studio. При **свернуть** кнопки (см. ниже), щелкнув **источника** кнопку **источника** области, а не **разработки** область.

-   **Вертикальное разделение** &ndash; отображает эту кнопку (который имеет вид вертикальной черты), **разработки** и **источника** панелей side-by-side. Это расположение по умолчанию.

-   **Горизонтальное разделение** &ndash; отображает кнопки (который похож на горизонтальную панель), **разработки** над панелью **источника** области. **Поменять панели местами** можно щелкнуть для размещения **источника** над панелью **разработки** области.

-   **Свернуть панель** &ndash; кнопки (который похож на две угловые скобки, вправо) «сворачивает» отображение панели двумя **разработки** и **источника** в одно представление одного из Эти области.
    Эта кнопка становится кнопкой **развернуть панель** кнопки, (в виде двух влево угловые скобки), который можно щелкнуть для возврата в представлении панели двумя (**разработки** и **источника**) режим отображения.

Когда **Свернуть область** нажатии только **разработки** откроется панель. Тем не менее, можно щелкнуть **источника** кнопку, чтобы вместо этого просмотреть только **источника** области. Нажмите кнопку **разработки** кнопку, чтобы вернуться к **разработки** области.

## <a name="source-pane"></a>Панель источника

**Источника** панель отображает базовую структуру, в XML-источник **конструктора**. Так как оба представления доступны в то же время, можно для создания макета пользовательского интерфейса, неоднократно переключаясь между визуальным представлением конструктора и базовый источник XML для входа в Интернете:

[![Пример XML-источник в исходной области](designer-basics-images/vs/22-source-pane-w158-sml.png)](designer-basics-images/vs/22-source-pane-w158.png#lightbox)

Изменения, внесенные в XML-источник, немедленно отображаются на **конструктора**; изменения, внесенные в **конструктора** вызвать источник XML, отображаемый в **источника** панели обновлено соответствующим образом. При внесении изменений в XML в **источника** области, автозавершение и возможности IntelliSense доступны скорость разработки пользовательского интерфейса на основе XML, как описано далее.

Для большей легкостью навигации при работе с длинные файлы XML **источника** область поддерживает полосы прокрутки Visual Studio (как показано в правой части на предыдущем снимке экрана). Дополнительные сведения о полосы прокрутки, см. в разделе [как отслеживание кода за счет настройки полосы прокрутки](https://msdn.microsoft.com/library/dn237345.aspx).

После ввода нового атрибута для мини-приложение и начать присвоить ей значение, IntelliSense появляется после символа триггера является типизированным и предоставляет список допустимых значений для этого атрибута. Например, после ввода первого кавычек для `android:layout_width` в следующем примере селектора автозавершение, предлагающее предоставить список допустимых значений для этой ширины:

[![Пример IntelliSense для ширины макета](designer-basics-images/vs/24-intellisense-w158-sml.png)](designer-basics-images/vs/24-intellisense-w158.png#lightbox)

В нижней части этого контекстного меню которые две кнопки (выделено красным на приведенном выше снимке экрана). Щелкнув **ресурсы проекта** слева кнопку ограничивает список ресурсов, которые являются частью проекта приложения, щелкая мышью **ресурсы платформы** кнопку справа ограничивает список для Отображение ресурсов, доступных из .NET framework.
Эти кнопки позволяют включать и выключать: можно щелкать их еще раз, чтобы отключить действие фильтра, каждый из которых обеспечивает.



## <a name="properties-pane"></a>Панель свойств

Конструктор поддерживает редактирование свойств мини-приложения с помощью **свойства** области: 

![Снимок экрана окна свойств](designer-basics-images/vs/08-property-pad.png)

Свойства, перечисленные в **свойства** изменение области, в зависимости от выбранного мини-приложение на **конструктора**.

### <a name="default-values"></a>Значения по умолчанию

Свойства большинство мини-приложения будут отображаться в **свойства** окна так, как наследовать их значения из выбранного Android темы.
**Свойства** окна будут отображаться только значения, заданные явным образом для выбранного мини-приложения; он не будут отображаться значения, которые наследуются от темы.

### <a name="referencing-resources"></a>Создание ссылок на ресурсы

Некоторые свойства могут ссылаться на ресурсы, которые определены в файлы, не являющиеся макет **.axml** файла. Наиболее распространенных случаев этого типа являются `string` и `drawable` ресурсы. Тем не менее, ссылки можно также использовать другие ресурсы, такие как `Boolean` значения и измерения. Если свойство поддерживает ссылки на ресурсы, значок "Обзор" (квадрат) отображается рядом с записью текста для свойства. Эта кнопка открывает выделения ресурсов, при щелчке.

Например, на следующем рисунке показан параметры, доступные при щелчке затемненном квадрат справа от текстового поля для `Text` мини-приложением в **свойства** окна:

[![Пример списка параметров текста](designer-basics-images/vs/09-text-options-sml.png)](designer-basics-images/vs/09-text-options.png#lightbox)

Когда **ресурсов...**  нажатии **Выбор ресурса** появится диалоговое окно:

[![Снимок экрана с примером ресурсы с несколько ресурсов в списке](designer-basics-images/vs/09b-resources-w158-sml.png)](designer-basics-images/vs/09b-resources-w158.png#lightbox)

## <a name="editing-properties-inline"></a>Редактирование встроенного свойства

Конструктор Android поддерживает прямое редактирование некоторые свойства в области конструктора (чтобы у вас нет этих свойств в списке свойств поиска). Свойства, которые можно непосредственно редактировать включать текст, поля и размер.

### <a name="text"></a>Текста

Свойства text Некоторые мини-приложения (такие как `Button` и `TextView`), можно изменить непосредственно на поверхности разработки. Дважды щелкните мини-приложение будет поместить его в режим редактирования, как показано ниже:

[![Текстовый ресурс для строки hello](designer-basics-images/xs/14-text-resource-sml.png)](designer-basics-images/xs/14-text-resource.png#lightbox)

Можно ввести новое значение текста, или можно ввести новую строку ресурса. В следующем примере `@string/hello` с текстом, который заменяется ресурсов `CLICK THIS BUTTON`:

[![SHIFT + ВВОД, чтобы автоматически связать текст в новый ресурс](designer-basics-images/xs/15-shift-enter-resource-sml.png)](designer-basics-images/xs/15-shift-enter-resource.png#lightbox)

Это изменение сохраняется в мини-приложении `text` свойство; он не изменяет значение, присваиваемое `@string/hello` ресурсов.
Когда в новую строку текста, можно нажать клавишу <kbd>Shift</kbd> +
<kbd>ввод</kbd> автоматической связи введенного текста в новый ресурс.

### <a name="margin"></a>Поля

Если щелкнуть мини-приложение в первый раз, отображаются маркеры полей. При перемещении указателя мыши на один из маркеров в конструкторе отображается свойство, которое приведет к изменению дескриптор (как показано ниже, для `layout_marginLeft` свойство):

### <a name="size"></a>Размер

Как упоминалось ранее, можно перейти в режим изменения размера, щелкнув мини-приложение, пока он уже выбран. Щелкните треугольный маркер для установки размера для указанного измерения для `wrap_content`:

Кроме того, можно перетащить маркер изменения размера циклическая (как показано на снимке экрана выше) для изменения размера мини-приложение, чтобы это произвольное `dp` значение. ## <a name="document-outline"></a>Структура документа

**Структура документа** отображается иерархия мини-приложения макета.
В следующем примере, содержащем `LinearLayout` выбран мини-приложения:

[![Структура документа](designer-basics-images/xs/22-outline-view-sml.png)](designer-basics-images/xs/22-outline-view.png#lightbox)

Структура выбранного мини-приложения (в данном случае `LinearLayout`) также выделяется в рабочей области конструирования. Выбранного мини-приложения в структуре документа синхронизируются с соответствующими командами в рабочей области конструирования. Это полезно для выбора Просмотр групп, которые не всегда легко выбрать в области конструктора.

Структура документа поддерживает копирование и вставка, или можно использовать перетаскивание перетаскивания. Операции перетаскивания поддерживается в структуре документа в область конструктора, а также из области конструктора в структуру документа. Кроме того щелкнув правой кнопкой мыши элемент на панели структуры документа отображается контекстное меню для этого элемента (этого же контекстного меню, открываемом вы же мини-приложения в области конструктора, щелкните правой кнопкой мыши).
