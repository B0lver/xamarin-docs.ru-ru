## <a name="application-spotlight-and-settings-icons"></a>Приложение, Spotlight и параметры значков

Таким же образом, для элементов управления пользовательского интерфейса и в качестве значков документов, в приложение Xamarin.iOS можно использовать графические активы ресурсы изображений можно использовать для предоставления значки приложения. Далее на снимках экрана из iPad иллюстрирует три использует значки в iOS:

- **Значок приложения** -каждое приложение iOS необходимо определить значка приложения. Это значок, который пользователь сможет нажать с начального экрана iOS для запуска приложения. Кроме того этот значок используется Game Center, если применимо. Пример 

    [![](app-icons-images/000.png "Значок приложения")](app-icons-images/000-full.png#lightbox)
- **Spotlight значок** - всякий раз, когда пользователь вводит имя приложения в поиске Spotlight, этот значок будет отображаться. Пример 

    [![](app-icons-images/000a.png "Значок Spotlight")](app-icons-images/000a-full.png#lightbox)
- **Значок "Параметры"** — Если пользователь вводит **параметры** приложение на своем устройстве iOS, этот значок будет отображаться в конце **параметры** список для приложения. Пример 

    [![](app-icons-images/000b.png "Значок \"Параметры\"")](app-icons-images/000b-full.png#lightbox)

Для обеспечения поддержки значок типы, необходимые для приложения Xamarin.iOS операционной системы iOS 5 через iOS 9 (или более поздней) потребуются следующие размеры изображений средства и способы их устранения:

## <a name="managing-icons-with-asset-catalogs"></a>Управление значки с каталоги активов

Для значков, специальный `AppIcon` набор изображений, которые могут добавляться к `Assets.xcassets` файл в проекте приложения. Все версии образа, должны поддерживать все разрешения, включаются в _xcasset_ и сгруппированы вместе. Специальный редактор в Visual Studio для Mac позволяет разработчику необходимо включить, а затем эти образы установки в графическом виде.

Чтобы использовать каталог ресурсов, сделайте следующее:

# <a name="visual-studio-for-mactabvsmac"></a>[Visual Studio для Mac](#tab/vsmac)

1. Дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования.
2. Прокрутите вниз до раздела **значки приложений** раздел.
3. Из **источника** раскрывающемся списке, убедитесь, **AppIcon** выбран: 

    ![](app-icons-images/migrate01.png "Убедитесь, что выбрана AppIcon")
4. Из **обозревателе решений**, дважды щелкните `Assets.xcassets` файл, чтобы открыть его для редактирования: 

    ![](app-icons-images/asset01.png "Файл Assets.xcassets в обозревателе решений")
5. Выберите `AppIcon` из списка ресурсов для отображения `Icon Editor`:

    ![](app-icons-images/asset02.png "Редактор AppIcon")
6. Либо щелкните значок типа выберите файл изображения для требуемого типа и размера или изображения из папки и перетащите его на нужный размер.
7. Нажмите кнопку **откройте** кнопку, чтобы включить изображения в проект и задайте его в xcasset.
8. Повторите эти действия для всех необходимых образов.

# <a name="visual-studiotabvswin"></a>[Visual Studio](#tab/vswin)

1. Дважды щелкните **Info.plist** файл **обозревателе решений**:

    ![](app-icons-images/icon01w.png "Выберите файл Info.plist")
2. Щелкните **визуальные активы** вкладку и выберите команду **использовать каталог ресурсов** под кнопкой **значки приложений**: 

    ![](app-icons-images/icon02w.png "Перейдите на вкладку визуальные активы")
4. Из **обозревателе решений**, разверните **каталог активов** папку: 

    ![](app-icons-images/image009.png "Разверните папку каталог активов")
5. Дважды щелкните **мультимедиа** файл, чтобы открыть его в редакторе: 

    ![](app-icons-images/image010.png "Откройте файл мультимедиа в редакторе")
6. В разделе **обозревателе свойств** можно выбирать различные типы и размеры значков, которые требуется.
7. Щелкните значок типа и выберите файл изображения для требуемого типа и размера.
8. Нажмите кнопку **откройте** кнопку, чтобы включить изображения в проект и задайте его в xcasset.
9. Повторите эти действия для всех необходимых образов.

-----

Это предпочтительный метод включая и управления активами изображений, которые будут использоваться для предоставления приложения, Spotlight и параметры значков для приложения.

### <a name="migrating-from-infoplist-to-asset-catalogs"></a>Переход с Info.plist на каталоги активов

Для существующего приложения Xamarin.iOS с помощью `Info.plist` файл для управления его значки, настоятельно рекомендуется, что разработчик переключить использовать `AppIcons` ресурса изображения внутри `Assets.xcassets`.

Выполните следующие действия:

# <a name="visual-studio-for-mactabvsmac"></a>[Visual Studio для Mac](#tab/vsmac)

1. Дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования.
2. Прокрутите вниз до раздела **значки приложений** раздел.
3. Из **источника** раскрывающемся списке выберите **миграции в каталоги активов**: 

    ![](app-icons-images/migrate02.png "Выберите перенос в каталоги активов")
4. Существующие значки, определенные в `Info.plist` файла будут перенесены в `AppIcons` образа наборе добавляемый `Assets.xcassets`: 

     ![](app-icons-images/migrate03.png "Задать изображение AppIcons в Assets.xcassets")

# <a name="visual-studiotabvswin"></a>[Visual Studio](#tab/vswin)

1. Дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования.
2. Щелкните на iPhone разделе "значки": 

    ![](app-icons-images/image007.png "Редактор значков iPhone заливки")
3. Прокрутите вниз до раздела **значки** раздел.
4. Из **каталог активов** раскрывающемся списке выберите **каталоги активов используйте**.
5. Существующие значки, определенные в `Info.plist` файла будут перенесены в `Images` добавлен в набор `Assets.xcassets`.
6. Сохраните изменения в файле `Info.plist`.

## <a name="itunes-artwork"></a>Иллюстрации iTunes

При использовании метода Ad-Hoc доставки приложения (либо для корпоративных пользователей, либо для бета-тестирование на реальных устройствах), разработчик также необходимо включить 512 x 512 и 1024 x 1024 образ, который будет использоваться для представления приложения в iTunes.

Чтобы указать иллюстрации iTunes, сделайте следующее:

# <a name="visual-studio-for-mactabvsmac"></a>[Visual Studio для Mac](#tab/vsmac)

1. Дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования.
2. Прокрутите страницу до **иллюстрации iTunes** в редакторе: 

    ![](app-icons-images/itunes01.png "Прокрутите страницу до iTunes иллюстрации разделе редактора")
3. Для отсутствии изображения щелкните эскиз в редакторе, выберите файл изображения для требуемой иллюстрацией iTunes в диалоговом окне открытия файла и нажмите кнопку **ОК** кнопки.
4. Повторите этот шаг, пока все необходимые образы были указаны для приложения.

# <a name="visual-studiotabvswin"></a>[Visual Studio](#tab/vswin)

1. Дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования.

2. Щелкните **визуальные активы** вкладку и раскройте **иллюстрации iTunes**: 

    ![](app-icons-images/itunes01w.png "Изменение иллюстрации в Visual Studio iTunes")
4. Для отсутствии изображения щелкните эскиз в редакторе, выберите файл изображения для требуемой иллюстрацией iTunes в диалоговом окне открытия файла и нажмите кнопку **откройте** кнопки.
5. Повторите этот шаг, пока все необходимые образы были указаны для приложения.
