## <a name="application-spotlight-and-settings-icons"></a>Значки приложений, Spotlight и параметров

Точно так же, как приложение Xamarin. iOS может использовать ресурсы изображений для элементов управления пользовательского интерфейса и в качестве значков документа, ресурсы изображений можно использовать для предоставления значков приложений. На следующих снимках экрана для iPad показаны три способа использования значков в iOS:

- **Значок приложения** — каждое приложение iOS должно определять значок приложения. Это значок, который будет коснуться пользователь на начальном экране iOS для запуска приложения. Кроме того, этот значок используется Game Center, если применимо. Пример 

    [![](app-icons-images/000.png "Application Icon")](app-icons-images/000-full.png#lightbox)
- **Значок "Прожектор** ". когда пользователь вводит имя приложения в поиск Spotlight, отображается этот значок. Пример 

    [![](app-icons-images/000a.png "Spotlight Icon")](app-icons-images/000a-full.png#lightbox)
- **Значок "Параметры** ". Если пользователь введет приложение **параметров** на устройстве iOS, этот значок будет отображаться в конце списка **параметров** для приложения. Пример 

    [![](app-icons-images/000b.png "Settings Icon")](app-icons-images/000b-full.png#lightbox)

Для поддержки всех типов значков, необходимых для приложения Xamarin. iOS, предназначенного для iOS 5 – iOS 9 (или более поздней версии), требуются следующие размеры и разрешения ресурса изображения.

## <a name="managing-icons-with-asset-catalogs"></a>Управление значками с помощью каталогов активов

Для значков Специальный набор изображений `AppIcon` можно добавить в файл `Assets.xcassets` в проекте приложения. Все версии образа, необходимые для поддержки всех разрешений, включены в _xcasset_ и группируются вместе. Специальный редактор в Visual Studio для Mac позволяет разработчику включать и настраивать эти изображения в графическом виде.

Чтобы использовать каталог активов, выполните следующие действия.

# <a name="visual-studio-for-mac"></a>[Visual Studio для Mac](#tab/macos)

1. Дважды щелкните файл `Info.plist` в **Обозреватель решений** , чтобы открыть его для редактирования.
2. Прокрутите вниз до раздела " **значки iPhone** ".
3. Нажмите кнопку **Миграция в каталог активов** :

    ![](app-icons-images/migrate01.png "Ensure AppIcon is selected")

4. В **Обозреватель решений**дважды щелкните файл `Assets.xcassets`, чтобы открыть его для редактирования: 

    ![](app-icons-images/asset01.png "The Assets.xcassets file in the Solution Explorer")

5. Выберите `AppIcon` из списка ресурсов для показа `Icon Editor`:

    ![](app-icons-images/asset02.png "The AppIcon editor")

6. Щелкните этот тип значка и выберите файл изображения для требуемого типа/размера или перетащите изображение из папки и поместите его на нужный размер.
7. Нажмите кнопку **Открыть** , чтобы включить изображение в проект и задать его в xcasset.
8. Повторите для всех необходимых образов.

# <a name="visual-studio"></a>[Visual Studio](#tab/windows)

1. Дважды щелкните сведения о * *.  \* * файл в **Обозреватель решений**:

    ![](app-icons-images/icon01w.png "Select Info.plist")

2. Откройте вкладку " **визуальные ресурсы** " и нажмите кнопку " **использовать каталог ресурсов** " в разделе " **значки приложений**": 

    ![](app-icons-images/icon02w.png "Select the Visual Assets tab")

    Если кнопка отсутствует, а раскрывающийся список, то каталог активов уже добавлен в этот проект.

3. В **Обозреватель решений**разверните папку **каталога активов** : 

    ![](app-icons-images/image009.png "Expand the Asset Catalog folder")

4. Дважды щелкните файл **мультимедиа** , чтобы открыть его в редакторе: 

    ![](app-icons-images/image010.png "Open the Media file in the editor")

5. В **обозревателе свойств** разработчик может выбрать различные типы и размеры значков.
6. Щелкните данный тип значка и выберите файл изображения для требуемого типа или размера.
7. Нажмите кнопку **Открыть** , чтобы включить изображение в проект и задать его в xcasset.
8. Повторите для всех необходимых образов.

-----

Это предпочтительный метод включения и управления ресурсами изображений, которые будут использоваться для предоставления приложений, важных значков и параметров приложения.

## <a name="itunes-artwork"></a>Иллюстрации iTunes

При использовании специального метода доставки приложения (для корпоративных пользователей или для тестирования бета-версии на реальных устройствах) разработчику также необходимо включить 512 x 512 и образ 1024x1024, который будет использоваться для представления приложения в iTunes.

Чтобы указать иллюстрации iTunes, сделайте следующее:

# <a name="visual-studio-for-mac"></a>[Visual Studio для Mac](#tab/macos)

1. Дважды щелкните файл `Info.plist` в **Обозреватель решений** , чтобы открыть его для редактирования.
2. Прокрутите страницу редактора до раздела **iTunes** : 

    ![](app-icons-images/itunes01.png "Scroll to the iTunes Artwork section of the editor")
3. Для любого отсутствующего изображения щелкните эскиз в редакторе, выберите файл изображения для нужной иллюстрации iTunes в диалоговом окне Открыть файл и нажмите кнопку **ОК** .
4. Повторите этот шаг, пока не будут указаны все необходимые образы для приложения.

# <a name="visual-studio"></a>[Visual Studio](#tab/windows)

1. Дважды щелкните файл `Info.plist` в **Обозреватель решений** , чтобы открыть его для редактирования.

2. Перейдите на вкладку " **визуальные ресурсы** " и разверните **иллюстрацию iTunes**: 

    ![](app-icons-images/itunes01w.png "Editing iTunes Artwork in Visual Studio")
3. Для любого отсутствующего изображения щелкните эскиз в редакторе, выберите файл изображения для нужной иллюстрации iTunes из диалогового окна открыть файл и нажмите кнопку **Открыть** .
4. Повторите этот шаг, пока не будут указаны все необходимые образы для приложения.
