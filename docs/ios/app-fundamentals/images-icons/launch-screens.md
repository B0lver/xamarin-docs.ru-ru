# <a name="launch-screens"></a>Запуска экранов

_В этой статье описывается создание приложения запуска экрана для всех устройств iOS в любой разрешение и ориентацию, с помощью одной единой раскадровки._

Перед iOS 8. Создание экрана запуска для приложения iOS требуется разработчику обеспечить актива образ для каждого из различных устройств и решения, в которых приложение может выполняться. С момента выпуска iOS 8 однако ее можно использовать одну раскадровку единой создание экрана запуска, верны во всех случаях.

Это краткое Пошаговое руководство описывает создание экрана запуска раскадровки, предоставляемые по умолчанию в новом проекте или с раскадровкой вручную добавить в существующий проект. Затем демонстрируется использование iOS конструктора раскадровки для задания ограничений на эти представления и убедитесь, что раскадровку верны для различных устройств и ориентации Добавление представление изображения и метки.

<a name="storyboard" />

## <a name="managing-launch-screens-with-storyboards"></a>Управление экраны запуска с помощью раскадровки

В iOS 8 (и более поздние версии) разработчик может создать специальные единой раскадровки для предоставления экрана запуска, вместо того чтобы использовать один или несколько статических запуска образов. При создании запуск раскадровки в конструкторе iOS используйте классы размер и автоматический макет определение различных разметок для различных сред. С помощью классов размер и автоматический макет, разработчик можно создать экран одного запуска, удобная на всех устройствах и отображать сред.

# <a name="visual-studio-for-mactabvsmac"></a>[Visual Studio для Mac](#tab/vsmac)

1. В Visual Studio для Mac, создайте новый проект, выбрав **файл > новое решение** и выбрав **одного представления приложения**: 

    ![Окно нового проекта с помощью единого приложения представление выбранного](launch-screens-images/launch01.png)

    - По умолчанию новый проект включает **LaunchScreen.storyboard** файл, определяющий интерфейс экрана запуска. 
    - Вместо этого добавление экрана запуска раскадровки в существующий проект, щелкните правой кнопкой мыши имя проекта в **Pad решения** и выберите **Добавить > новый файл...**  , а затем выберите **запуска экрана**:

    ![Окно новый файл с iOS выбранного экрана запуска](launch-screens-images/launch01b.png)

    - Назовите файл **LaunchScreen** или другое имя по своему выбору.

2. Настройте проект для использования соответствующего раскадровки для его запуска экрана.

    - Дважды щелкните **Info.plist** файла в **Pad решения** чтобы открыть его для редактирования.
    - В **запуска изображения** статьи, убедитесь, что **запуска экрана** равно имени соответствующего раскадровки:

    ![Экран запуска селектор в файле Info.plist.](launch-screens-images/launch02.png)

    - По умолчанию новый проект настроен для использования **LaunchScreen.storyboard** как экран его запуска.

3. Добавить изображение в **Assets.xcassets** активов каталога, чтобы он доступен для вывода на экран запуска. Дополнительные сведения см. в разделе [Добавление изображения, задаваемое активов каталога изображения](~/ios/app-fundamentals/images-icons/displaying-an-image.md) раздел [отображения изображения](~/ios/app-fundamentals/images-icons/displaying-an-image.md) руководства.

4. Откройте **LaunchScreen.storyboard** для редактирования, дважды щелкнув его в **Pad решения**.

5. Выберите устройства и ориентация, на котором для предварительного просмотра запуска раскадровки экрана в конструкторе iOS. Откройте панель выбора устройства, на нижней панели инструментов и выберите **iPhone 4S** и **Книжная**.

    ![Выбор инструментов устройства](launch-screens-images/launch05.png)

    - Обратите внимание, что устройство и ориентацию изменяет только как iOS конструктор выполняет предварительный просмотр конструктора. Вне зависимости от выбора, совершенные здесь, добавленным ограничения применяются для всех устройств и ориентацию, если не **изменить признаки** кнопку использовался для указания, в противном случае. 

6. Задать **фона** цвет основного представления View Controller. Выберите представление, щелкнув в середине View-Controller и изменить цвет фона с помощью **Pad свойства**:

    ![Одно представление с цветом фона фиолетовой](launch-screens-images/launch06.png)

7. Добавить **представление изображения** экран запуска и установите его источник **изображения**:

    - Перетащите **представление изображения** из **панель элементов** в центр представления.
    - С **представление изображения** выбранных в **мини-приложение** раздел **панель свойств** задать **изображения** задаваемого свойства изображения уже Добавляемый **Assets.xcassets** каталога активов. Измените положение и размер **представление изображения** при необходимости:
    
    ![Представление изображения с набором свойств изображения](launch-screens-images/launch07.png)

8. Добавить **метка** ниже **представление изображения** и использовать **Pad свойства** для задайте его атрибуты: 

    ![Метка с набором текст и цвет](launch-screens-images/launch08.png)

9. Переключиться в режим редактирования ограничение с помощью правой кнопки в **инструментов ограничения**:
    
    ![Кнопка режима редактирования ограничения](launch-screens-images/launch09.png)

10. Добавление ограничений для **представление изображения**, задание, его высота и ширина и центрирование его по горизонтали и вертикали:

    ![Представление изображения с ограничениями макета](launch-screens-images/launch10.png)

    - Дополнительные сведения о том, как добавить ограничения см. в разделе [автоматический макет с помощью конструктора Xamarin для iOS](~/ios/user-interface/designer/designer-auto-layout.md).

11. Добавление ограничений для **метка**, выравнивание по горизонтали центру, предоставляя ему высоты и ширины и позиционирования фиксированный расстояние по вертикали от **представление изображения**:

    ![Метка с ограничениями макета](launch-screens-images/launch11.png)

12. Проверьте другие устройства и ориентации экрана, чтобы убедиться, что проект выглядит должным образом во всех сценариях. В случаях, где должны быть выполнены для конкретного устройства или ориентацией корректировки, использовать **изменить признаки** кнопку, чтобы добавить ограничения для определенного размера классов:

    ![К просмотру как iPhone X с использованием альбомной ориентации экрана запуска](launch-screens-images/launch12.png)

13. Сохраните изменения в раскадровку. Запустить приложение на симуляторе или устройства, и экран запуска отображаются при запуске приложения.

## <a name="migrating-to-launch-screen-storyboards"></a>Перенос для запуска раскадровки экрана

# <a name="visual-studio-for-mactabvsmac"></a>[Visual Studio для Mac](#tab/vsmac)

При обновлении существующего приложения для использования раскадровки для его запуска экранов, щелкните правой кнопкой мыши **имя проекта** в **обозревателе решений** и выберите **добавить**  >  **Новый файл...** . Выберите **iOS** > **запуска экрана** и нажмите кнопку **New** кнопки:

![](launch-screens-images/storyboard02.png "Выберите экран запуска iOS")

Далее дважды щелкните `Info.plist` файла в **обозревателе решений** чтобы открыть его для редактирования. В разделе **запуска экрана**, выберите новый файл раскадровки, созданной ранее.

![](launch-screens-images/storyboard09.png "Выберите новый файл раскадровки, созданной выше")


Чтобы использовать новую раскадровку, экран запуска, выполните следующее:

1. Дважды щелкните `Info.plist` файла в **обозревателе решений** чтобы открыть его для редактирования.
2. Перейдите к пункту **универсальной образов запустите** раздел в редакторе откройте **запуска экрана** раскрывающийся список и выберите имя раскадровки, созданной ранее: 

    ![](launch-screens-images/storyboard08.png "Настройка экрана запуска в раскадровку")

# <a name="visual-studiotabvswin"></a>[Visual Studio](#tab/vswin)

1. Щелкните правой кнопкой мыши имя проекта в **обозревателе решений** и выберите **добавить** > **новый файл...** : 

    ![](launch-screens-images/image012.png "Добавьте новый файл")
2. Введите имя для экрана запуска и нажмите кнопку **добавить** кнопки: 

    ![](launch-screens-images/image013.png "Введите имя для экрана запуска")
3. В **обозревателе решений**, дважды щелкните файл созданный раскадровки, чтобы открыть его для редактирования.
4. Убедитесь, что **размер класса** равно **любой: любой** и **представление как** — **универсального**: 

    ![](launch-screens-images/image016.png "Убедитесь, что размер класса в любой: любой и представление как является универсальным")
5. Сборка экран запуска из классов, размер, простые элементы пользовательского интерфейса (например, `UIImageView`) и изображения, которые включены в пакет приложения: 

    ![](launch-screens-images/image017.png "Экран запуска в конструкторе iOS сборки")
6. Сохраните изменения в раскадровку.
