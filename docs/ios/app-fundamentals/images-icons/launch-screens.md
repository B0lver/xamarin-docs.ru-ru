# <a name="launch-screens-for-xamarinios-apps"></a>Запуска экранов для приложений Xamarin.iOS

_В этой статье объясняется, как создать приложение запуска экрана для всех устройств iOS, в любой разрешения и ориентации, с помощью одной единой раскадровки._

Прежде чем iOS 8 Создание экрана запуска для приложения iOS требуется разработчику предоставить ресурс изображения для каждого из различных конструктивных параметрах устройств и способы их устранения, в которых приложение может работать. Начиная с выпуска iOS 8 однако ее можно использовать одну раскадровку единой создание экрана запуска, который выглядит правильно, во всех случаях.

Это краткое Пошаговое руководство описывает создание экран запуска с помощью раскадровки, предоставляемых по умолчанию в новый проект или с помощью раскадровки вручную добавить в существующий проект. Затем демонстрирует использование конструктора iOS для добавления в представлении изображения и метки к раскадровке, задать ограничения для этих представлениях и убедитесь, что раскадровка выглядит правильно, для различных устройств и ориентации.

<a name="storyboard" />

## <a name="managing-launch-screens-with-storyboards"></a>Управление экранов запуска с помощью Storyboard

В iOS 8 (и более поздние версии) разработчик может создать специальные единой раскадровки для предоставления экрана запуска, вместо того чтобы использовать один или несколько статических запуска образов. При создании запуска раскадровки в конструкторе iOS, используйте классы размера и автоматический макет определить различные разметки для различных сред. С помощью классов размера и автоматический макет, разработчик может создать экран одного запуска, в который хорошо выглядит на всех устройствах и отображения сред.

# <a name="visual-studio-for-mactabmacos"></a>[Visual Studio для Mac](#tab/macos)

1. Создайте новый проект в Visual Studio для Mac, выбрав **файл > новое решение** и выбрав **приложение с одним представлением**: 

    ![Окно нового проекта с выбранной приложение с одним представлением](launch-screens-images/launch01.png)

    - По умолчанию новый проект включает **LaunchScreen.storyboard** файл, определяющий интерфейс экрана запуска. 
    - Вместо этого добавить раскадровки для экрана запуска в существующий проект, щелкните правой кнопкой мыши имя проекта в **панели решения** и выберите **Добавить > новый файл...**  , а затем выберите **запуска экрана**:

    ![В окне «новый файл» с iOS выбрав экран запуска](launch-screens-images/launch01b.png)

    - Назовите файл **LaunchScreen** или другое имя по своему выбору.

2. Настройте проект для использования соответствующих раскадровки для экрана его запуска.

    - Дважды щелкните **Info.plist** файл **панели решения** чтобы открыть его для редактирования.
    - В **изображения при запуске** разделе, убедитесь, что **запуска экрана** присваивается имя, соответствующее раскадровки:

    ![Средство выбора запуска экрана в файле Info.plist.](launch-screens-images/launch02.png)

    - По умолчанию новый проект настроен на использование **LaunchScreen.storyboard** как экран его запуска.

3. Добавить изображение в **Assets.xcassets** ресурса каталога, чтобы он был доступен для использования на экране запуска. Дополнительные сведения см. в разделе [Добавление изображения в ресурс каталога образа группу](~/ios/app-fundamentals/images-icons/displaying-an-image.md) раздел [отображение изображения](~/ios/app-fundamentals/images-icons/displaying-an-image.md) руководства.

4. Откройте **LaunchScreen.storyboard** для редактирования, дважды щелкнув его в **панели решения**.

5. Выберите устройства и ориентация, на котором для предварительного просмотра раскадровки для экрана запуска в конструкторе iOS. Откройте панель выбора устройства, на нижней панели инструментов и выберите **iPhone 4S** и **книжной**.

    ![Панель выбора устройства](launch-screens-images/launch05.png)

    - Обратите внимание, что устройства и ориентация изменяет только как конструктор iOS выполняет предварительный просмотр конструктора. Независимо от того, выбор, сделанный здесь, добавляемых в ограничения применяются на всех устройствах и ориентации, если не **изменить признаки** кнопку использовался для указано иное. 

6. Задайте **фона** цвет основное представление контроллера представления. Выберите представление, щелкнув в середине контроллер представления и изменить цвет фона с помощью **панели свойств**:

    ![Единое представление с цветом фона сиреневый](launch-screens-images/launch06.png)

7. Добавить **представление изображений** на экране запуска и задать его источник **изображение**:

    - Перетащите **представление изображений** из **панель элементов** в центре представления.
    - С **представление изображений** выбран, в **мини-приложение** раздел **панели свойств** задать **изображение** задаваемое свойство изображения уже Добавляемый **Assets.xcassets** каталог активов. Изменить положение и размер **представление изображений** при необходимости:
    
    ![Это представление изображения с набором свойств образа](launch-screens-images/launch07.png)

8. Добавить **метка** ниже **представление изображений** и использовать **панели свойств** установить его атрибуты: 

    ![Метка с его текст и цвет набор](launch-screens-images/launch08.png)

9. Переключитесь в режим изменения ограничений с помощью правой кнопки в **панель инструментов ограничений**:
    
    ![Режим изменения ограничений кнопки](launch-screens-images/launch09.png)

10. Добавление ограничений к **представление изображений**, установка его высоту и ширину и центрирование его по горизонтали и вертикали:

    ![Это представление изображения с помощью ограничения макета](launch-screens-images/launch10.png)

    - Дополнительные сведения о том, как добавить ограничения, см. в разделе [автоматический макет в конструкторе Xamarin для iOS](~/ios/user-interface/designer/designer-auto-layout.md).

11. Добавление ограничений к **метка**, центрирование его по горизонтали, указав его высоту и ширину и позиционирования фиксированный расстояние по вертикали от **представление изображений**:

    ![Метка с ограничения макета](launch-screens-images/launch11.png)

12. Тестирование с другими устройствами и ориентаций, чтобы убедиться, что структуры выглядит так, как предполагалось во всех сценариях. В случаях, где нужно будет создать для конкретного устройства или ориентации корректировки, использовать **изменить признаки** кнопку, чтобы добавить системные ограничения для определенного размера классов:

    ![На экране запуска, отображаемой в качестве iPhone X с использованием альбомной ориентации](launch-screens-images/launch12.png)

13. Сохраните изменения в раскадровку. Запустите приложение на симуляторе или на устройстве, и будут видны на экране запуска при запуске приложения.

## <a name="migrating-to-launch-screen-storyboards"></a>Миграция для запуска раскадровки экрана

# <a name="visual-studio-for-mactabmacos"></a>[Visual Studio для Mac](#tab/macos)

При обновлении существующего приложения до его экраны запуска при помощи объектов Storyboard, щелкните правой кнопкой мыши **имя_проекта** в **обозревателе решений** и выберите **добавить**  >  **Новый файл...** . Выберите **iOS** > **запуска экрана** и нажмите кнопку **New** кнопки:

![](launch-screens-images/storyboard02.png "Выберите экран запуска iOS")

Далее дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования. В разделе **запуска экрана**, выберите новый файл раскадровки, созданной ранее.

![](launch-screens-images/storyboard09.png "Выберите новый файл раскадровки, созданной выше")


Чтобы использовать новую раскадровку, экран запуска, сделайте следующее:

1. Дважды щелкните `Info.plist` файл **обозревателе решений** чтобы открыть его для редактирования.
2. Прокрутите страницу до **универсальной изображения при запуске** в редакторе, откройте **запуска экрана** раскрывающийся список и выберите имя раскадровки, созданной выше: 

    ![](launch-screens-images/storyboard08.png "Настройка экрана запуска в раскадровку")

# <a name="visual-studiotabwindows"></a>[Visual Studio](#tab/windows)

1. Щелкните правой кнопкой мыши имя проекта в **обозревателе решений** и выберите **добавить** > **новый файл...** : 

    ![](launch-screens-images/image012.png "Добавьте новый файл")
2. Введите имя для экрана запуска и нажмите кнопку **добавить** кнопки: 

    ![](launch-screens-images/image013.png "Введите имя для экрана запуска")
3. В **обозревателе решений**, дважды щелкните файл только что созданный раскадровки, чтобы открыть его для редактирования.
4. Убедитесь, что **размер класса** присваивается **любой: любой** и **просмотреть как** — **универсального**: 

    ![](launch-screens-images/image016.png "Убедитесь, что размер класса к любому: любой и просмотреть как — универсальная")
5. Сборки на экране запуска из классов размера, простые элементы пользовательского интерфейса (такие как `UIImageView`) и образы, которые включены в пакет приложения: 

    ![](launch-screens-images/image017.png "Сборки на экране запуска в конструкторе iOS")
6. Сохраните изменения в раскадровку.
