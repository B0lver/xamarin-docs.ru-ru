С помощью нового iOS 10 пакет SDK, Apple включила новые интерфейсы API и службам, которые позволяют разработчикам создавать новые категории приложений и компонентов. Приложение iOS теперь можно расширить сообщения, Siri, Phone и карт приложения для предоставления привлечение набор функциональных возможностей для конечного пользователя, которая ранее была недоступна.

Дополнительные сведения о iOS 10 см. в разделе Apple [iOS + приложений](https://developer.apple.com/ios/) документации.

## <a name="adapting-to-the-true-tone-display"></a>Адаптация дисплей True тон

Технология отображения True тон Apple использует окружения датчика в устройства iOS для динамического изменения цвета и интенсивности экрана, чтобы соответствовать текущие условия освещения. предоставляет новые iOS 10 [UIWhitePointAdaptivityStyle](https://developer.apple.com/library/prerelease/content/documentation/General/Reference/InfoPlistKeyReference/Articles/iPhoneOSKeys.html#//apple_ref/doc/uid/TP40009252-SW31) ключ, который может быть добавлен в приложение `Info.plist` файл и управляет как True тон применяет стандартный цвет shift. 

Доступны следующие значения:

- `UIWhitePointAdaptivityStyleStandard` **По умолчанию** -использовать стандартный adaptivity белой точек.
- `UIWhitePointAdaptivityStyleReading` -Используется для приложений, направленные на чтение.
- `UIWhitePointAdaptivityStyleGame` -Используется для приложений, ориентированный на игры.
- `UIWhitePointAdaptivityStyleVideo` -Используется для приложений, ориентированный на видео.
- `UIWhitePointAdaptivityStylePhoto` -Используется для приложений, ориентированный на фотографии где цветовоспроизведения важнее корректировки среды белый точки.

## <a name="sirikit"></a>SiriKit

Новые для iOS 10 SiriKit позволяет приложения Xamarin.iOS для предоставления услуг, доступных для пользователя с помощью Siri на устройстве iOS. Эта функциональность обеспечивается в один или несколько расширения приложения с использованием нового **целей** и **пользовательского интерфейса целей** платформ.

SiriKit поддерживает следующие домены службы:

- Аудио или видео вызова.
- Резервирования расстояния.
- Управление тренировки.
- Обмен сообщениями.
- Поиск фотографий.
- Отправки или приема платежей.

Когда пользователь выполняет запрос Siri, связанных с одним расширения приложения служб, SiriKit отправляет расширение **намерение** , описывающий запроса пользователя, а также все дополнительные данные. Расширение приложения создает соответствующий **ответ** объекта для заданного **намерение**, с подробными сведениями о том, как расширение сможет обработать запрос.

Хотя обычно Siri обрабатывает все взаимодействие с пользователем, можно использовать расширение приложения **пользовательского интерфейса намерение** framework для представления сложных, пользовательские пользовательский интерфейс на базе приложения торговой марки и Дополнительные сведения.

Чтобы узнать больше, см. в разделе нашей [введение в SiriKit](~/ios/platform/sirikit/index.md) руководства.

Чтобы узнать больше, см. в разделе нашей [введение в распознавания речи](~/ios/platform/speech.md) руководства.

## <a name="wide-color"></a>Широкую цветовую

iOS 10 расширяет поддержку форматов пикселей расширенный диапазон и широким пробелов во всей системе, включая платформы, например Core графики, образ Core, системы и AVFoundation. Поддержка устройства с дисплеями широкую цветовую дальнейшей упрощают, предоставляя это поведение во всем стеке всей графики.

Кроме того [UIKit](https://developer.xamarin.com/api/namespace/UIKit/) был изменен для работы в новом расширенных **sRGB** цветовое пространство, что упрощает смешивания цветов в широкую цветовую охваты без потери значительного увеличения производительности.

Apple предоставляет следующие рекомендации при работе с шириной цветов:

- [UIColor](https://developer.xamarin.com/api/type/UIKit.UIColor/) теперь использует sRGB цветовое пространство и больше не будут выполните приведение значения к `0.0` для `1.0` диапазона. Если приложение зависит от предыдущего поведения приведения, будет необходимо изменить для iOS 10.
- При выполнении пользовательского рисования среды будет настроен цветов sRGB `UIView` рисование на iPad Pro.
- Если приложение выполняет отрисовку пользовательского `UIImages`, использовать новую [UIGraphicsImageRender](https://developer.apple.com/reference/uikit/uigraphicsimagerenderer) класс, чтобы указать использование форматы расширенный диапазон или диапазон standard.
- При использовании низкого уровня API, например двухмерной графики или исходного состояния системы для обеспечения обработки изображений, разработчик должен использовать расширенный диапазон цвет пространства и пикселей формат, поддерживающий 16-разрядных значений с плавающей запятой. При необходимости, разработчику придется вручную выполните приведение значений компонентов цвета.
- Основные графики, образ Core и шейдеров производительности системы предоставляют новые методы для преобразования между двумя цветовых схем.

Чтобы узнать больше, см. в разделе нашей [введение в широкую цветовую](~/ios/platform/wide-color.md) руководства.

## <a name="widget-enhancements"></a>Усовершенствования мини-приложения

Apple был представлен ряд дополнительных возможностей для системы мини-приложения, чтобы гарантировать, что удобна в мини-приложений на фоновые, имеющиеся на новый iOS 10 экран блокировки. [NotificationCenterVibrancyEffect](https://developer.apple.com/reference/uikit/uivibrancyeffect/1613917-notificationcentervibrancyeffect) свойство является устаревшим и заменена новой [WidgetPrimaryVibrancyEffect](https://developer.apple.com/reference/uikit/uivibrancyeffect/1771278-widgetprimaryvibrancyeffect) или [WidgetSecondaryVibrancyEffect](https://developer.apple.com/reference/uikit/uivibrancyeffect/1771277-widgetsecondaryvibrancyeffect) свойства. Кроме того, теперь содержат мини-приложения [NCWidgetDisplayMode](https://developer.apple.com/reference/notificationcenter/ncwidgetdisplaymode) свойство, которое позволяет разработчику для описания объем содержимого доступен и пользователь может разворачивать и сворачивать содержимое.

Чтобы узнать больше, см. в разделе нашей [поиска и Главная улучшения экрана мини-приложение](~/ios/platform/search/widgets.md) руководства.

## <a name="deprecated-apis"></a>Устаревшие интерфейсы API

Следующие API-интерфейсы являются устаревшими в iOS 10:

- `CKDiscoverAllContactsOperation`, `CKDiscoveredUserInfo`, `CKDiscoverUserInfosOperation` И `CKFetchRecordChangesOperation` классы являются устаревшими в CloudKit для iOS 10. Используйте [CKDiscoverAllUserIdentitiesOperation](https://developer.xamarin.com/api/type/CloudKit.CKDiscoverUserIdentitiesOperation/), [CKUserIdentity](https://developer.xamarin.com/api/type/CloudKit.CKUserIdentity/) и [CKFetchRecordZoneChangesOperation](https://developer.xamarin.com/api/type/CloudKit.CKFetchRecordZoneChangesOperation/) классы (которые поддерживают запись управления доступом) вместо него.
- Несколько [CKSubscription](https://developer.apple.com/reference/cloudkit/cksubscription) API-интерфейсы (например, на основе зон и основано на запросе подписок) являются устаревшими. Используйте [CKRecordZoneSubscription](https://developer.xamarin.com/api/type/CloudKit.CKRecordZoneSubscription/) и [CKQuerySubscription](https://developer.xamarin.com/api/type/CloudKit.CKQuerySubscription/) API-интерфейсы вместо него.
- [NSPersistentStoreCoordnator](https://developer.xamarin.com/api/type/CoreData.NSPersistentStoreCoordinator/) символы, связанные с содержимым широкого являются устаревшими.
- `ADBannerView`, `ADInterstitialAd` и связанные с ним символы в [UIViewController](https://developer.xamarin.com/api/type/UIKit.UIViewController/) класса являются устаревшими.
- [SKUniform](https://developer.apple.com/reference/spritekit/skuniform) символов, связанных значений с плавающей запятой являются устаревшими.
- `UILocalNotification`, `UIMutableUserNotificationAction`, `UIMutableUserNotificationCategory`, `UIUserNotificationAction`, `UIUserNotificationCategory` И `UIUserNotificationSettings` UIKit классы являются устаревшими. Используйте [уведомления пользователя](#User-Notifications) framework вместо него.
- `HandleActionForLocalNotification`, `HandleActionForRemoteNotification`, `DidReceiveLocalNotification` И `DidReceiveRemoteNotification` WatchKit методы являются устаревшими. Используйте `HandleActionForNotification` и `DidReceiveNotification` методов вместо него.
- `DidReceiveLocalNotification` И `DidReceiveRemoteNotification` методы [WKExtensionDelegate](https://developer.apple.com/reference/watchkit/wkextensiondelegate) являются устаревшими. Создайте экземпляр класса [UNUserNotificationCenterDelegate](https://developer.apple.com/reference/usernotifications/unusernotificationcenterdelegate) , реализует методы, соответствующие и назначьте его `Delegate` свойство [UNUserNotificationCenter](https://developer.apple.com/reference/usernotifications/unusernotificationcenter) объекта.
- **Приложения Game Center** устаревшими и удалены из операций ввода-вывода. Если приложение использует GameKit, он _должен_ предоставить свой собственный интерфейс для отображения GameKit функции, такие как списки лидеров и т. д.

В разделе Apple [iOS 9.3 для отличия API iOS 10.0](https://developer.apple.com/library/prerelease/content/releasenotes/General/iOS10APIDiffs/index.html) документации для получения полного списка возражения.
