# <a name="ios-7-user-interface-overview"></a>iOS 7 Обзор пользовательского интерфейса

_iOS 7 предоставляет множество изменения пользовательского интерфейса. В этой статье описаны некоторые изменения большего размера, внешний вид элементов управления и в API, которые поддерживают новый дизайн._

iOS 7 посвящена содержимого через chrome. Элементы пользовательского интерфейса в iOS 7 Коррекция постискажений chrome, удалив атрибуты, такие как лишние границы строки состояния и панели навигации, которые уменьшают объем пространства экрана, используемого содержимого представления. В iOS 7 содержимое предназначено для использования на весь экран.

iOS 7 представлено несколько других изменений: цвет используется для различения элементов пользовательского интерфейса, вместо атрибутов, таких как границы кнопки. Многие элементы, такие как панели навигации и строки состояния, стали размытая и полупрозрачными или прозрачными, с помощью представления содержимого область под ними ведения. Эти представления содержимого визуализации через размытая столбцы, передачи чувство глубины в пользовательском интерфейсе.

В этой статье рассматриваются некоторые изменения в элементы пользовательского интерфейса в iOS 7, а также различные интерфейсы API, связанные в новый проект интерфейса пользователя.

## <a name="view-and-control-changes"></a>Просмотр и управление изменениями

Все представления в UIKit соответствует нового вида iOS 7. В этом разделе описаны некоторые изменения в эти представления, а также связанные интерфейсы API, которые были изменены для поддержки нового пользовательского интерфейса.

### <a name="uibutton"></a>UIButton

Кнопки, созданные из `UIButton` класс стали границ, без фона по умолчанию, как показано ниже:

 ![](ios7-ui-images/button.png "Пример UIButton")

`UIButtonType.RoundedRect` Стиля является устаревшим. При использовании в iOS 7, `UIButtonType.RoundedRect` приведет к `UIButtonType.System` используется, что порождает стиль кнопки по умолчанию без фона или модели, как показано выше.

### <a name="uibarbuttonitem"></a>UIBarButtonItem

Аналогичную `UIButton`, кнопки строки также являются без рамки, по умолчанию используется новый `UIBarButtonItemStyle.Plain` стиль, показано ниже:

 ![](ios7-ui-images/barbuttonplain.png "Пример UIBarButtonItem")

Кроме того `UIBarButtonItemStyle.Bordered` стиля является устаревшим. Установка `UIBarButtonItemStyle.Bordered` в iOS 7 приведет к тому `UIBarButtonItemStyle.Plain` стиля используется.

`UIBarButtonItemStyle.Done` Стиля не является устаревшим. Тем не менее он также создаст кнопке без рамки только со стилем полужирным шрифтом, как показано:

 ![](ios7-ui-images/barbuttondone.png "Пример UIBarButtonItem в стиле Готово")

### <a name="uialertview"></a>UIAlertView

Помимо изменение стиля для новый внешний вид iOS 7 представления уведомлений больше не поддерживает настройку с помощью вложенное представление. Несмотря на то что `UIAlertView` наследует от `UIView`, вызов `AddSubview` на `UIAlertView` не оказывает влияния. Рассмотрим следующий пример кода:

```csharp
UIBarButtonItem button = new UIBarButtonItem ("Bar Button", UIBarButtonItemStyle.Plain, (s,e) =>
{
    UIAlertView alert = new UIAlertView ("Title", "Message", null, "Cancel", "OK");

    alert.AddSubview (new UIView () {
        Frame = new CGRect(50, 50,100, 100),
        BackgroundColor = UIColor.Green
    });

    alert.Show ();
});
```

Это создает стандартные представления предупреждений с вложенное представление, которые обрабатываются, как показано ниже:

 ![](ios7-ui-images/alert.png "Пример UIAlertView")
 
 Примечание: UIAlertView был объявлен устаревшим в iOS 8. Представление [контроллер оповещения](https://github.com/xamarin/recipes/tree/master/Recipes/ios/standard_controls/alertcontroller) инструкций об использовании представления "предупреждения" в iOS 8 и более поздних версий.

Сегментированные элементы управления в iOS 7 являются прозрачными, а также поддерживают оттенок цвета. Оттенок цвета, используемого в качестве цвета текста и границы. Если выбран сегмент, цвет меняется местами между фон и текст, с оттенок цвета, используемым для выделения выбранного сегмента, как показано ниже:

Кроме того `UISegmentedControlStyle` был объявлен устаревшим в iOS 7.

Внутреннее представление иерархии также немного отличаются. Это изменение не влияет на большинство приложений, но пока это следует учитывать.

### <a name="full-screen-content"></a>Содержимое на весь экран

iOS 7 разработано для приложений воспользоваться преимуществами весь экран. Контроллеры представлений появиться перекрывающегося в строке состояния и панель навигации: при наличии — в отличие от указывались ниже состояния и панели навигации.

При подготовке приложения для iOS 7, можно выровнять визуально с помощью вложенных представлений *Interface Builder* или *Xamarin iOS Designer*. Также можно использовать один из новых интерфейсов API для обработки содержимого на весь экран программным способом. Эти API-интерфейсы, представленные ниже.

#### <a name="toplayoutguide-and-bottomlayoutguide"></a>TopLayoutGuide и BottomLayoutGuide

 `TopLayoutGuide` и `BottomLayoutGuide` использовать в качестве ссылки для которых представления следует начать или завершить, таким образом, чтобы содержимое не перекрывается с прозрачным `UIKit` строке, как показано в следующем примере:

 [![](ios7-ui-images/clipped.png "Пример содержимого, не накладываются полосой UIKit Полупрозрачный")](ios7-ui-images/clipped.png#lightbox)

Эти интерфейсы API можно использовать для вычисления смещения представления из верхней или нижней части экрана и соответствующим образом настроить размещения содержимого:

```csharp
public override void ViewDidLayoutSubviews ()
{
    base.ViewDidLayoutSubviews ();

    if (UIDevice.CurrentDevice.CheckSystemVersion (7, 0)) { 
        nfloat displacement_y = this.TopLayoutGuide.Length;

        //load subviews with displacement
    }

}
```

Можно использовать значение, вычисленном выше, чтобы задать наших `ImageView`его смещение в верхней части экрана, поэтому отображается всего изображения:

 [![](ios7-ui-images/good2.png "Пример ImageViews смещение в верхней части экрана")](ios7-ui-images/good2.png#lightbox)

Ссылаться на [ImageViewer](https://developer.xamarin.com/samples/mobile/iOS7-ui-updates) рабочий образец.

Значение смещения создается динамически, после добавления представления к иерархии, поэтому при попытке чтения `TopLayoutGuide` и `BottomLayoutGuide` значения в `ViewDidLoad` вернет 0. Вычислить значение представления загрузки — например, в `ViewDidLayoutSubviews`.

> [!IMPORTANT]
> `TopLayoutGuide` и `BottomLayoutGuide` являются устаревшими в iOS 11 пользу нового макета безопасной области. Apple указано, что с помощью безопасной области совместим с iOS версии более ранней, чем iOS 11. Дополнительные сведения см. в разделе [обновление приложения для iOS 11](~/ios/platform/introduction-to-ios11/updating-your-app/visual-design.md#fullscreen) руководства.

#### <a name="edgesforextendedlayout"></a>EdgesForExtendedLayout

Этот API определяет, какие границы представления оно должно быть воспроизведено в полноэкранный режим, вне зависимости от полосы полупрозрачность. В iOS 7 панелей навигации и панели инструментов отображаются располагаются над контроллера представления — в отличие от в iOS предыдущих версий, где они не занимают то же пространство. Приложение iOS 7 фотографии показано значение по умолчанию `UIViewController.EdgesForExtendedLayout` значение `UIRectEdge.All`. Этот параметр заполняет всех четырех краев в представлении с содержимым, создание силу перекрывающиеся и весь экран:

 [![](ios7-ui-images/photos.png "Пример EdgesForExtendedLayout")](ios7-ui-images/photos.png#lightbox)

При касании изображение удаляет панели показано и в полноэкранном режиме изображение.

 [![](ios7-ui-images/photos2.png "EdgesForExtendedLayout по отношению к столбцам, которые удалены")](ios7-ui-images/photos2.png#lightbox)

Поскольку содержимое весь экран по умолчанию, приложения, настроенные для iOS 6 будет иметь части представления обрезается, как показано на снимке экрана ниже:

 [![](ios7-ui-images/clipped.png "Приложений, настроенных для iOS 6 будет иметь части представления обрезается, как показано на этом снимке экрана")](ios7-ui-images/clipped.png#lightbox)

Изменение `UIViewController.EdgesForExtendedLayout` настраивает свойство для данного поведения. Мы можно указать, что представление не заполняет краям, поэтому нашего представления позволит избежать отображения содержимого в пространство, занимаемое навигации и панели инструментов (в каждую ориентацию):

```csharp
if (UIDevice.CurrentDevice.CheckSystemVersion (7, 0)) { 
    this.EdgesForExtendedLayout = UIRectEdge.None;
}
```

В наше приложение мы увидим представление снова перемещен, поэтому всего изображения отображается:

 [![](ios7-ui-images/good.png "Пример с видимым всего изображения")](ios7-ui-images/good.png#lightbox)

Следует отметить, что последствия `TopLayoutGuide/BottomLayoutGuide` и `EdgesForExtendedLayout` интерфейсы API очень похожи, они предназначены для заполнения других целей. Изменение `EdgesForExtendedLayout` параметр по умолчанию, чтобы устранить усеченные представления в приложениях, предназначенных для iOS 6, но разработки хороший iOS 7 должны соблюдать делает в эстетических весь экран, а также назначить полноэкранного просмотра качества, полагаясь на `TopLayoutGuide` и `BottomLayoutGuide`для правильно размещения содержимого, который указывает для обработки в месте, удобном для пользователя.

Ссылаться на [ImageViewer](https://developer.xamarin.com/samples/mobile/iOS7-ui-updates) рабочий образец.

### <a name="status-and-navigation-bars"></a>Состояние и панели навигации

Строка состояния и панели навигации отображаются с прозрачностью. Строки состояния являются прозрачными, хотя панелей инструментов и панели навигации являются полупрозрачными и размытым для передачи чувство глубины в пользовательском интерфейсе. На следующем рисунке показан этот размытие и прозрачности, где показано синего цвета фона представления коллекции через полосы навигации и состояние, предоставляя им более светлой синей вид:

 ![](ios7-ui-images/transparent-navbar.png "Пример состояния и панели Размытие навигации")

#### <a name="status-bar-styles"></a>Стили строки состояния

Вместе с размытие и прозрачность переднего плана в строке состояния может быть light или темной (темно-используется по умолчанию). Стиль строки состояния можно задать из контроллера в представление. Контроллер представления можно также задать ли строка состояния показывается или скрывается.

Например, в следующем коде переопределения `PreferredStatusBarStyle` метод контроллера представления, чтобы панель состояния отображения светло передний план:

```csharp
public override UIStatusBarStyle PreferredStatusBarStyle ()
{
    return UIStatusBarStyle.LightContent;
}
```

Это приводит к строке состояния отображается следующим образом:

 ![](ios7-ui-images/light-status-bar.png "Пример строки состояния")

Чтобы скрыть строку состояния из кода контроллера представления, переопределите `PrefersStatusBarHidden`, как показано ниже:

```csharp
public override bool PrefersStatusBarHidden ()
{
    return true;
}
```

Это скрывает строку состояния:

 ![](ios7-ui-images/status-bar-hidden.png "Скрытые строки состояния")

### <a name="tint-color"></a>Оттенок цвета

Кнопки теперь отображаются как текст менее chrome. Цвет текста можно управлять с помощью нового `TintColor` свойство `UIView`. Параметр `TintColor` применение цвета всего представления иерархии для представления, устанавливает его. Чтобы применить `TintColor`на протяжении всего приложения, задайте его в `Window`. Можно также обнаружить при изменении оттенок цвета с помощью `UIView.TintColorDidChange` метод.

Например, на следующем рисунке показан эффект от изменения оттенок цвета для представления контроллера навигации для фиолетовым:

 ![](ios7-ui-images/tint-color.png "Цвет фиолетовым цветом для представления контроллеров навигации")

Оттенок цвета могут применяться к изображениям, а также когда `RenderingMode` присваивается `UIImageRenderingMode.AlwaysTemplate`.

> [!IMPORTANT]
> Оттенок цвета нельзя задать с помощью `UIAppearance`.


### <a name="dynamic-type"></a>Динамический тип

В iOS 7 пользователь может указать размер текста в системных параметрах. С помощью динамического типа шрифта динамически настраивается в правильное отображение независимо от размера. `UIFont.PreferredFontForTextStyle` можно использовать для получения шрифта, которая оптимизирована для размера управляемой пользователем.
