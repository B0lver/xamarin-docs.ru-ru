# <a name="introduction-to-ios-9"></a>Введение в iOS 9

_В этой статье рассматриваются все новые и измененные API-интерфейсы и функциях, доступных в iOS 9 для разработчиков Xamarin.iOS._

![](images/ios9-sml.png "Логотип iOS 9")

Apple добавлено несколько новых API-интерфейсов и служб в iOS 9, а также множество улучшений существующих функций.

## <a name="3d-touch"></a>Трехмерные сенсорные технологии

Знакомы с iOS 9 и iPhone версии 6s и iPhone 6s Plus, 3D Touch добавляет конфиденциальных жесты давление приложения iOS. С 3D Touch, приложения iPhone теперь возможность не только определять, что пользователь касается экрана устройства, его можно находить давление, усиливая пользователя и реагировать на них уровни различных давления.

3D Touch предоставляет следующие возможности для приложения:

- **Давление чувствительности** - приложений теперь можно измерить потребляемую или свет пользователь касается экрана и пользоваться преимуществами этой информации. Например рисования приложение может сделать толще или тонкие основываться на потребляемую пользователь касается экрана строки.
- **Просмотр и открытие в окне** -приложения теперь можно разрешить пользователю взаимодействовать с данными без необходимости перехода из текущего контекста. Нажав жестких на экране, они могут *Показать* элемента, которые их интересуют (например, предварительный просмотр сообщения). Нажав сложнее, они могут *Pop* в элемент.
- **Быстрые действия** -представить из Быстрые действия как контекстные меню, которые могут быть извлекается доступ при щелчке элемента в настольном приложении. С помощью быстрых действий, можно добавить общие, быстрый и простой для быстрого доступа к функциям в приложении из значок экрана главной страницы на устройстве iOS.

Чтобы узнать больше, ознакомьтесь с разделом наших [введение в 3D Touch](~/ios/platform/3d-touch.md) руководства.

## <a name="app-transport-security"></a>Защита транспорта приложения

Новое для iOS 9, но приложение Transport Security (ATS) применяет безопасных соединений между Интернет-ресурсов (например, server серверной части приложения) и приложения. ATS гарантирует все Интернет-коммуникациями соответствия безопасное подключение рекомендации, тем самым предотвращая случайное раскрытие конфиденциальной информации, либо напрямую с помощью приложения или библиотеку, которая потребляет.

Поскольку ATS включена по умолчанию в приложениях, созданных для iOS 9 и OS X 10.11 (El Capitan), все подключения с помощью [NSUrlConnection](xref:Foundation.NSUrlConnection), [CFUrl](xref:CoreFoundation.CFUrl) или [NSUrlSession](xref:Foundation.NSUrlSession) будет предметом Требования к безопасности ATS. Если эти требования не соответствуют подключений, они вызовут сбой с исключением.

Чтобы подробнее узнать о ATS, см. наш [безопасность транспорта приложения](~/ios/app-fundamentals/ats.md) руководства.

## <a name="multitasking-for-ipad"></a>Многозадачность для iPad

С iOS 9 Apple имеет дополнительную поддержку многозадачности, для выполнения приложений двумя в то же время на оборудовании конкретного iPad. В результате приложений Xamarin.iOS больше не могут считать, что они только приложение, работающее в любой момент времени, или что они имеют доступ к ресурсам устройства или во весь экран.

Многозадачность для iPad поддерживает следующие функции:

- **Слайд через** -пользователь может временно запуск второго приложения iOS в выдвижными панели (либо на правой или левой части экрана, в зависимости от направления языка), который охватывает примерно 25% основного приложения, выполняемого в данный момент. Проведите пальцем по доступен только на iPad Pro, iPad Air, iPad Air 2, iPad Mini 2, iPad Mini 3 или iPad Mini 4.
- **Разделенное представление** -на оборудовании, поддерживаемых iPad (iPad 2 Air, iPad Mini 4 и iPad только Pro), пользователь может выбрать второе приложение и запустить его side-by-side с выполняющегося приложения в режиме экрана разбиения. Пользователь может управлять процент основной экран, который занимает каждого приложения.
- **Картинка в картинке** — для приложений, которые воспроизведения видео-контента, видео, которые теперь могут воспроизводиться в перемещаемой и окна регулируемого размера, расположенном поверх других приложений, выполняющихся на устройстве iOS. Пользователь имеет полный контроль над размер и положение этого окна. Картинка в картинке доступна только на iPad Pro, iPad Air, iPad Air 2, iPad Mini 2, iPad Mini 3 или iPad Mini 4.

Чтобы подробнее узнать о новой возможности многозадачность iOS 9, см. наш [многозадачность для iPad](~/ios/platform/multitasking.md) руководства.

## <a name="new-contacts-and-contacts-ui-frameworks"></a>Новые контакты и контакты инфраструктур пользовательского интерфейса

С появлением iOS 9 Apple выпустила два новых платформ [контакты](xref:Contacts) и [ContactsUI](xref:ContactsUI), замените существующий адресной книги и используемые платформы пользовательского интерфейса адресной книги iOS 8 и более ранние версии.

Эти новые, объектно ориентированная платформы содержатся следующие сведения.

- **Контакты** — Xamarin.iOS предоставляет доступ к контактные данные пользователя. Так как большинство приложений требуется только доступ только для чтения, эта инфраструктура была оптимизирована для доступа потоками безопасно, только для чтения.
- **ContactsUI** — предоставляет Xamarin.iOS элементы пользовательского интерфейса для отображения, редактирования выберите и создайте контактам на устройствах iOS.

Дополнительные сведения см. наш [контакты и пользовательского интерфейса контакты](~/ios/platform/contacts.md) документации.


## <a name="new-search-apis"></a>Новые API-интерфейсы поиска

Поиск была расширена в iOS 9, чтобы предоставить новые способы доступа к данным внутри приложения Xamarin.iOS. С помощью новых интерфейсов API поиска, можно сделать содержимое своего приложения для поиска в центре внимания и Safari результатах поиска, передачи и напоминания Siri и предложения. Это позволяет пользователям быстрый доступ к действий и сведений, глубоко внутри приложения.

Кроме того новые API-интерфейсы поиска облегчают интегрировать систему поиска в приложении без опыта реализации предыдущего поиска. По этой причине Apple заявляет, что обычно занимает несколько часов, чтобы обеспечить универсально для поиска с помощью приложения поиска содержимого с приложением iOS 9.

Дополнительные сведения см. в разделе наших [улучшенные возможности поиска](~/ios/platform/search/index.md) документации.

## <a name="new-stack-view"></a>Новое представление стека

Элемент управления представление стека ([UIStackView](xref:UIKit.UIStackView) использует возможности автоматический макет и размер классов для управления стек представлений (горизонтальное или вертикальное), которое динамически реагировать на устройстве iOS ориентации и размера экрана.

С помощью элемента управления представление стека, объем работы требуется макета пользовательского интерфейса, значительно снижается. Макет всех вложенных представлений, прикреплено к представлению стека осуществляется автоматически на основании разработчика определенные свойства, такие как оси, распространение, выравнивание и интервал.

Дополнительные сведения см. в разделе наших [введение в представление стека](~/ios/user-interface/controls/uistackview.md) документации.


## <a name="collection-view-changes"></a>Изменение представления коллекции

В iOS 9 Apple упростилась установки, расширен типы стандартные поддерживается и предоставляются дополнительные аксессуаров взаимодействия (например, управление периферийного устройства удаленно с помощью iCloud).

## <a name="new-extension-points"></a>Новые точки расширения

В iOS 8, Apple представила расширения — библиотеки, которые представлены в операционной системе, в стандартный контекстах, такие как в центре уведомлений, когда пользователь запрашивает клавиатуры или, редактирующего фотографию.

С iOS 9, Apple, расширение, предоставляя несколько новых поддержка расширения _точки расширения_ , определение политик использования и предоставляет API для работы в пределах заданной области, как показано ниже:

- **Новую точку расширения модульного аудио** — использовать эту точку расширения для предоставления звуковые эффекты, звуки музыкальных инструментов, звуковой генераторы т. д. для использования в других приложениях узла Audio Unit (например, GarageBand). Эта точка расширения позволяет вам продавать _аудио единиц_ (аудио подключаемые модули) на App Store.
- **Новую точку расширения обслуживания индекса** — использовать эту точку расширения для поддержки повторного индексирования данных приложения, не требуя перезапуск приложения.
- **Новые точки расширения сети** (для этого требуется специальное разрешение от Apple):
    - **Расширение поставщика прокси-сервера приложения** — использовать эту точку расширения для реализации пользовательских прозрачной сети клиентского прокси-сервер.
    - **Фильтрация данных поставщика / расширения поставщика фильтра элемента управления** -использовать эти точки расширения для реализации фильтрации на устройстве сети динамического содержимого.
    - **Расширение поставщика пакетов туннеля** — использовать эту точку расширения для реализации пользовательской сети VPN, туннелирование протокола на стороне клиента.
- **Новые точки расширения Safari**:
    - **Расширение блокирования содержимого** — использовать эту точку расширения для определения списка заблокированных содержимого, которое будет отображаться, когда пользователь просматривает веб-узла.
    - **Общие ссылки расширение** — использовать эту точку расширения для просмотра содержимого вашего приложения в общих ссылок Safari.

Дополнительные сведения см. в разделе наших [введение расширения](~/ios/platform/extensions.md) и Apple [руководство по программированию расширения приложения](https://developer.apple.com/library/prerelease/ios/documentation/General/Conceptual/ExtensibilityPG/index.html#//apple_ref/doc/uid/TP40014214) документации.

## <a name="keychain-enhancements"></a>Усовершенствования в цепочке ключей

В iOS 9 Apple была усовершенствована цепочку ключей, укажите новый тип ключа шифрования для защиты Анклава и дополнительные параметры защиты для элемента, как показано ниже:

- Новое ограничение Touch ID сводит цепочки ключей элементов, при изменении базы данных по отпечатку пальца.
- Новые ограничения, которые разрешить только создание записей списка управления доступом с помощью Touch ID и секретный код.
- Новый контекст проверки подлинности, который позволяет выполнять проверку подлинности, отдельно от `SecItem` вызовов.
- Доступ к энтропии список управления (с помощью параметра пароль приложений) для шифрования элемента приложением цепочки ключей.
- Поддержка создания и использования ключей внутри безопасным анклавом (через `kSecAttrTokenIDSecureEnclave` атрибут).

Дополнительные сведения см. в разделе наших [Общие сведения о Touch ID](~/ios/platform/touchid.md) документации.


## <a name="right-to-left-language-support"></a>Поддержка языка справа налево

В iOS 9 Apple сделала представления должно быть перевернуто пользовательского интерфейса проще, чем когда-либо за счет полной поддержки языков справа налево. Это поведение характеризуется следующим образом.

- Стандартный [UIKit](xref:UIKit) элементов управления будет автоматически менять справа налево на основе параметров языка и языковых устройства iOS.
- [UIView](xref:UIKit.UIView) класс предоставляет атрибуты, которые позволяют определить, каким образом данного представления должно появляться при Перевернутая справа налево.
- Возможность перевернуть изображения программным способом с помощью [FlipsForRightToLeftLayoutDirection](xref:UIKit.UIImage.FlipsForRightToLeftLayoutDirection) свойство [UIImage](xref:UIKit.UIImage) класса.

Дополнительные сведения см. в разделе Apple [языков вспомогательные справа-налево](https://developer.apple.com/library/prerelease/ios/documentation/MacOSX/Conceptual/BPInternational/SupportingRight-To-LeftLanguages/SupportingRight-To-LeftLanguages.html#//apple_ref/doc/uid/10000171i-CH17) документации.

## <a name="deprecated-apis-and-functions"></a>Устаревшие интерфейсы API и функции

Следующие API-интерфейсы и функции в iOS 9 Apple объявлена устаревшей:

- **Адрес книги и книги адреса в Интерфейсе** -эти интерфейсы API были заменены на платформах контакт и контакт пользовательского интерфейса. Дополнительные сведения см. наш [контакты и пользовательского интерфейса контакты](~/ios/platform/contacts.md) документации.
- **CBCentralManager** - `RetrievePeripherals` и `RetrieveConnectedPeripherals` методы `CBCentralManager` класса будут удалены в iOS 9. Вызов этих методов вызовет приложение дает сбой при связывании периферийного устройства или при запуске приложения.
- **FetchAllChanges** - `FetchAllChanges` из `CKFetchRecordChangesOperation` класс был амортизированы и будет удален в iOS 9.
- **Проигрыватель мультимедиа** -платформа проигрывателя мультимедиа был объявлен устаревшим в iOS 9. Вместо этого используйте AVKit или API-интерфейсы AV Foundation.

Полный список конкретных устаревании API, см. в разделе Apple [iOS 9.0 различия API](https://developer.apple.com/library/prerelease/ios/releasenotes/General/iOS90APIDiffs/index.html#//apple_ref/doc/uid/TP40016222) документации.
