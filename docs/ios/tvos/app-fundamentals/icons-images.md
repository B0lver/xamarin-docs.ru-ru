# <a name="working-with-tvos-icons-and-images-in-xamarin"></a>Работа с tvOS, значков и изображений в Xamarin

Создание привлекательной среды значков и изображений являются важной частью разработки иммерсивных пользовательский интерфейс для приложений Apple TV. В этом руководстве рассматриваются шаги, необходимые для создания и включения графические ресурсы, необходимые для приложения Xamarin.tvOS:

- [Изображение при запуске](#Launch-Image) -отображает изображение запуска, когда приложение запускается первый раз и заменяется первого экрана приложения после завершения запуска.
- [Многослойные образы](#Layered-Images) — для Apple TV, Apple многослойные образы новых приложений с эффект фокусировки, чтобы создать трехмерный эффект для выбранных элементов. Существует несколько способов [создавать многоуровневые образы](#Creating-Layered-Images).
- [Значок приложения](#App-Icons) -значки требуются для экрана не только Apple TV Home, но для App Store. Их должно быть указано как многослойные изображение.
- [TOP изображение панели](#Top-Shelf-Image) — Если приложение размещается в верхней строке экрана домашней, ему понадобится основное изображение чтобы ознакомиться с функциями приложения. При необходимости можно предоставить [динамическое содержимое полки верхней](#Dynamic-Top-Shelf-Content) выделите содержимое в приложении.
- [Игр образы Center](#Game-Center-Images) -Если ваше приложение — это игра и использует Game Center, требуется несколько дополнительных образов.
- [Установка изображений проекта Xamarin.tvOS](#Setting-Xamarin.tvOS-Project-Images) -рассматриваются шаги, необходимые для настройки изображений запуска и значок приложения для приложения Xamarin.tvOS.

> [!IMPORTANT]
> Все изображения на Apple TV, с разрешением 1 x (`@1x`) и вы должны _только_ использовать образы такого размера. Включая большего размера, более высоким разрешением графика не только на времени загрузить и использовать больше памяти и хранилища, но они должны быть динамически масштабируется повторно во время выполнения и отрицательно повлиять на производительность рисования.

<a name="Launch-Image" />

## <a name="launch-image"></a>Изображение при запуске

Изображение запуска — это первое, отображаемый, когда приложение Xamarin.tvOS изначально запустится на Apple TV, и таким образом, каждое приложение tvOS необходимо указать образ запуска. 

Запустите образ быстро и создает впечатление, что приложение является быстрый и отвечает на запросы. Apple TV заменит запуска образа с первого экрана приложения вскоре существует после.

Изображения при запуске не возможности для рекламы или художественного выражения, они существуют только для того, чтобы создать впечатление, что приложение запускает быстро и готов для использования.

|Размер образа для запуска|Примечания|
|---|---|
|1920x1080px|Только отличные от многоуровневых PNG-файлы|

Apple вносит следующие рекомендации по проектированию изображений запуска вашего приложения:

- **Практически идентично на первом экране** -свой экран запуска должно быть как можно ближе к первого экрана приложения максимально. Включая различные графики или элемент может привести к раздражающих «flash», при открытии первого экрана.
- **Избегать использования текста с помощью** -изображения при запуске, являются статическими и таким образом, не будет локализовано перед отображением.
- **Рассказ запуска** -Apple TV, поскольку пользователи часто переключаются приложений, не следует обратить внимание в процессе запуска приложения.
- **Без рекламы или продвижения** -Your изображений запуска не следует использовать как экран сообщения об About или включить любой продвижения, если он не является статической части первого экрана приложения. ADS строго запрещено.

<a name="Layered-Images" />

## <a name="layered-images"></a>Многоуровневые образы

Новые и Apple TV, многослойные образы работают с эффект фокусировки, чтобы вызывает трехмерных эффектов, помогающий сделать пользователя на диване, путешественник подключенными к содержимому на экране в комнате.

Многоуровневые образы содержат из двух (2) для пяти (5) отдельными слоями, которые объединяются для формирования полного образа. За исключением фон каждый слой использует его Z-порядка, а также прозрачности для создания иллюзии глубины. Когда пользователь взаимодействует с помощью слоев образа, более высоких уровнях, упорядоченную по Z масштабируется и overlapped, чтобы создать этот эффект.

[![](icons-images-images/layered01.png "Многоуровневая схема упорядоченную по Z образов")](icons-images-images/layered01.png#lightbox)

> [!IMPORTANT]
> Многоуровневые образы являются обязательными для значков приложения и не являются обязательными для других [, способному получать фокус элементов](~/ios/tvos/app-fundamentals/navigation-focus.md#Focus-and-Selection) (например, изображение для верхней панели). Тем не менее Apple предлагает, использовать многоуровневые образы на любое другое изображение, которую может получить фокус в приложении.




Apple вносит следующие рекомендации по проектированию многослойные образы:

- **Преобразование непрозрачный слой фона** -уровень фона (уровень 1) **необходимо** быть непрозрачным, или вы получите ошибку при попытке использовать образ располагаются на Apple TV. Все другие слои могут содержать несколько уровней прозрачности для улучшения трехмерные эффекты.
- **Изолировать переднего плана, среднего и фон элементов** -важных элементов (например, игры символов) на переднем плане, используйте середине для вторичного элементов или тени. Наконец включите нейтральный фон для обеспечения стадии на верхние уровни.
- **Сохранить текст на переднем плане** — Если вам не нужно текста, чтобы быть скрыт от более высоких уровней, обычно следует верхнего слоя.
- **Используйте простой наложение** -эффекта фокусировки были разработаны как слабая закрывайте слои значению "Минимальный" во избежание эффекты резать глаз, нереально.
- **Включить зоны Safe** -так, как верхние уровни может обрезаться во время эффекта фокусировки, необходимое для создания границы зоны безопасности в каждом слое. Если содержимое слишком закрыть edge слои, он может стать обрезанные. Верхние уровни могут возникнуть дополнительные масштабирования и обрезки, чем нижних слоев. См. в разделе [слои изменения размера изображения](#Sizing-Image-Layers) разделе ниже.
- **Предварительный просмотр часто** -многослойные образы должны просматриваться часто, чтобы убедиться, что происходит желаемого эффекта трехмерной и что ни один из содержимого в отдельных слоях обрезается. Его следует просмотреть многослойные образы на реальном оборудовании Apple TV чтобы убедиться в том, что они так, как ожидается.

По возможности следует всегда использовать встроенное `UIKit` элементы управления для отображения изображений многоуровневых, как они автоматически получат эффекта фокусировки при их роль в фокус.

<a name="Creating-Layered-Images" />

### <a name="creating-layered-images"></a>Создание многоуровневых образов

tvOS работает с многослойные образы в следующих форматах:

- **Файлы автомобиль** — это собственный формат каталог активов, созданная компанией Apple. АВТОМОБИЛЬ файлы не создаются напрямую, они создаются во время компиляции, из всех файлов LSR и состава набора приложений.
- **Образы LSR** -это формат собственный образ, созданный Apple. Используйте [фокусировки экспорта Adobe Photoshop, подключаемый модуль](https://itunespartner.apple.com/assets/downloads/ParallaxExporter_Apps.zip) или [средства предварительного просмотра фокусировки](http://itunespartner.apple.com/assets/downloads/Parallax%20Previewer.dmg) для создания и многоуровневой изображения предварительного просмотра в формате LSR.
- **Assets.xcassets** — из двух (2) для пяти (5) стандартный `.png` форматированный образов, включенных в каталог активов, где будут скомпилированы в автомобиль или LSR формате слоев образа во время компиляции.
- **Файлы LCR** — это собственный формат, созданная компанией Apple. LCR файлы предназначены для использования в качестве дополнительных содержимое, загруженное из одного из ваших серверов содержимого. Файл LCR не должны включаться в набора приложений. LCR файлы создаются из файлов LSR или Photoshop, с помощью `layerutil` средство командной строки, входящее в Xcode.

<a name="The-Parallax-Previewer" />

### <a name="the-parallax-previewer"></a>Средство предварительного просмотра фокусировки

Компания Apple создала [средства предварительного просмотра фокусировки](http://itunespartner.apple.com/assets/downloads/Parallax%20Previewer.dmg) для предварительной версии и созданный многоуровневой образов, необходимых для значков приложений и дополнительных элементов, способному получать фокус. Средство предварительного просмотра показывает каждый слой, относящиеся к формирующему завершенного слоев образа:

[![](icons-images-images/layered03.png "Средство предварительного просмотра фокусировки")](icons-images-images/layered03.png#lightbox)

Во время просмотра образ слоев, можно использовать мышь Поворот изображения и предварительный просмотр эффекта фокусировки. Используйте **+** (плюс) и **-** (кнопки для добавления и удаления слои минус).

При создании нового образа многоуровневой, его можно экспортируются в формате LSR и включенные в пакет приложения.

Дополнительные сведения о создании и предварительный просмотр многоуровневой образов см. в разделе Apple [Создание иллюстрации фокусировки](https://developer.apple.com/library/prerelease/tvos/documentation/General/Conceptual/AppleTV_PG/CreatingParallaxArtwork.html#//apple_ref/doc/uid/TP40015241-CH19-SW1) раздел [приложения руководство по программированию для tvOS](https://developer.apple.com/library/prerelease/tvos/documentation/General/Conceptual/AppleTV_PG/).

## <a name="app-icons"></a>Значки приложений

Приложение Xamarin.tvOS потребует не только значка приложения для Apple TV начального экрана, а также значок App Store. Значок приложения — это первого изменить впечатление отличный потенциальных пользователей и должно передавать цель вашего приложения с первого взгляда.

[![](icons-images-images/icon01.png "Значок приложения")](icons-images-images/icon01.png#lightbox)

Каждое приложение необходимо указать небольших и больших версию значка приложения. Мелкий значок будет использоваться на экране Apple TV Главная при установке приложения. Большой версии используется App Store. Крупный значок приложения должна имитировать внешний вид версии мелкого значка.

|Мелкий значок||Крупный значок||
|---|---|---|---|
|Фактический размер|400x240px|Размер|1280x768px|
|Размер зоны безопасности|370x222px|||
|Размер без фокуса ввода|300x180px|||
|Размер с фокусом ввода|370x222px|||

> [!IMPORTANT]
> Значки приложения должно быть указано как **многослойные образы**. См. в разделе [слоев образа](#Layered-Images) разделе выше для получения дополнительных сведений.




Компания Apple предоставляет следующие рекомендации по созданию вашей значки приложений:

- **Укажите одну точку фокуса** — разработки значка с точкой единый фокус помещаются непосредственно в центре значка.
- **Простой фон** — усложнять фон значка, чтобы выделить состояние объектов верхних уровней. Рассмотрите возможность использования простого цвет или слабая градиент.
- **Ограничить объем текста** — так как имя приложения будет отображаться под значком, когда он выбран пользователем, необходимо включать только текст при очень важно для проектирования значок.
- **Не используйте снимки экрана** — снимки экрана, слишком сложны для значка и запретить пользователю видеть цель приложения с первого взгляда.
- **Квадратные значки Keep** — tvOS автоматически применяет маску, которая может привести к некоторой округление углов значки. Не включайте округление самостоятельно.
- **Тщательно Your слои разделения** — большинство слой, вторичный элементов в середине и нейтральный фон, позволяющий на верхние уровни проявляются текст должен быть в верхнем углу.
- **Использовать градиенты и теней тщательно** — градиенты и теней может пересекаться с эффекта фокусировки, их следует использовать осторожно. Простой сверху вниз, лучше всего работают градиента стили светлого к темному. Shadows обычно наиболее эффективны как оттенки sharp, с.
- **Различаются прозрачности слоя** — используйте различные уровни прозрачности на верхние уровни значок приложения для увеличения трехмерные эффекты. Фон должен быть непрозрачным, или он будет привести к ошибке.

<a name="Top-Shelf-Image" />

## <a name="top-shelf-image"></a>Изображение для верхней панели

Если пользователь разместил Xamarin.tvOS приложения в верхней строке на экране Apple TV Главная, при выборе пользователем приложения будет отображаться большое изображение для верхней панели. Этот образ следует ознакомиться с функциями приложения или приведены прямые ссылки на его содержимое.

[![](icons-images-images/topshelf01.png "Верхнем примере изображение панели")](icons-images-images/topshelf01.png#lightbox)

Изображение для верхней панели, либо может быть представлен в виде один статический `.png` или `.lsr` файл (см. в разделе [создание образов многоуровневой](#Creating-Layered-Images)) или его можно создать динамически во время выполнения как одна строка, способному получать фокус элементов (см. в разделе [ Содержимое динамического верхняя Полка](#Dynamic-Top-Shelf-Content) ниже).

|Размер изображения верхняя Полка|Примечания|
|---|---|
|1920x720px|Статические PNG- или многоуровневой .lsr файла|

Компания Apple предоставляет следующие рекомендации по созданию изображений полки верхней:

- **Используйте статические изображения Rich** — Если приложение не поддерживает динамическое содержимое, его основное изображение будет не способному получать фокус. Используйте это изображение, чтобы ознакомиться с функциями приложения или вашего бренда.
- **Ссылки на содержимое приложения** – динамические макеты полки верхней обеспечивают быструю ссылку на содержимое, которое пользователь находит наиболее важные для приложения. Эта область используется для предоставления быструю ссылку для запуска приложения и сразу перейти к следующим содержимым.
- **Последние содержимым** — полки верхней форматированного содержимого можно нарисовать пользователя в приложение и сделать их требуется использовать дополнительные. Используйте в качестве области для демонстрации наивысшей оценкой. или более поздней содержимого.
- **Персонализированные содержимое** — их наиболее часто используемый месте пользователи или часто используемыми приложениями, в верхней строке начальном экране. Используйте Top Shelf для отображения содержимого, они будут наибольший интерес.
- **ADS не допускается** — рекламные объявления, строго запрещено, отображаемого в верхней полки. Может отображаться последние отдельной содержимое, но не сведения о ценах должны отображаться.

<a name="Dynamic-Top-Shelf-Content" />

### <a name="dynamic-top-shelf-content"></a>Содержимое динамического верхняя Полка

Вместо отображения статических изображение для верхней панели, Top Shelf может содержать динамические строки [, способному получать фокус элементов](~/ios/tvos/app-fundamentals/navigation-focus.md#Focus-and-Selection) или динамический набор прокрутки ленты. Оба эти динамические стиля позволяют выделить содержимое, предоставляемое с приложения или переход в его наиболее часто используемых функций.

<a name="Sectioned-Content-Row" />

#### <a name="sectioned-content-row"></a>Строка разделенных содержимого

Этот тип содержимого динамических Top Shelf представляется одной строке прокрутки, способному получать фокус элементов, при необходимости разбить на разделы. Он обычно используется для выделения новых, избранного или недавно просмотренные содержимое приложения.

Содержимое представляется в виде единого, горизонтальной прокрутки список содержимого с меткой, которые располагаются под частью выбранное содержимое (который в данный момент имеет фокус). Если пользователь выбирает конкретной части содержимого, приложение будет запускаться и должна применяться непосредственно в этого содержимого.

Потребуются следующие размеры содержимого:

||Плакат (2:3)|Квадрат (1:1)|HDTV (16:9)|
|---|---|---|---|
|Фактический размер|404x608px|608x608px|908x512px|
|Размер зоны безопасности|380x570px|570x570px|852x479px|
|Размер без фокуса ввода|333x500px|500x500px|782x440px|
|Размер с фокусом ввода|380x570px|570x570px|852x479px|

Компания Apple предоставляет следующие рекомендации для разделенных содержимого строки:

- **Заполнить строку** — необходимо предоставить достаточно содержимое охватывать всю ширину экрана.
- **Масштабирование изображений смешанной** — строка разделенных содержимого может содержать сочетание размеров изображения (из списка, приведенные выше). Если размеры изображения следует смешивать тем не менее, имейте в виду, что дополнительным масштабированием, будет применяться для нормализации отображение содержимого.

<a name="Scrolling-Inset-Banners" />

#### <a name="scrolling-inset-banners"></a>Прокрутка вставки баннеров

При необходимости приложение Xamarin.tvOS можно представить его содержимое полке верхней автоматически прокрутка и циклы сбора баннеров, которые почти весь экран. Этот стиль обычно используется для демонстрации богатую содержимого, например новый ТВ-передачи.

Помимо автоматическую прокрутку, пользователь может получить контроль над баннеров и прокрутки в любом направлении, с помощью удаленного Siri. Внесение небольших, циклические жестов на удаленном Siri при заголовка в фокусе активирует эффекта фокусировки для этого заголовка.

**Изображение баннера (дополнительный широкий)**

|   |   |
|---|---|
|Фактический размер|1940x624px|
|Размер зоны безопасности|1740x620px|
|Размер без фокуса ввода|1740x560px|
|Размер с фокусом ввода|1740x620px|

Прокрутка вставки баннеров можно либо указать статический `.png` или многоуровневый `.lsr` файл.

Компания Apple предоставляет следующие рекомендации для баннеров отступ прокрутка:

- **Объем содержимого** -прокрутка, которую естественным необходимо обеспечить как минимум три (3) ленты. Должен содержать не более 8 (восьми) баннеров, или это затруднить навигации для конечного пользователя.
- **Содержимое текста** — Если требуется плаката текста в должны быть включены в изображение баннера. Если вы используете многослойные образы, ваш текст должен быть верхнего уровня.

См. в разделе Apple [ссылка на платформу TVServices](https://developer.apple.com/library/prerelease/tvos/documentation/TVServices/Reference/TVServices_Ref/index.html#//apple_ref/doc/uid/TP40016412) Дополнительные сведения о добавлении в приложение для предоставления содержимого динамических Top Shelf расширение полки верхней.

<a name="Game-Center-Images" />

## <a name="game-center-images"></a>Образы Game Center

Если вы включили поддержку Game Center Xamarin.tvOS приложения — это игра, потребуются некоторые дополнительные ресурсы изображений:

- **Значки награду** -непрозрачное изображение является обязательным для каждого перехода, который будет автоматически обрезан в круг. Достижения — это элементы, не способному получать фокус.
- **Панель мониторинга иллюстрации** -Необязательное изображение может предоставляться, которые появятся в верхней части панели мониторинга вашего приложения в Game Center. Эти образы, не способному получать фокус.
- **Объект "список лидеров"** -необходимо указать между один (1) для трех (3) образы соотношение сторон 16:9 для каждого списка лидеров, которую поддерживает приложение. Это может быть статических `.png` или многоуровневый `.lsr` файлов. Иллюстрации списка лидеров, способному получать фокус.

||Достижение значки|Иллюстрации панели мониторинга|Объект "список лидеров"|
|---|---|---|---|
|Отображается размер|200x200px|923x150px|Н/Д|
|Фактический размер|320x320px|Н/Д|659x371px|
|Размер зоны безопасности|Н/Д|Н/Д|618x348px|
|Размер без фокуса ввода|Н/Д|Н/Д|548x309px|
|Размер с фокусом ввода|Н/Д|Н/Д|618x348px|

Дополнительные сведения о работе с Game Center см. в разделе Apple [руководство по программированию на Game Center](https://developer.apple.com/library/prerelease/tvos/documentation/NetworkingInternet/Conceptual/GameKit_Guide/Introduction/Introduction.html).
