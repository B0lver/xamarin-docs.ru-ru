# <a name="working-with-tvos-icons-and-images-in-xamarin"></a>Работа со значками и изображениями tvOS в Xamarin

Создание привлекая значков и изображений является важной частью разработки впечатляющих пользовательских интерфейсов для приложений Apple TV. В этом руководстве рассматриваются шаги, необходимые для создания и включения графических ресурсов для приложений Xamarin. tvOS.

- [Изображение запуска](#Launch-Image) — образ запуска отображается при первом запуске приложения и заменяется первым экраном приложения после завершения запуска.
- [Многоуровневые изображения](#Layered-Images) , относящиеся к Apple TV, новые многоуровневые образы Apple работают с эффектом фокусировки для создания объемных эффектов для выбранных элементов. Существует несколько способов [создания многоуровневых изображений](#Creating-Layered-Images).
- [Значок приложения](#App-Icons) — значки требуются не только для домашнего экрана Apple TV, но и для магазина приложений. Они должны быть представлены в виде многоуровневого изображения.
- [Изображение верхнего уровня полки](#Top-Shelf-Image) . Если ваше приложение размещено в верхней строке главного экрана, для выделения функций приложения потребуется изображение в верхнем полке. При необходимости можно предоставить [динамическое содержимое для верхней полки](#Dynamic-Top-Shelf-Content) , чтобы выделить содержимое в приложении.
- [Game Center образы](#Game-Center-Images) . Если приложение является игрой и использует Game Center, потребуется несколько дополнительных образов.
- [Настройка образов проектов Xamarin. tvOS](#Setting-Xamarin.tvOS-Project-Images) . Описание действий, необходимых для задания значка запускаемого образа и приложения для приложения Xamarin. tvOS.

> [!IMPORTANT]
> Все изображения Apple TV имеют разрешение 1x ( `@1x` ), поэтому следует использовать _только_ образы такого размера. Включение более крупной графики с более высоким разрешением не только требует времени для скачивания и использования большего объема памяти и хранилища, но их необходимо динамически масштабировать во время выполнения и негативно повлияет на производительность отрисовки.

<a name="Launch-Image"></a>

## <a name="launch-image"></a>Изображение запуска

Образ запуска — первое, что отображается при первоначальном запуске приложения Xamarin. tvOS на Apple TV, поэтому каждое приложение tvOS должно предоставить образ запуска.

Изображение запуска отображается быстро и дает впечатление, что приложение работает быстро и быстрее. После этого приложение Apple TV заменит образ запуска первым экраном приложения.

Образы запуска не являются возможностью для рекламы или художественного выражения, они существуют только для того, чтобы дать впечатление, что ваше приложение быстро запускается и готово к использованию.

|Размер образа запуска|Примечания|
|---|---|
|1920x1080px|Только неуровневые PNG-файлы|

Компания Apple предлагает следующие рекомендации по проектированию образа запуска приложения:

- **Практически аналогично первому экрану** , экран запуска должен быть как можно ближе к первому экрану приложения. Включение различных рисунков или элементов может привести к неприятию вспышки при появлении первого экрана.
- **Старайтесь не использовать** изображения для запуска текста, так как они статичны и не будут отображаться.
- **Запуск довнплай** . Поскольку пользователи Apple TV часто переключают приложения, вы не должны обратить внимание на процесс запуска приложения.
- **Нет рекламы или фирменной символики** — образ запуска не должен использоваться как экран about или включать фирменную символику, если только она не является статической частью первого экрана приложения. Реклама строго запрещена.

<a name="Layered-Images"></a>

## <a name="layered-images"></a>Многоуровневые изображения

Новые многоуровневые изображения в Apple TV работают с эффектом фокусировки для создания объемных эффектов, позволяющих пользователям, мысленно на диване, подключаться к содержимому на экране в комнате.

Многоуровневые изображения содержат от двух (2) до пяти (5) отдельных слоев, Объединенных для формирования полного образа. За исключением фонового слоя, каждый слой использует Z-порядок вместе с прозрачностью для создания иллюзии глубины. Когда пользователь взаимодействует с многослойным изображением, более высокие Z-упорядоченные слои масштабируются и перекрываются для создания этого результата.

[![Многоуровневых изображений Z-упорядоченная диаграмма](icons-images-images/layered01.png)](icons-images-images/layered01.png#lightbox)

> [!IMPORTANT]
> Многоуровневые изображения необходимы для значков вашего приложения и являются необязательными для других [фокусных элементов](~/ios/tvos/app-fundamentals/navigation-focus.md#Focus-and-Selection) (таких как изображение верхнего полки). Однако компания Apple предлагает использовать многоуровневые изображения для любого изображения, которое может получить фокус в приложении.

Компания Apple предлагает следующие рекомендации по проектированию многоуровневых изображений.

- **Сделать фоновый слой непрозрачным** . фоновый слой (слой 1) **должен** быть непрозрачным, иначе при попытке использования многоуровневого изображения на Apple TV вы получите сообщение об ошибке. Все остальные слои могут содержать несколько уровней прозрачности для улучшения объемного эффекта.
- **Изоляция элементов переднего плана, среднего и фонового элемента** — размещение выразительного элемента (например, игровых символов) на переднем плане. используйте середину для вторичных элементов или теней. Наконец, включите нейтральный фон, чтобы предоставить этап для верхних слоев.
- **Хранить текст на переднем плане** . Если вы не хотите, чтобы текст был скрыт более высоким уровнем, обычно он должен находиться на самом верхнем слое.
- **Использование простого Разуровневого слоя** . эффект фокусировки был незаметным, поэтому слои нужно свести к минимуму, чтобы предотвратить жарринг, нереалистичные эффекты.
- **Включить Сейф в зону** . так как верхние слои можно обрезать во время фокусировкиного воздействия, необходимо создать границу защищенной зоны в каждом слое. Если содержимое слишком близко к границе слоев, оно может быть обрезано. Верхние уровни повышают масштабирование и обрезки, чем нижние слои. См. раздел [изменение размера слоев изображений](#Sizing-Image-Layers) ниже.
- **Предварительный просмотр часто** используемых многослойных изображений следует часто для того, чтобы убедиться в том, что нужно трехмерное воздействие, и что ни одно из содержимого на отдельных слоях не обрезано. Вы должны просмотреть многоуровневые изображения на реальном устройстве Apple TV, чтобы убедиться, что они отображаются должным образом.

По возможности всегда следует использовать встроенные `UIKit` элементы управления для показа многоуровневых изображений, так как они автоматически получат фокусировкиный результат, когда они появятся в фокусе.

### <a name="sizing-image-layers"></a>Изменение размеров слоев изображения

Важно не забывать включить границу _безопасности_ на каждый слой, который будет состоять из многоуровневого изображения. Поскольку отдельные слои можно масштабировать и обрезать во время фокусировки, содержимое слоев можно обрезать, если оно слишком близко к границе слоя:

<a name="Creating-Layered-Images"></a>

### <a name="creating-layered-images"></a>Создание многоуровневых изображений

tvOS работает с многослойными изображениями в следующих форматах:

- **Файлы автомобиля** — это собственный формат каталога активов, созданный Apple. Вы не создаете файлы автомобилей напрямую, они создаются во время компиляции из любых файлов ЛСР и включаются в пакет приложений.
- **Образы ЛСР** — это собственный формат образа, созданный Apple. Используйте [подключаемый модуль Adobe Photoshop фокусировки Export](https://itunespartner.apple.com/assets/downloads/ParallaxExporter_Apps.zip) или средство предварительного [просмотра фокусировки](https://itunespartner.apple.com/assets/downloads/Parallax%20Previewer.dmg) , чтобы создать и предварительно просмотреть многослойные изображения в формате ЛСР.
- **Assets. xcassets** -from 2 (2) до пяти (5) стандартных `.png` отформатированных изображений, включаемых в каталог активов, которые будут скомпилированы в многоуровневый образ в формате автомобиля или ЛСР во время компиляции.
- **Файлы локальной непрерывной репликации** — это собственный формат файла, созданный Apple. Файлы LCR предназначены для использования в качестве дополнительного содержимого, скачанного с одного из серверов содержимого. Файл LCR не должен включаться в пакет приложений. Файлы LCR создаются из файлов ЛСР или Photoshop с помощью `layerutil` средства командной строки, входящего в состав Xcode.

В области предварительного просмотра отображается каждый слой, который образует завершенное многослойное изображение:

При предварительном просмотре многоуровневого изображения можно использовать мышь для поворота изображения и предварительного просмотра фокусировки. **+** **-** Для добавления и удаления слоев используйте кнопки (плюс) и (минус).

При создании многоуровневого изображения его можно экспортировать в формате ЛСР и включать в пакет приложения.

Дополнительные сведения о создании и предварительном просмотре многоуровневых изображений см. в разделе " [Создание иллюстрации фокусировки](https://developer.apple.com/library/prerelease/tvos/documentation/General/Conceptual/AppleTV_PG/CreatingParallaxArtwork.html#//apple_ref/doc/uid/TP40015241-CH19-SW1) " компании Apple по [программированию приложений для tvOS](https://developer.apple.com/library/prerelease/tvos/documentation/General/Conceptual/AppleTV_PG/).

<a name="App-Icons"></a>

## <a name="app-icons"></a>Значки приложений

Для вашего приложения Xamarin. tvOS требуется не только значок приложения на начальном экране Apple TV, но также значок для магазина приложений. Значок приложения — это первое изменение, которое поможет вам получить представление о потенциальных пользователях и должно быстро сообщить о назначении приложения.

[![Значок приложения](icons-images-images/icon01.png)](icons-images-images/icon01.png#lightbox)

Каждое приложение должно предоставить как небольшое, так и крупную версию своего значка приложения. Маленький значок будет использоваться на начальном экране Apple TV при установке приложения. В App Store используется большая версия. Значок крупного приложения должен имитировать вид мелкой версии значка.

|Маленький значок|Решение|Крупный значок|Решение|
|---|---|---|---|
|Фактический размер|400x240px||1280x768px|
|Размер зоны безопасности|370x222px|||
|Нефокусный размер|300x180px|||
|Размер с сортировкой|370x222px|||

> [!IMPORTANT]
> Значки приложения должны быть представлены в виде **многоуровневых изображений**. Дополнительные сведения см. в разделе [многослойное изображение](#Layered-Images) выше.

Компания Apple предоставляет следующие рекомендации по созданию значков приложений:

- **Предоставьте единую точку фокусировки** — Разработайте значок с одной фокусной точкой, размещенной непосредственно в центре значка.
- **Используйте простой фон** — не заключайте фон значка, чтобы выделить верхние уровни. Рассмотрите возможность использования простого цвета или нетонкого градиента.
- **Ограничьте объем текста** , так как имя приложения будет отображаться под значком, когда оно выбрано пользователем, следует включить только текст, если он необходим для проектирования значка.
- **Не используйте снимки экрана** . снимки экрана слишком сложны для значков и не позволяют пользователю быстро видеть назначение приложения.
- Не **Отображать значки** — tvOS автоматически применяет маску, которая немного округляет углы значков. Не включайте это округление.
- **Разделяйте слои с осторожностью** — текст должен находиться на верхнем уровне, вторичных элементах в середине и нейтральном фоне, который позволяет использовать верхние слои.
- **Тщательно Используйте градиенты и тени** — градиенты и тени могут конфликтовать с эффектом фокусировки, поэтому их следует использовать аккуратно. Простейшие стили с верхним и темным градиентом работают наилучшим образом. Обычно тени работают наилучшим образом в виде четких краев.
- Изменение **прозрачности слоя** — используйте разные уровни прозрачности на верхних уровнях значка приложения, чтобы увеличить объемный эффект. Фоновый слой должен быть непрозрачным или приведет к ошибке.

<a name="Top-Shelf-Image"></a>

## <a name="top-shelf-image"></a>Изображение верхнего полки

Если пользователь поместил ваше приложение Xamarin. tvOS в верхней строке на начальном экране Apple TV, то при выборе приложения пользователем будет отображаться большое изображение в верхнем полке. Этот образ должен выделять функции приложения или предоставлять прямые ссылки на его содержимое.

[![Пример изображения верхнего полки](icons-images-images/topshelf01.png)](icons-images-images/topshelf01.png#lightbox)

Верхнее изображение полки может быть предоставлено как один статический `.png` или `.lsr` файл (см. раздел [Создание многоуровневых изображений](#Creating-Layered-Images)) или динамически создаваться во время выполнения в виде одной строки элементов с фокусом (см. статью [динамическое содержимое с максимальным сроком хранения](#Dynamic-Top-Shelf-Content) ниже).

|Размер верхнего образа полки|Примечания|
|---|---|
|1920x720px|Файл static. PNG или слоеd. ЛСР|

Компания Apple предоставляет следующие рекомендации по созданию ваших основных образов.

- **Использовать насыщенные статические изображения** . Если ваше приложение не предоставляет динамическое содержимое, его верхнее изображение с полкой не сможет сосредоточиться. Используйте это изображение для выделения функций приложения или вашей торговой марки.
- **Ссылка на содержимое приложения** — динамические макеты с максимальным сроком хранения предоставляют быструю ссылку на содержимое, которое ваш пользователь находит наиболее важные в приложении. Используйте эту область, чтобы создать быструю ссылку для запуска приложения и немедленного перехода к заданному содержимому.
- **Демонстрация новейшего содержимого** . доступное содержимое с максимальным сроком хранения может нарисовать пользователя в приложении и сделать его более использующим его. Используйте его в качестве области для демонстрации самого высокого рейтинга или самого нового содержимого.
- **Персонализированное содержимое** — пользователи размещают наиболее часто используемые или избранные приложения в верхней строке главного экрана. Используйте верхнюю полку, чтобы отобразить содержимое, которое им наиболее интересно.
- **Реклама** запрещена — реклама строго запрещена для отображения на верхнем полке. Вы можете показать Последнее содержимое для покупки, но не должны отображаться сведения о ценах.

<a name="Dynamic-Top-Shelf-Content"></a>

### <a name="dynamic-top-shelf-content"></a>Динамическое содержимое верхней полки

Вместо того чтобы отображать статическое верхнее изображение, верхняя полка может содержать динамическую строку [элементов с фокусом](~/ios/tvos/app-fundamentals/navigation-focus.md#Focus-and-Selection) или динамический набор заголовков с прокруткой. Оба этих динамического стиля позволяют выделить содержимое, предоставленное приложением, или перейти к наиболее часто используемым функциям.

<a name="Sectioned-Content-Row"></a>

#### <a name="sectioned-content-row"></a>Строка содержимого с разделом

Этот динамический тип содержимого «верхняя полка» представляет собой одну строку прокрутки, прокручиваемые элементы при необходимости разбивается на разделы. Обычно он используется для выделения нового, избранного или недавно просмотренного содержимого приложения.

Содержимое представляется в виде одного горизонтально прокручиваемого списка содержимого с меткой, отображаемой в текущем фрагменте содержимого (в данный момент фокус). Если пользователь выберет заданный фрагмент содержимого, приложение будет запущено и его следует будет использовать непосредственно в этом содержимом.

Потребуется следующий размер содержимого:

|Size|Плакат (2:3)|Квадратный (1:1)|ТВВЧ (16:9)|
|---|---|---|---|
|Фактический размер|404x608px|608x608px|908x512px|
|Размер зоны безопасности|380x570px|570x570px|852x479px|
|Нефокусный размер|333x500px|500x500px|782x440px|
|Размер с сортировкой|380x570px|570x570px|852x479px|

Apple предоставляет следующие советы по строке с разделами содержимого:

- **Заполните строку** — необходимо предоставить достаточное содержимое, чтобы охватить всю ширину экрана.
- **Масштабирование смешанных изображений** — строка с разделенным содержимым была разработана для хранения комбинаций размеров изображений (из приведенного выше списка). Если же вы выполняете смешение размеров образов, имейте в виду, что для нормализации отображаемого содержимого будет применено дополнительное масштабирование.

<a name="Scrolling-Inset-Banners"></a>

#### <a name="scrolling-inset-banners"></a>Прокрутка отступов для заголовков

При необходимости приложение Xamarin. tvOS может представлять свое содержимое в верхней полке как автоматическую прокрутку и циклическую коллекцию баннеров, которые почти заполняют экран. Этот стиль обычно используется для демонстрации богатого нового содержимого, такого как новые телепередачи.

Помимо автоматической прокрутки, пользователь может управлять баннерами и прокручивать в любом направлении с помощью Siri Remote. Создание небольшого, циклического жеста на Siri удаленном, когда баннер находится в фокусе, активирует для этого баннера Фокусировкиный результат.

**Изображение баннера (очень широкой)**

| Size | Решение |
|---|---|
|Фактический размер|1940x624px|
|Размер зоны безопасности|1740x620px|
|Нефокусный размер|1740x560px|
|Размер с сортировкой|1740x620px|

Чередующиеся заголовки можно указать как статический `.png` или многоуровневый `.lsr` файл.

Apple предоставляет следующие рекомендации для плакатов с прокруткой.

- **Сумма содержимого** . для работы с прокруткой необходимо предоставить как минимум три (3) баннера. Необходимо включить не более восьми (8) баннеров или сделать навигацию для конечного пользователя несложно.
- **Текст содержимого** . Если в баннере требуется текст, он должен быть добавлен в изображение баннера. Если используются многослойные изображения, текст должен находиться на верхнем слое.

Дополнительные сведения о добавлении расширения верхнего полки в приложение для предоставления динамического содержимого в верхней полке см. в [справочнике по платформе твсервицес](https://developer.apple.com/library/prerelease/tvos/documentation/TVServices/Reference/TVServices_Ref/index.html#//apple_ref/doc/uid/TP40016412) для Apple.

<a name="Game-Center-Images"></a>

## <a name="game-center-images"></a>Образы Game Center

Если ваше приложение Xamarin. tvOS является игрой и вы включили поддержку Game Center, потребуется еще несколько ресурсов изображений:

- **Значки достижений** — непрозрачный образ требуется для каждого достижения, которое будет автоматически обрезано до окружности. Достижения — это элементы, не имеющие фокуса.
- **Иллюстрация панели мониторинга** . можно указать дополнительный образ, который будет отображаться в верхней части панели мониторинга приложения в Game Center. Эти образы не могут получать фокус.
- **Список лидеровная Иллюстрация** . для каждого список лидеров, поддерживаемого приложением, необходимо указать от одного (1) до трех (3 16:9) изображений пропорций. Это могут быть либо статические `.png` , либо многослойные `.lsr` файлы. Иллюстрация список лидеров доступна для фокусировки.

|Size|Значки достижений|Иллюстрация панели мониторинга|Иллюстрация список лидеров|
|---|---|---|---|
|Видимый размер|200x200px|923x150px|н/д|
|Фактический размер|320x320px|н/д|659x371px|
|Размер зоны безопасности|Недоступно|Недоступно|618x348px|
|Нефокусный размер|Недоступно|Недоступно|548x309px|
|Размер с сортировкой|Недоступно|Недоступно|618x348px|

Дополнительные сведения о работе с Game Center см. в разделе [руководств по программированию Game Center](https://developer.apple.com/library/prerelease/tvos/documentation/NetworkingInternet/Conceptual/GameKit_Guide/Introduction/Introduction.html)Apple.
