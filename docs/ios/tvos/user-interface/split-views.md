# <a name="working-with-tvos-split-view-controllers-in-xamarin"></a>Работа с tvOS разбиения контроллеров представления в Xamarin

Контроллер представления разбиения предоставляет и управляет Master и контроллер представления сведений side-by-side, на экране, в то же время. Контроллеры представлений разбиения представляют постоянные, способному получать фокус содержимое в основное представление (меньший раздел слева) и связанные с сведений в представлении «Подробности» (разделе большего размера в правом углу).

[![](split-views-images/intro01.png "Пример разделенное представление")](split-views-images/intro01.png#lightbox)

<a name="About-Split-View-Controllers" />

## <a name="about-split-view-controllers"></a>О контроллеров представлений разбиения

Как уже говорилось выше, контроллер представления разбиения управляет основной таблицы и контроллер представления сведений, которые предоставляются side-by-side, вместе с шаблоном выполняется меньшее представление в левой части, подробные данные большего размера в правом. 

Кроме того, было скрыто или показано Master View Controller может при необходимости: 

[![](split-views-images/intro02.png "Контроллер представления главного скрыто")](split-views-images/intro02.png#lightbox)

Для представления списка фильтрацию содержимого с помощью категорий в основное представление и отфильтрованные результаты в представлении «Подробности» часто используют контроллеры представления разбиения. Это обычно представляется в виде таблицы представления с левой стороны экрана и [представление коллекции](~/ios/tvos/user-interface/collection-views.md) справа.

При разработке пользовательский интерфейс, который требуется контроллер разделенного представления, Apple предлагает, использовать Master и контроллеров представления сведений, которые не изменяются (только изменения содержимого, не структуры). Если требуется для переключения от контроллеров представлений, лучше использовать контроллер навигации в качестве базового контроллера представления, которые требуется изменить (основной или подробности).

Apple имеет следующие рекомендации по работе с контроллерами представлений разбиения.

- **Используйте правильный процент разбиения** -по умолчанию контроллер разделенного представления использует треть высоты экрана для контроллера представления главного и две трети для контроллера представления сведений. При необходимости можно использовать разбиваемые разбиения. Выберите правильный процент, чтобы сделать ваши содержимого отображаются с балансировкой на экране.
- **Сохранять выделение Main** — при содержимое на подробное представление могут изменить ответ на выбор пользователя в основное представление, образец содержимого должна быть устранена. Кроме того ясно показывают текущего выбранного элемента в представлении главного.
- **Использовать единую единой Title** -как правило, необходимо использовать заголовок единый, выровненные по центру в представлении «Подробности», а не с названием в подробные данные и основное представление.

<a name="Working-with-Split-View-Controllers" />

## <a name="working-with-split-view-controllers"></a>Работа с контроллерами представлений разбиения

Как уже говорилось выше, контроллер разделенного представления часто используется в ситуациях, где отображается отфильтрованное содержимое для пользователя. Основные категории отображаются в левой части основного представления и отфильтрованные результаты справа в представлении «Подробности» на основании выбора пользователя.

<a name="Accessing-Master-and-Detail" />

### <a name="accessing-master-and-detail"></a>Доступ к основные и подробные

Если вам требуется доступ к основные и контроллеров представлений сведений программными средствами, используйте `ViewControllers ` свойство контроллер разделенного представления. Пример:

```csharp
// Gain access to master and detail view controllers
var masterController = ViewControllers [0] as MasterViewController;
var detailController = ViewControllers [1] as DetailViewController;
```

Он представляется в виде массива, где первый элемент (0) в контроллер представления Master, а второй элемент (1) заключается в степени детализации.

<a name="Accessing-Detail-from-Master" />

### <a name="accessing-detail-from-master"></a>Доступ к детализации из шаблона

Поскольку подробные сведения обычно отображается в представлении «Подробности», в зависимости от выбора пользователя в базе данных Master, вам потребуется получить доступ к детализации с главного сервера.

Для этого проще всего предоставить свойство в классе контроллера главного представления, например:

```csharp
public DetailViewController DetailController { get; set;}
```

В контроллер разделенного представления, переопределите `ViewDidLoad` метод tie два представления и друг с другом. Пример:

```csharp
public override void ViewDidLoad ()
{
    base.ViewDidLoad ();

    // Gain access to master and detail view controllers
    var masterController = ViewControllers [0] as MasterViewController;
    var detailController = ViewControllers [1] as DetailViewController;

    // Wire-up views
    masterController.SplitViewController = this;
    masterController.DetailController = detailController;
    detailController.SplitViewController = this;
}
```

На контроллере представления сведений, образце можно использовать для представления новых данных, при необходимости можно предоставить свойства и методы.

<a name="Showing-and-Hiding-Master" />

### <a name="showing-and-hiding-master"></a>Отображение и скрытие Master

При необходимости можно отобразить и скрыть Master контроллера представления с помощью `PreferredDisplayMode` свойство контроллер разделенного представления. Пример:

```csharp
// Show hide split view
if (SplitViewController.DisplayMode == UISplitViewControllerDisplayMode.PrimaryHidden) {
    SplitViewController.PreferredDisplayMode = UISplitViewControllerDisplayMode.AllVisible;
} else {
    SplitViewController.PreferredDisplayMode = UISplitViewControllerDisplayMode.PrimaryHidden;
}
```

`UISplitViewControllerDisplayMode` Перечисление определяет представления контроллера представления Master как одно из следующих:

- **Автоматическое** -tvOS будет контролировать представление основные и подробные представления.
- **PrimaryHidden** -это скрывает контроллера представления Master.
- **AllVisible** -отображение главного и контроллеров представлений сведений side-by-side. Это нормально, представление по умолчанию.
- **PrimaryOverlay** -контроллер представления сведений расширяет в группе и покрывается Master.

Чтобы получить текущее состояние представления, используйте `DisplayMode` свойство контроллер разделенного представления.
