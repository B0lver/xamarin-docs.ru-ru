# <a name="hello-tvos-quick-start-guide"></a>Привет, tvOS краткое руководство

_В этом руководстве описывается создание первого приложения Xamarin.tvOS и его инструментов разработки. Поддерживает также конструктор Xamarin, который предоставляет элементы управления пользовательского интерфейса для кода и показывает, как построить, выполнять и тестировать приложения Xamarin.tvOS._

Apple выпустила 5-го поколения Apple TV, Apple TV 4 КБ, которая выполняет tvOS 11.

Платформа Apple TV доступен для разработчиков, позволяя им для создания многофункциональных, насыщенные приложения и освободить их в магазине Apple большом встроенных приложений.

Если вы знакомы с разработкой Xamarin.iOS, вы должны найти переход tvOS довольно проста. Большинство интерфейсов API и возможности идентичны, однако многие общие интерфейсы API недоступны (например, WebKit). Кроме того, работаете с удаленной Siri может возникнуть некоторые проблемы проектирования, которые не существуют в сенсорный экран на основе устройств iOS.

Это руководство позволит получить общие сведения о работе с tvOS в приложение Xamarin. Дополнительные сведения о tvOS см. в разделе Apple [Подготовка для Apple TV 4K](https://developer.apple.com/tvos/) документации.

## <a name="overview"></a>Обзор

Xamarin.tvOS позволяет разрабатывать полностью собственных приложений Apple TV в C# и .NET с помощью же библиотеки OS X и элементы управления интерфейса, которые используются при разработке в *Swift* (или *Objective-C*) и *Xcode*.

Кроме того так как они написаны на C# и .NET Xamarin.tvOS, общие, серверный код можно использовать совместно с Xamarin.iOS, Xamarin.Android и Xamarin.Mac приложений; При этом собственным взаимодействием доставки для каждой платформы.

В этой статье приведены основные понятия, необходимые для создания с помощью Visual Studio и Xamarin.tvOS посредством процесс создания базового приложения Apple TV **Hello, tvOS** на кнопке есть приложение, которое подсчитывает число попыток выполнен щелчок:

[![](hello-tvos-images/run05.png "Запуска примера приложения")](hello-tvos-images/run05.png#lightbox)

Мы расскажем следующие понятия:

-  **Visual Studio для Mac** — Общие сведения о Visual Studio для Mac и создание приложений Xamarin.tvOS с ним.
-  **Составляющие приложение Xamarin.tvOS** — что Xamarin.tvOS приложение состоит из.
-  **Создание пользовательского интерфейса** — как использовать конструктор Xamarin для операций ввода-вывода для создания пользовательского интерфейса.
-  **Развертывание и тестирование** — как запуск и тестирование приложения в tvOS симулятора и на реальных tvOS оборудования.

## <a name="anatomy-of-a-xamarintvos-app"></a>Составляющие приложение Xamarin.tvOS

Если вы знакомы с iOS программирования, вы заметите, что много здесь сходства. На самом деле tvOS 9 является подмножеством iOS 9, поэтому здесь пересекает много основные понятия.

Давайте рассмотрим файлы в проекте:

-   `Main.cs` — этот файл содержит основную точку входа приложения. При запуске приложения она содержит самый первый класс и выполняемый метод.
-   `AppDelegate.cs` — Этот файл содержит класс основного приложения, который отвечает за прослушивание событий из операционной системы.
-   `Info.plist` — Этот файл содержит свойства приложения, такие как имя приложения, значки, и т. д.
-   `ViewController.cs` — Это класс, представляющий главное окно и управляет жизненным циклом.
-   `ViewController.designer.cs` — Этот файл содержит коммуникационного кода, позволяет интегрировать с главного экрана пользовательского интерфейса.
-  `Main.storyboard` — Пользовательский Интерфейс для главного окна. Этот файл можно созданных и сохраненных в конструкторе Xamarin iOS.

В следующих разделах мы будем посмотрите через некоторые из этих файлов. Мы изучим их более подробно далее, но рекомендуется понять основы их сейчас.

### <a name="maincs"></a>Main.cs

`Main.cs` Файл содержит статическую `Main` методу, который создает новый экземпляр приложения Xamarin.tvOS и передает имя класса, который будет обрабатывать события операционной системы, в нашем случае это `AppDelegate` класса:

```csharp
using UIKit;

namespace Hello_tvOS
{
    public class Application
    {
        // This is the main entry point of the application.
        static void Main (string[] args)
        {
            // if you want to use a different Application Delegate class from "AppDelegate"
            // you can specify it here.
            UIApplication.Main (args, null, "AppDelegate");
        }
    }
}
```

`AppDelegate.cs` Файл содержит нашей `AppDelegate` класс, который отвечает за создание окна и прослушивание событий операционной системы:

Этот код является, возможно, для ознакомления, если вы создали приложение iOS до, но это довольно просто. Давайте рассмотрим важные строки.

Во-первых давайте рассмотрим объявлении переменной уровня класса:

```csharp
public override UIWindow Window {
            get;
            set;
        }

```

`Window` Свойство предоставляет доступ к главного окна. использует tvOS, называемым *Model View Controller* шаблон (MVC). Как правило для каждого окна, создаваемые вами (и для многих других задач в windows), отсутствует контроллер, который отвечает за окна жизненного цикла, например, для отображения, добавление новых представлений (элементы управления) он и т. д.

Теперь у нас есть `FinishedLaunching` метод. Этот метод выполняется после создания экземпляра приложения, и он отвечает за фактически Создание окна приложения и начать процесс отображения представления в нем. Поскольку нашего приложения использует раскадровки для определения его пользовательского интерфейса, без дополнительного кода, здесь не требуется.

Существует множество других методов, которые входят в шаблон, например `DidEnterBackground` и `WillEnterForeground`. Их можно безопасно удалить, если в приложении не используются события приложения.

### <a name="accessing-the-controls-in-the-code-behind"></a>Доступ к элементам управления, в коде

Существует два основных способа для доступа к элементам управления, которые были добавлены в конструкторе операций ввода-вывода из кода:

* Создание обработчика событий для элемента управления.
* Задание элемента управления имя, чтобы мы могли сослаться позже.

При любой из этих добавляются, частичный класс в `ViewController.designer.cs` обновляется для отражения изменений. Это позволит получить доступ к элементам управления в представление-контроллер.

## <a name="writing-the-code"></a>Написание кода

С нашей пользовательский интерфейс, который создан и открыт для кода через элементы пользовательского интерфейса **торговцам** и **действия**, и наконец можно написать код, обеспечивающий функции программы.

В нашем приложении каждый раз при нажатии кнопки первой мы собираемся обновить метку, чтобы показать, сколько раз была нажата кнопка. Для выполнения этой задачи необходимо открыть `ViewController.cs` для редактирования, дважды щелкнув его в файл **Pad решение**:

```csharp
using System;
using Foundation;
using UIKit;

namespace Hello_tvOS
{
    public partial class ViewController : UIViewController

        ...
```

Далее в этом классе (`ViewController`), нам нужно переопределить **ViewDidLoad** метода и добавьте код для задания начального сообщения для метку:

```csharp
public override void ViewDidLoad ()
{
    base.ViewDidLoad ();

    // Set the initial value for the label
    ClickedLabel.Text = "Button has not been clicked yet.";
}
```

Необходимо использовать `ViewDidLoad `, вместо другой метод, такой как `Initialize`, так как `ViewDidLoad ` называется *после* ОС загружается и создать экземпляр пользовательского интерфейса из `.storyboard` файла. При попытке доступа к элементу управления подпись перед `.storyboard` файл полностью загружены и создать экземпляр, получаются `NullReferenceException` ошибка, так как элемент управления label может быть еще не создан.


## <a name="testing-the-application"></a>Тестирование приложения

Пришло время для построения и запуска нашего приложения, чтобы убедиться в том, что оно работает ожидаемым образом. Можно собрать и запустить за один шаг, или можно создать не запуская его.

-   **Освободить** — также создает сборки выпуска '' файл, но он не содержит отладочную информацию, он меньше и выполняется быстрее.
