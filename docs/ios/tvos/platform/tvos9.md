# <a name="introduction-to-tvos-9"></a>Общие сведения о tvOS 9

_В этой статье описаны все новые и измененные интерфейсы API и возможности, доступные в tvOS 9 для разработчиков Xamarin.tvOS._


Apple выпустила 4-го поколения оборудования Apple TV, содержащих модернизированный, удаленное включение сенсорного ввода, новый tvOS операционной системой (с учетом iOS 9).

В первый раз tvOS открывает Apple TV платформы для разработчика, что позволяет создавать информативные, насыщенные приложения и освободить их в магазине Apple большом встроенных приложений в процессе, то же самое записи и выпуск приложения для iOS с помощью iTunes App Хранилище.

Если вы знакомы с разработкой Xamarin.iOS, вы должны найти переход tvOS довольно проста. Большинство интерфейсов API и возможности идентичны, однако многие общие интерфейсы API недоступны (например, WebKit). Кроме того, работаете с удаленной Siri может возникнуть некоторые проблемы проектирования, которые не существуют в сенсорный экран на основе устройств iOS.

Это руководство позволит получить основные сведения о всех новых и измененных интерфейсы API и возможности, доступные в tvOS 9 для разработчиков Xamarin.tvOS. Дополнительные сведения о tvOS см. в разделе Apple [разработка для нового Apple TV](https://developer.apple.com/tvos/) документации.

<a name="Supported-and-Unsupported-Capabilities" />

## <a name="supported-and-unsupported-capabilities"></a>Поддерживаемые и неподдерживаемые возможности

tvOS приложения, работающие на Apple TV имеются следующие поддерживаемые возможности и функции:

 - Группы приложений
 - Фоновые режимы
 - Защита данных
 - Game Center
 - Игровые устройства
 - iCloud
 - Покупки из приложений
 - Обмен связками ключей

Не поддерживаются следующие функции и возможности:

 - Apple Pay
 - Приложение "песочницы"
 - Связанные домены
 - HealthKit
 - HomeKit
 - Звук между приложениями
 - Карты
 - Личная VPN
 - Push-уведомления
 - Wallet
 - Конфигурация беспроводных периферийных устройств

См. в разделе нашей [поддерживается сборки](~/ios/tvos/internals/assemblies.md) и [поддерживается платформами](~/ios/tvos/internals/frameworks.md) Дополнительные сведения см.

<a name="Apple-TV-Hardware" />

## <a name="apple-tv-hardware"></a>Apple TV оборудования

Новый Apple TV имеет следующие характеристики оборудования:

 - 64-разрядный процессор A8
 - 32 или 64 ГБ хранилища
 - 2 ГБ ОЗУ
 - 10/100 Мбит/с Ethernet
 - 802.11a/b/g/n/ac Wi-Fi
 - разрешение 1080p
 - HDMI
 - USB-порт C (для разработчиков и диагностики используйте только)
 - Новый Siri удаленный или Apple TV удаленного (в зависимости от региона)

### <a name="siri-remote"></a>Удаленное Siri

На основе области, предоставленных Apple TV удаленного будет иметь либо одной конфигурации: удаленный Siri или Apple TV удаленной.

Удаленное Siri в настоящее время доступна в следующих странах:

 - Австралия
 - Канада
 - Франция
 - Германия
 - Япония
 - Испания
 - Великобритания
 - США

Все остальные страны получит Apple TV удаленного замены кнопки Siri на кнопку поиска, которая появится экран поиска по умолчанию с помощью ввода текста для поиска:

[ ![](tvos9-images/remote02.png "Удаленное Siri")](tvos9-images/remote02.png)

Дополнительные сведения см. в разделе нашей [Siri удаленного и контроллеров Bluetooth](~/ios/tvos/platform/remote-bluetooth.md) документации.

<a name="Apple-TV-Provisioning" />

## <a name="apple-tv-provisioning"></a>Подготовка Apple TV

Так же, как разработка для операций ввода-вывода, новые tvOS потребуется нужный профиль подготовки для разработки и распространения на основе членства и удостоверений подписи, уже установлен в Apple.

Правильную подготовку также необходим для доступа к возможностям tvOS, например iCloud KVS или CloudKit хранилищ данных. См. в разделе нашей [ресурсами и хранилищем данных](~/ios/tvos/app-fundamentals/resources-data-storage.md) сведения о поддержке в приложениях Xamarin.tvOS iCloud.

Профили подготовки создаются и устанавливаются так же, как работа с приложениями Xamarin.iOS. Таким образом, см. в разделе iOS [подготовки устройства](~/ios/get-started/installation/device-provisioning/index.md) документации для получения дополнительных сведений.

<a name="Apple-TV-Apps" />

## <a name="apple-tv-apps"></a>Apple TV приложений

Новое оборудование для Apple TV и tvOS 9 поддерживает два типа приложений: традиционные и клиент серверных приложений.

<a name="Traditional-Apps" />

### <a name="traditional-apps"></a>Традиционные приложения

Традиционные приложения, приобретенные в магазине приложений Apple TV и устанавливаются непосредственно на устройстве. Эти приложения могут быть игры, служебных программ или приложений мультимедиа, разработанных с использованием того же платформы и методы, как приложения Xamarin.iOS.

Приложения Apple TV размером не более 200 МБ и можно загрузить дополнительные 2ГБ содержимого с помощью ресурсы по требованию. См. в нашем [ресурсами и хранилищем данных](~/ios/tvos/app-fundamentals/resources-data-storage.md) для получения дополнительной информации.

См. наш [Hello, tvOS краткое руководство по](~/ios/tvos/get-started/hello-tvos.md) ознакомиться с инструментами и понятиями, необходимые для разработки приложений tvOS, с помощью Xamarin.tvOS.

<a name="Summary" />

### <a name="client-server-apps"></a>Клиент сервер приложений

Помимо установленных традиционные приложения Apple TV упрощает создание потоковая передача клиент сервер веб-приложений с помощью веб-технологий (HTTPS, XML и JavaScript). Будет разработки пользовательского интерфейса с помощью языка разметки TVML Apple и использовать JavaScript для определения поведения приложения, с помощью TVMLKit.

Дополнительные сведения см. в разделе Apple [языка Apple TV разметки](https://developer.apple.com/library/prerelease/tvos/documentation/LanguagesUtilities/Conceptual/ATV_Template_Guide/index.html#//apple_ref/doc/uid/TP40015064), [ссылке платформы TVJS](https://developer.apple.com/library/prerelease/tvos/documentation/TVMLJS/Reference/TVJSFrameworkReference/index.html#//apple_ref/doc/uid/TP40016076), [ссылке платформы TVMLKit](https://developer.apple.com/library/prerelease/tvos/documentation/TVMLKit/Reference/TVMLKit_Collection/index.html#//apple_ref/doc/uid/TP40016429), [о HTTP Live Streaming](https://developer.apple.com/library/prerelease/tvos/referencelibrary/GettingStarted/AboutHTTPLiveStreaming/about/about.html#//apple_ref/doc/uid/TP40013978) и [HLS разработки спецификаций для Apple TV](https://developer.apple.com/services-account/download?path=/Documentation/HLS_Authoring_Specification_for_Apple_TV/HLS_Authoring_Specification_for_Apple_TV.pdf) документации.

<a name="User-Interface-Challenges" />

## <a name="user-interface-challenges"></a>Проблемы пользовательского интерфейса

В отличие от iOS или OS X Apple TV нет сенсорного экрана или мышь, пользователи могут напрямую выберите и взаимодействовать с приложение или его содержимое. Вместо этого они пользователя нового удаленного Siri или Bluetooth игры контроллер для навигации пользовательского интерфейса приложения. Дополнительные сведения см. в разделе нашей [Siri удаленного и контроллеров Bluetooth](~/ios/tvos/platform/remote-bluetooth.md) документации.

Кроме того на производительность работы пользователя значительно отличается от iOS или Mac приложений, как правило, один пользовательским интерфейсом. С Apple TV взаимодействий с пользователем, как правило, быть более социальных сетей, по своей природе, где несколько человек может располагаться на диване взаимодействия с одного приложения или друг от друга. Для проектирования успешно работа приложений Apple TV (новое приложение или перенос существующего), эти изменения необходимо принять во внимание. 

<a name="Summary" />

### <a name="working-with-focus-and-parallax-images"></a>Работа с фокусом и фокусировки изображения

Как уже говорилось выше, пользователи Xamarin.tvOS приложения не будут взаимодействовать с его интерфейс непосредственно как с iOS они tap изображения на экране устройства, но косвенно из в комнате с помощью удаленного Siri. Для представления и управления взаимодействием с пользователем, Apple TV использует модель на основе фокуса. 

При изменении фокуса слабая анимации и эффектов (например, эффект фокусировки на образах) используются четко определить элемент пользовательского интерфейса, который в данный момент имеет фокус.

Если пользователь делает медленно, циклические жестов на удаленном Siri, элемент с фокусом ввода будет sway в режиме реального времени в ответ на такое перемещение. По мере sway оформленный sheen применяется до создания рабочей области отображаются для освещения образа. После заданного периода бездействия любое содержимое элемента в фокусе затемняет и фокуса элемента будет дальнейшее расширение.

Дополнительные сведения см. в разделе нашей [работа с навигации и фокус](~/ios/tvos/app-fundamentals/navigation-focus.md) и [работы со значками и образы](~/ios/tvos/app-fundamentals/icons-images.md) документации.

<a name="The-Home-Screen" />

### <a name="the-home-screen"></a>Начальный экран

Apple TV начального экрана показано все приложения, которые устанавливаются и предоставляет способ доступа к персональных настроек пользователя.

[ ![](tvos9-images/home01.png "Начальный экран")](tvos9-images/home01.png)

Пользователь переходит сетки значков приложения на сенсорный ввод Siri удаленного доступа с помощью фокус, чтобы выбрать приложение и запустить его. Значок приложения — первого вероятность значительные впечатление на потенциальных пользователей и необходимо связаться цели приложения с одного взгляда.

Все приложения должны предоставить небольшого и большой версии его значка приложения. Мелкий значок будет использоваться для Apple TV начального экрана при установке приложения. Большой версии используется App Store. Большой значок приложения должен имитировать внешний вид версии маленького значка.

Дополнительные сведения см. в разделе нашей [работы со значками и образы](~/ios/tvos/app-fundamentals/icons-images.md) документации.

<a name="The-Top-Shelf" />

### <a name="the-top-shelf"></a>Верхняя Полка

Если отправки Xamarin.tvOS приложения в верхней строке на Apple TV начального экрана, при выборе пользователем приложения будет отображаться большое изображение Полка верхней. Этот образ необходимо ознакомиться с функциями приложения или прямых ссылок на содержимое.

[ ![](tvos9-images/topshelf01.png "Верхняя Полка")](tvos9-images/topshelf01.png)

Изображение сверху Полка либо могут быть предоставлены как один статический `.png` или `.lsr` файл или его можно создать динамически во время выполнения как одна строка может иметь фокус элементов.

Вместо отображения статическое изображение Полка верхней, оно может содержать динамические строки или элементов может иметь фокус или динамический набор прокрутки ленты. Оба эти динамические стиля позволяют выделить содержимое, предоставляемый приложением или переход в наиболее часто используемыми функциями.
