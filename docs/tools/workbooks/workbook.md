# <a name="interactive-workbooks"></a>Интерактивные Workbooks

Книги можно использовать как отдельное приложение, отдельно от интегрированной среды разработки.

Чтобы приступить к созданию новой книги, запустите приложение Workbooks. Если это еще не установили, посетите [установки](~/tools/workbooks/install.md#install) страницы. Вам будет предложено создать книгу в платформе, которая будет автоматически подключаться к агента приложение позволяет визуализировать документа в режиме реального времени.

Если приложение Workbooks уже выполняется, можно создать новый документ, перейдя по адресу **файл > создать**.

Книги можно сохранить и позже открыть в приложении. Вы можете затем использовать их с другими людьми для демонстрации идеи, изучать новый API или изучения новых концепций.

## <a name="code-editing"></a>Редактирование кода

Окно обеспечивает Автозавершение кода, выделение синтаксиса цветом, live встроенный-diagnostics и поддержку многострочного оператора.

Xamarin Workbooks сохраняются в `.workbook` файл, который является файлом CommonMark с определенными метаданными, вверху (см. в разделе [типы файлов книг](#workbooks-files-types) узнать больше о том, как можно сохранять книги).

### <a name="nuget-package-support"></a>Поддержка пакета NuGet

Непосредственно в Xamarin Workbooks поддерживаются многие популярные пакеты NuGet. Пакеты можно найти, перейдя по адресу **файл > добавить пакет**. Добавление пакета автоматически добавит `#r` инструкций, ссылающихся на сборки пакетов, что позволяет использовать их прямо сейчас.

При сохранении книги со ссылками на пакеты, также сохраняются эти ссылки. Если книга совместно с другими пользователями, указанные в ссылках пакеты будут загружены автоматически.

Существуют некоторые известные ограничения с поддержкой пакета NuGet в книгах:

- Собственные библиотеки поддерживается только в iOS и только в том случае, когда он связан с управляемой библиотеке.
- Пакеты, которые зависят от `.targets` файлы или сценарии PowerShell скорее всего не будут работать должным образом.
- Чтобы удалить или изменить зависимость пакета, измените манифест книги в текстовом редакторе. Правильный пакет управления находится на пути.

### <a name="xamarinforms-support"></a>Поддержка Xamarin.Forms

Если нужно сослаться на пакет Xamarin.Forms NuGet в книге, приложение книги изменится его главное представление, чтобы быть основаны на Xamarin.Forms. Вы сможете открыть его через `Xamarin.Forms.Application.Current.MainPage`.

На вкладке представления инспектор также имеет специальную поддержку для отображения иерархии представления Xamarin.Forms помогут вам понять ваши макеты.

### <a name="markdown-authoring"></a>Разработка markdown

Авторы книг может иногда проще для непосредственного редактирования CommonMark «источник» книги с помощью их любимых редактора.

Имейте в виду, что если затем изменить и сохранить книгу в клиентском книг, CommonMark текст может быть переформатированы.

Обратите внимание, что из-за расширения CommonMark мы используем для включения yaml-ФАЙЛ метаданных в файлах книги `---` зарезервирован для этой цели. Если вы хотите создать [тематической разрывы](http://spec.commonmark.org/0.27/#thematic-break) в текст, следует использовать `***` или `___` вместо этого. Следует избегать таких разрывы в версии 1.2 книг и ранее, из-за ошибки во время сохранения.

### <a name="improvements-in-workbooks-13"></a>Усовершенствования в книгах 1.3

Мы также расширил синтаксис Markdown блок квоты немного, чтобы улучшить презентации. Добавляя эмодзи в качестве первого символа в вашей цитаты, могут повлиять на цвет фона квоты:

- `> [!NOTE]`
    > отрисовывается как примечание с синим фоном
- `> [!IMPORTANT]`
    > будет отображаться как предупреждение с желтым фоном
- `> [!WARNING]`
    > отрисовывается как проблема с красным фоном

Можно также связать верхние колонтитулы документа книги. Мы создаем привязки для каждого заголовка с Идентификатором привязки, что текст заголовка, обрабатываются следующим образом:

1. Заголовок в нижнем регистре.
1. Удаляются все символы, за исключением буквы, цифры и дефисы.
1. Все пробелы заменяются на дефисы.

Это означает, что заголовок как «Важные Header» Получает идентификатор `important-header` и может быть связан с помощью вставку ссылки `#important-header` в книге.

## <a name="document-structure"></a>Структура документа

### <a name="cell"></a>ячейки

Дискретные единица содержимое, представляющее исполняемого кода или разметки. В состав входят ячейку кода до четырех компонентов.

- Редактор
  - Буфер
- Диагностика компилятора
- Выходные данные консоли
- Результаты выполнения

### <a name="editor"></a>Редактор

Компонент интерактивной текстовой ячейки. Для ячейки кода это фактический код редакторе с помощью возможностей выделения синтаксических конструкций и т. д. Для ячеек markdown это содержимого Редактор форматированного текста с помощью контекстной форматирования и создания панели инструментов.

### <a name="buffer"></a>Буфер

Фактический текстовое содержимое редактора.

### <a name="compiler-diagnostics"></a>Диагностика компилятора

Диагностики, созданных при компиляции кода, только в том случае, когда запрашивается явное выполнение. Сразу же отображается под областью редактора ячейки.

### <a name="console-output"></a>Выходные данные консоли

Никаких выходных данных, записанных в стандартный выход или стандартной ошибки во время выполнения ячейки. Стандартный выход, отображаются черным шрифтом и Стандартная ошибка отображается красным шрифтом.

### <a name="execution-results"></a>Результаты выполнения

Потенциально в интерактивном режиме представлений результатов для ячейки будут передаваться после успешной компиляции, предоставляемых фактически создается результат выполнения. Исключения считаются результаты в этом контексте, так как они создаются в результате реального выполнения компиляции.

## <a name="workbooks-files-types"></a>Типы файлов книг

### <a name="plain-files"></a>Обычные файлы

По умолчанию сохраняет книгу в виде простого текста `.workbook` файл, содержащий текст в формате CommonMark.

### <a name="packages"></a>Пакеты

Пакет книги — каталог с именем `.workbook` расширения.
Для поиска компьютера Mac, а также в диалоговое окно открытия Xamarin Workbooks и последние файлы меню этого каталога будут распознаваться как будто файл.

Каталог должен содержать `index.workbook` файл, который является книгой фактическое обычного текста, в которой будут загружены в Xamarin Workbooks. Каталог также может содержать ресурсы, необходимые `index.workbook`, в том числе изображения или другие файлы.

Если обычный текст `.workbook` открывается файл, который ссылается на ресурсы из одной папки в книгах 0.99.3 или более поздней версии, при его сохранении, он будет преобразован в `.workbook` пакета. Это справедливо на компьютерах Mac и Windows.

> [!NOTE]
> Пользователи Windows будет открывать `package.workbook\index.workbook` файл напрямую, но в противном случае пакет будет функционировать так же, как и на компьютере Mac.

### <a name="archives"></a>Архивы

Пакеты книги, каталогов, может быть трудно легко распространять через Интернет. Решением является архивы книги. Архив книги — это пакет сжатых zip книги, с именем `.workbook` расширения.

Диалоговое окно сохранения, начиная с версии 1.1 книг, при сохранении пакета книги, позволяет выбирать вместо сохранения как архив. Книги 1.0 не имеет встроенные возможности создания и сохранения архивов.

В книги 1.0 при открытии окна архива книги, он прозрачно был преобразован в пакет книги и ZIP-файле было потеряно. В версии 1.1 книг остается ZIP-файл. Когда пользователь сохраняет архив, он заменяется новый ZIP-файл.

Архив книгу можно создать вручную, щелкнув правой кнопкой мыши пакет книгу и выбрав **сжимать** на Mac или **Отправить > Сжатая ZIP-папку** на Windows. Переименуйте ZIP-файл, чтобы `.workbook` расширение файла. Это работает только с пакетами книги, файлы не обычный книги.
