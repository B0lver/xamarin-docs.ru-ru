## <a name="200-march-9-2015"></a>2.0.0: 9 марта 2015 г.

[Загрузить v2.0.0](https://download.xamarin.com/objective-sharpie/ObjectiveSharpie-2.0.0.pkg)

Цели Sharpie 2.0 является основном выпуске, улучшенная драйвера, на основе Clang и средство синтаксического анализа и механизм привязки на основе NRefactory. Эти улучшенные компоненты обеспечения _большая_ более высокий уровень привязки, особенно вокруг тип привязки. Были внесены несколько улучшений внутренних Sharpie цель, которое создаст множество пользовательских возможностей в будущих выпусках.

Цели Sharpie 2.0 основана на Clang 3.6.1.

### <a name="type-binding-improvements"></a>Усовершенствования типа привязки

* Блоки Objective-C теперь поддерживаются. Это включает в себя анонимных или встроенных блоков и блоков с именем через `typedef`. Анонимные блоки будут привязаны как `System.Action` или `System.Func` делегаты, а именованные блоки будут действовать как строго именованной `delegate` типов.

* Нет улучшенные эвристики именования для анонимного перечисления, перед которым идет `typedef` разрешения в целочисленный тип builtin, таких как `long` или `int`.

* Указатели C сейчас привязаны как в C# `unsafe` указатели вместо `System.IntPtr`. Это приведет к более ясности в привязке для Если требуется включить параметров-указателей в `out` или `ref` параметров. Невозможно вывести, всегда ли параметр `out` или `ref`, поэтому курсор сохраняется в привязке, чтобы разрешить для упрощения аудита.

* В приведенном выше привязку указатель исключение при обнаружении ранг 2 указатель на объект Objective-C в качестве параметра. В этих случаях соглашение преобладающим и параметр будет привязан как `out` (например `NSError **error` → `out NSError error`).

### <a name="verify-attribute"></a>Проверьте атрибут

Вы обнаружите, что созданные Sharpie цели привязки теперь будет сопровождаться атрибутом `[Verify]` атрибута. Эти атрибуты указывают, что следует _проверить_ , Sharpie цель выполнялось правильнее сравнение привязка, с помощью исходного объявления C, Objective-C (который будет предоставляться в комментарии перед объявлением связанных).

Проверка будет _рекомендуется_ для всех связанных объявлений, но вероятнее всего _необходимые_ для объявления, сопровождаемых `[Verify]` атрибута. Это так, как во многих ситуациях, не хватает метаданных в исходного машинного кода, чтобы определить, как лучше всего создать привязку. Необходимо ссылаться документации или комментарии к коду в файлах заголовка для принятия наиболее привязки.

В разделе [Проверьте атрибуты](~/cross-platform/macios/binding/objective-sharpie/platform/verify.md) документации для получения дополнительных сведений.

### <a name="other-notable-improvements"></a>Другие важные улучшения

* `using` операторы теперь создаются на основе типов привязан. Например если `NSURL` был привязан `using Foundation;` также создание инструкций.

* `struct` и `union` объявления привязывается, с помощью `[FieldOffset]` перебор для объединения.

* Значения перечисления с инициализаторами константное выражение правильно привязывается; Полное выражение преобразуется в C#.

* Методы с переменным числом аргументов и блоки теперь привязан.

* Платформы теперь поддерживаются через `-framework` параметр. См. в документации на [привязки встроенных инфраструктур](http://developer.xamarin.com/guides/ios/advanced_topics/binding_objective-c/objective_sharpie/#frameworks) для получения дополнительных сведений.

* Objective-C исходного кода будет автоматически, которого следует устранить необходимость в передаче `-ObjC` или `-xobjective-c` для Clang вручную.

* Использование модуля clang (`@import`) определяется теперь автоматически, которого следует устранить необходимость в передаче `-fmodules` для Clang вручную для библиотек, которые используют новый модуль поддержки в Clang.

* API единой Xamarin теперь является целевым объектом привязки по умолчанию; использовать `-classic` параметр ориентироваться только классический API 32-разрядной.

### <a name="notable-bug-fixes"></a>Важные исправления ошибок
