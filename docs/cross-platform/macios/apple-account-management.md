# <a name="apple-account-management"></a>Управление учетными записями Apple

Интерфейс управления учетной записью Apple предоставляет способ для просмотра всех групп разработчиков, связанных с идентификатор Apple ID. Он также позволяет просмотреть дополнительные сведения о каждой команде, отображая список _удостоверений подписывания_ и _профили подготовки_ , установленных на компьютере.

Проверка подлинности своего идентификатора Apple выполняется в командной строке с [fastlane](https://fastlane.tools/). fastlane должны устанавливаться на компьютере можно успешно пройти проверку подлинности. Дополнительные подробные сведения о fastlane и его установке см. в [fastlane](~/ios/deploy-test/provisioning/fastlane/index.md) руководства.

В диалоговом окне учетной записи Apple можно осуществлять следующее:

* **Создание и управление сертификатами**
* **Создание и управление ими, профили подготовки**

В этом руководстве описывается сведения о том, как это сделать.

> [!NOTE]
> Xamarin средств для управления учетными записями Apple только отобразить информацию о платных учетных записей разработчиков Apple. Узнайте, как протестировать приложение на устройстве без на платную учетную запись разработчика Apple, см. в разделе [бесплатной подготовки для приложений Xamarin.iOS](~/ios/get-started/installation/device-provisioning/free-provisioning.md) руководства.

Средства автоматической подготовки iOS может использоваться для автоматического создания и управления удостоверений подписывания, идентификаторами приложений и профили подготовки. Дополнительные сведения об использовании этих функций см. [подготовки устройств](~/ios/get-started/installation/device-provisioning/index.md) руководства.

## <a name="requirements"></a>Требования

Управление учетными записями Apple можно найти в Visual Studio для Mac, 2019 г. Visual Studio и Visual Studio 2017 (версия 15.7 и более поздних версий).

Необходимо иметь учетную запись разработчика Apple для использования этой функции. Дополнительные сведения об учетных записей разработчиков Apple в [подготовки устройств](~/ios/get-started/installation/device-provisioning/index.md) руководства.

- Убедитесь, что вы подключены к Интернету. Это обусловлено fastlane взаимодействует непосредственно с портала разработчиков Apple.
- Убедитесь в наличии [установлены инструменты fastlane](~/ios/deploy-test/provisioning/fastlane/index.md#Installation).
- Убедитесь, новейшие средства fastlane из [ https://download.fastlane.tools ](https://download.fastlane.tools).
- Перед началом работы убедитесь принять все лицензионные соглашения в [портал разработчика](https://developer.apple.com/account/).

## <a name="adding-an-apple-developer-account"></a>Добавление учетной записи разработчика Apple

# [<a name="visual-studio-for-mac"></a>Visual Studio для Mac](#tab/macos)

1. Чтобы открыть диалоговое окно управления учетной записи перейдите к **Visual Studio > Параметры > учетная запись разработчика Apple**:

    ![Параметры учетной записи разработчика Apple](apple-account-management-images/image1.png)

2. Нажмите клавишу **+** кнопки для отображения знака в диалоговом окне, как показано ниже: 

    ![диалоговое окно fastlane.](apple-account-management-images/image2.png)

4. Введите Apple ID и пароль и нажмите кнопку **Sign In** кнопки. Учетные данные будут сохранены в цепочке ключей безопасности на этом компьютере. [fastlane](~/ios/deploy-test/provisioning/fastlane/index.md) позволяет надежно обрабатывать свои учетные данные и передавать их в портале разработчика Apple.
 
5. Выберите **всегда разрешать** в диалоговом окне оповещения, чтобы разрешить Visual Studio, чтобы использовать учетные данные:

    ![Всегда разрешать диалоговое окно оповещения](apple-account-management-images/image4.png)

6. После успешного добавления учетной записи, вы увидите свой идентификатор Apple ID и какие-либо группы, которые ваш идентификатор Apple ID является частью.

    ![Диалоговое окно учетной записи разработчика Apple с учетными записями, которые добавлены](apple-account-management-images/image5.png)

7. Выберите все группы и нажмите клавишу **Просмотр сведений о...** . Отобразится список всех удостоверений подписи и профилей подготовки, которые установлены на вашем компьютере:

    ![Просмотр сведений о экран, отображающий на удостоверений подписывания и профили на вашем компьютере подготовки](apple-account-management-images/image6.png)

## <a name="managing-signing-identities-and-provisioning-profiles"></a>Управление удостоверениями подписывания и профили подготовки

Сведения о диалоговом окне team отображает список удостоверений подписывания, упорядоченных по типу. **Состояние** столбец будет указано, является ли сертификат: 

* **Допустимые** — удостоверения подписывания (сертификат и закрытый ключ) устанавливается на компьютере, и не истек.

* **Не в цепочке ключей** – имеется допустимое удостоверение подписи на сервере компании Apple. Чтобы установить это на вашем компьютере, его необходимо экспортировать с другого компьютера. Не удается загрузить удостоверения подписывания с портала разработчиков Apple, так как он не будет содержать закрытый ключ.

* **Отсутствует закрытый ключ** — сертификат без закрытого ключа устанавливается в цепочке ключей.

* **Истек срок действия** — срок действия сертификата. Вам следует удалить из цепочки ключей.

  ![сведения о диалоговом окне сведения о команде](apple-account-management-images/image7.png)

## <a name="create-a-signing-identities"></a>Создание удостоверения подписывания

Чтобы создать новое удостоверение подписывания, выберите **Create Certificate** кнопку раскрывающегося списка и выберите нужный тип, который требуется. Если у вас есть необходимые разрешения новой подписи удостоверений появится через несколько секунд.

Если один из вариантов в раскрывающемся списке серым и не выбран, это означает, что у вас нет разрешения соответствующей команде разработчиков для создания такого типа.

## <a name="troubleshooting"></a>Устранение неполадок

### <a name="view-details-dialog-is-empty"></a>Диалоговое окно сведений о представлении пуст

В настоящее время это известная проблема, связанная с ошибкой [#53906](https://bugzilla.xamarin.com/show_bug.cgi?id=53906). Убедитесь, что используется последняя стабильная версия Visual Studio для Mac

### <a name="if-you-are-experiencing-issues-logging-in-your-account-please-try-the-following"></a>Если возникают проблемы со входом в вашей учетной записи, воспользуйтесь одной из следующих:

* Откройте приложение цепочки ключей и категории выберите *пароли*. Поиск `deliver.`, а затем удалите все записи.

### <a name="error-adding-account-please-sign-in-with-an-app-specific-password"></a>«Произошла ошибка при добавлении учетной записи. Выполните вход в систему с паролем конкретного приложения»

Это обусловлено двухфакторная проверка подлинности включена в вашей учетной записи. Убедитесь, что используется последняя стабильная версия Visual Studio для Mac

### <a name="failed-to-create-new-certificate"></a>Не удалось создать новый сертификат
«Достигнут предел для сертификатов этого типа»

![диалоговое окно сертификата ограничение](apple-account-management-images/image10.png)

Максимальное допустимое число сертификатов созданы. Чтобы устранить эту проблему, перейдите к [Apple Developer Center](https://developer.apple.com/account/ios/certificate/distribution) и отмените один из рабочих сертификатов.
