# <a name="building-html-views-using-razor-templates"></a>Построение HTML представления с помощью шаблонов Razor

В мире Разработка мобильных приложений термин «гибридное приложение» обычно относится к приложение, которое представляет некоторые (или все) его экранов в виде HTML-страницы внутри просмотра размещенного веб-элемент управления.

Существует некоторых сред разработки, которые позволяют созданию мобильного приложения, полностью на языках HTML и Javascript, однако эти приложения может снизиться из проблем с производительностью, при попытке выполнить сложную обработку или эффекты пользовательского интерфейса, а также являются в платформе также ограниченный функции, которые они могут получить доступ.

Xamarin предоставляет преимущества обоих миров, особенно в том случае, при использовании подсистемы шаблонов Razor HTML. С помощью Xamarin у вас есть возможность построить кросс платформенных шаблона HTML представлений, использующих Javascript и CSS, а также имеют полный доступ к интерфейсам API базовой платформы и быстрая обработка с помощью C#.

В этом документе описывается использование Razor, модуль шаблонов построения представления HTML + Javascript + CSS, которые могут использоваться на мобильных платформах с помощью Xamarin.

## <a name="using-web-views-programmatically"></a>С помощью веб-представления программными средствами

Прежде чем мы изучаем Razor в этом разделе описывается использование веб-представления для отображения HTML-содержимое непосредственно — в частности HTML-содержимого, которые созданы в приложение.

Xamarin предоставляет полный доступ к интерфейсам API базовой платформы на iOS и Android, поэтому его легко создавать и отображать HTML с помощью C#. Ниже показан основной синтаксис для каждой платформы.

### <a name="ios"></a>iOS

Отображение HTML в элементе управления UIWebView в Xamarin.iOS также принимает несколько строк кода:

```csharp
var webView = new UIWebView (View.Bounds);
View.AddSubview(webView);
string contentDirectoryPath = Path.Combine (NSBundle.MainBundle.BundlePath, "Content/");
var html = "<html><h1>Hello</h1><p>World</p></html>";
webView.LoadHtmlString(html, NSBundle.MainBundle.BundleUrl);
```

В разделе [iOS UIWebView](http://docs.xamarin.com/recipes/ios/content_controls/web_view/) рецепты Дополнительные сведения об использовании UIWebView управления.

### <a name="android"></a>Android

Отображение HTML в элементе управления WebView, с помощью Xamarin.Android достигается с помощью всего нескольких строк кода:

```csharp
// webView is declared in an AXML layout file
var webView = FindViewById<WebView> (Resource.Id.webView);
var html = "<html><h1>Hello</h1><p>World</p></html>";
webView.LoadDataWithBaseURL("file:///android_asset/", html, "text/html", "UTF-8", null);
```

В разделе [Android WebView](http://docs.xamarin.com/recipes/android/controls/webview/) рецепты Дополнительные сведения об использовании элемента управления WebView.

### <a name="specifying-the-base-directory"></a>Определение базового каталога

На обеих платформах есть параметр, который определяет базовый каталог для HTML-страницы. Это расположение в файловой системе устройства, используемый для разрешения относительных ссылок на ресурсы, такие как изображения и CSS-файлах. Например теги, такие как

```html
<link rel="stylesheet" href="style.css" />
<img src="monkey.jpg" />
<script type="text/javascript" src="jscript.js">
```

ссылаться на эти файлы: **style.css**, **monkey.jpg** и **jscript.js**. Параметр базовый каталог сообщает веб-представление, в которой содержатся такие файлы, их можно загрузить на странице.

#### <a name="ios"></a>iOS

Выходные данные шаблона обрабатывается в iOS с помощью следующего кода C#:

```csharp
webView.LoadHtmlString (page, NSBundle.MainBundle.BundleUrl);
```

Базовый каталог указан как `NSBundle.MainBundle.BundleUrl` обозначает, что приложение установлено в каталог. Все файлы в **ресурсов** папки копируются в этом расположении, таких как **style.css** файла, показано ниже:

```csharp

```

Специальной строки **file:///android_asset/** указывает на папку средств Android в приложении, показанные здесь содержащий **style.css** файла.

### <a name="calling-c-from-html-and-javascript"></a>Вызов C# из HTML и Javascript

При загрузке HTML-страницы в веб-представление, обрабатывает ссылки и формы как это происходит, если страница была загружена с сервера. Это означает, что если пользователь щелкает ссылку или отправке формы веб-представление будет пытаться перейти в указанный целевой объект.

Если связь является к внешнему серверу (например, google.com) веб-представление попытается загрузить внешний веб-сайт (при условии, что имеется подключение к Интернету).

```html
<a href="http://google.com/">Google</a>
```

Если ссылка является относительной веб-представление попытается загрузить содержимое из базового каталога. Очевидно, что отсутствует сетевое подключение не требуется для правильной работы, содержимое хранится в приложение на устройстве.

```html
<a href="somepage.html">Local content</a>
```

Действий формы выполните такие же правила.

```html
<form method="get" action="http://google.com/"></form>
<form method="get" action="somepage.html"></form>
```

Вы не собираетесь размещения веб-сервера на клиентском компьютере; Тем не менее можно использовать те же способы связи сервера, задействованных в современных шаблонах гибкий Дизайн для вызова служб через HTTP GET и обрабатывают асинхронно с помощью выпуска Javascript (или вызывающий Javascript, уже размещенные в веб-представление). Это позволяет легко передавать данные из HTML обратно в код C# для обработки и отображения результатов обратно на HTML-странице.

IOS и Android предоставляет механизм для кода приложения может перехватывать эти события навигации, чтобы код приложения мог отвечать (при необходимости). Эта функция крайне важны для построения гибридного приложения, потому, что она позволяет взаимодействовать с веб-представление машинного кода.

#### <a name="ios"></a>iOS

Событие ShouldStartLoad на веб-страниц в iOS может быть переопределен для обработки запроса перехода (например, щелчок ссылки) кода приложения. Предоставьте все сведения, параметры метода

```csharp
bool HandleShouldStartLoad (UIWebView webView, NSUrlRequest request, UIWebViewNavigationType navigationType) {
    // return true if handled in code
    // return false to let the web view follow the link
}
```

и затем назначить обработчик события.

```csharp
webView.ShouldStartLoad += HandleShouldStartLoad;
```

#### <a name="android"></a>Android

В Android просто подкласс WebViewClient, а затем реализуйте код для ответа на запрос навигации.

```csharp
class HybridWebViewClient : WebViewClient {
    public override bool ShouldOverrideUrlLoading (WebView webView, string url) {
        // return true if handled in code
        // return false to let the web view follow the link
    }
}
```

а затем установите клиент на веб-представление:

```csharp
webView.SetWebViewClient (new HybridWebViewClient ());
```

### <a name="calling-javascript-from-c"></a>Вызов Javascript из C#

Помимо о том, чтобы загрузить новую HTML-страницу для веб-просмотра, код C# можно также запустить Javascript в пределах текущей страницы. Все блоки кода Javascript можно создать с помощью C# строк и выполнения или можно создать вызовов Javascript уже доступен на странице через `script` тегов.

#### <a name="android"></a>Android

Создайте код Javascript, который будет выполнен и затем с помощью префикса «javascript:» и настройте веб-представление для загрузки этой строки:

```csharp
var js = "alert('test');";
webView.LoadUrl ("javascript:" + js);
```

#### <a name="ios"></a>iOS

веб-представления iOS предоставляют метод специально для вызова Javascript:

```csharp
var js = "alert('test');";
webView.EvaluateJavascript (js);
```

## <a name="what-is-razor"></a>Что такое Razor

Razor — это механизм шаблонов, которая впервые появилась в ASP.NET MVC, изначально будет выполняться на сервере и создавать HTML, чтобы обслуживать веб-браузеры.

Подсистемы шаблонов Razor расширяет стандартный синтаксис HTML с помощью C#, чтобы можно express макета и легко встраивать таблицы стилей CSS и Javascript. Шаблон может ссылаться на класс модели, который может быть любой тип и свойства которого может осуществляться непосредственно из шаблона. Один из его основных преимущества — возможность легко смешивать синтаксис HTML и C#.

Шаблоны Razor не только для использования на сервере, они также могут быть включены в приложениях Xamarin. С помощью шаблонов Razor, а также возможность работать с веб-представления программным образом позволяет приложениям сложных гибридное межплатформенных, созданных с помощью Xamarin.

### <a name="razor-template-basics"></a>Основы шаблонов Razor

Файлы шаблонов Razor имеют **.cshtml** расширение имени файла. Могут быть добавлены в проект Xamarin из раздела текстовых шаблонов в **новый файл** диалогового окна:

 ![Новый файл - шаблона Razor](images/image5_400x201.png)

Простой шаблон Razor ( **RazorView.cshtml**) показано ниже.

```html
@model string
<html>
    <body>
    <h1>@Model</h1>
    </body>
</html>
```

Обратите внимание, следующие отличия от регулярных HTML-файл:

-  `@` Символ имеет особое значение в шаблонах Razor — он указывает, что следующее выражение C# для оценки.
- `@model` Директива всегда отображается в первой строке файла шаблона Razor.
-  `@model` Следует после директивы по типу. В этом примере простая строка передается в шаблон, но это может быть любой пользовательский класс.
-  Когда `@Model` упоминается в шаблоне, он предоставляет ссылку на объект, передаваемый в шаблоне, во время создания (в данном примере, он будет строка).
-  Интегрированная среда разработки автоматически создаст разделяемого класса для шаблонов (файлы с **.cshtml** расширения). Этот код можно просматривать, но его не следует изменять.
 ![RazorView.cshtml](images/image6_125x34.png) разделяемый класс называется RazorView должно соответствовать имени файла шаблона .cshtml. Это это имя, используемое для ссылки на шаблон в коде C#.
- `@using` операторы также могут быть включены в верхней части шаблона Razor для включения дополнительных пространств имен.


После этого можно сгенерировать выходных HTML с помощью следующего кода C#. Обратите внимание, что мы модели в качестве строки «Hello World» которого будут включены в выходные данные шаблона в готовом для просмотра.

```csharp
var template = new RazorView () { Model = "Hello World" };
var page = template.GenerateString ();
```

### <a name="more-razor-syntax"></a>Дополнительные сведения о синтаксисе Razor

В этом разделе, которое мы будем представить некоторые базовый синтаксис Razor, чтобы помочь вам приступить к работе с помощью его. Примеры в этом разделе заполнить следующий класс данных и откройте его с помощью Razor:

```csharp
public class Monkey {
    public string Name { get; set; }
    public DateTime Birthday { get; set; }
    public List<string> FavoriteFoods { get; set; }
}
```

Все примеры используют следующий код инициализации данных

```csharp
var animal = new Monkey {
    Name = "Rupert",
    Birthday=new DateTime(2011, 04, 01),
    FavoriteFoods = new List<string>
        {"Bananas", "Banana Split", "Banana Smoothie"}
};
```

#### <a name="displaying-model-properties"></a>Отображение свойств модели

При использовании модели со свойствами класса, они являются легко ссылаться на них в шаблоне Razor как показано в этом примере шаблоне:

```html
@model Monkey
<html>
    <body>
    <h1>@Model.Name</h1>
    <p>Birthday: @(Model.Birthday.ToString("d MMMM yyyy"))</p>
    </body>
</html>
```

Это может быть подготовлен к просмотру в строку, используя следующий код:

```csharp
var template = new RazorView () { Model = animal };
var page = template.GenerateString ();
```

Окончательный результат показан здесь в веб-представление на iOS Simulator и эмулятор Android:

 ![Rupert](images/image8_516x160.png)

#### <a name="c-statements"></a>Операторы C#

Более сложные C# могут быть включены в шаблон, например обновления свойств модели и вычисления возраста в этом примере:

```html
@model Monkey
<html>
    <body>
    @{
        Model.Name = "Rupert X. Monkey";
        Model.Birthday = new DateTime(2011,3,1);
    }
    <h1>@Model.Name</h1>
    <p>Birthday: @Model.Birthday.ToString("d MMMM yyyy")</p>
    <p>Age: @(Math.Floor(DateTime.Now.Date.Subtract (Model.Birthday.Date).TotalDays/365)) years old</p>
    </body>
</html>
```

Можно написать сложные однострочный выражения C# (например, форматирование возраст), заключив код в `@()`.

Несколько операторов C# могут быть записаны, заключив их с `@{}`.

#### <a name="if-else-statements"></a>If-else-операторы

Ветви кода могут выражаться с помощью `@if` , как показано в следующем примере шаблон.

```html
@model Monkey
<html>
    <body>
    <h1>@Model.Name</h1>
    <p>Birthday: @(Model.Birthday.ToString("d MMMM yyyy"))</p>
    <p>Age: @(Math.Floor(DateTime.Now.Date.Subtract (Model.Birthday.Date).TotalDays/365)) years old</p>
    <p>Favorite Foods:</p>
    @if (Model.FavoriteFoods.Count == 0) {
        <p>No favorites</p>
    } else {
        <p>@Model.FavoriteFoods.Count favorites</p>
    }
    </body>
</html>
```

#### <a name="loops"></a>Циклы

Циклические конструкции, такие как `foreach` также могут быть добавлены. `@` Префикс может быть использован для переменной цикла ( `@food` в данном случае) для его отображения в формате HTML.

```html
@model Monkey
<html>
    <body>
    <h1>@Model.Name</h1>
    <p>Birthday: @Model.Birthday.ToString("d MMMM yyyy")</p>
    <p>Age: @(Math.Floor(DateTime.Now.Date.Subtract (Model.Birthday.Date).TotalDays/365)) years old</p>
    <p>Favorite Foods:</p>
    @if (Model.FavoriteFoods.Count == 0) {
        <p>No favorites</p>
    } else {
        <ul>
            @foreach (var food in @Model.FavoriteFoods) {
                <li>@food</li>
            }
        </ul>
    }
    </body>
</html>
```

### <a name="static-content"></a>Статическое содержимое

Статическое содержимое включает таблицы стилей CSS, изображения, файлы Javascript или другое содержимое, которое могут быть связаны с или ссылается на HTML-файл, отображаемых в веб-представление.

Проекты шаблонов включают минимальной стилей чтобы продемонстрировать, как включить статическое содержимое гибридного приложения. Таблица стилей CSS упоминается в шаблоне следующим образом:

```html
<link rel="stylesheet" href="style.css" />
```

Можно добавить любой таблицы стилей и файлы Javascript, вам требуется, включая платформ, таких как JQuery.

### <a name="razor-cshtml-templates"></a>Razor cshtml шаблонов

Шаблон включает Razor **.cshtml** файл, который предварительно записан кода для обмена данными между HTML и Javascript и C#. Это позволит вам построения сложных гибридные приложения, которые не просто отобразить только для чтения данных из модели, но также принимают пользовательский ввод в HTML и передать его обратно в код C# для обработки или хранения.

#### <a name="calling-c-code-from-the-template"></a>Вызов кода C# на основе шаблона

Связь с отображаемой веб-просмотра обратного вызова для C# выполняется путем настройки URL-адреса для веб-представление, а затем перехват запроса на языке C# для обработки собственного запроса без перезагрузки веб-представление.

Пример можно просмотреть в обработку RazorView на кнопку. Эта кнопка имеет следующий код HTML:

```html
<input type="button" name="UpdateLabel" value="Click" onclick="InvokeCSharpWithFormValues(this)" />
```

`InvokeCSharpWithFormValues` Функции Javascript, которая считывает все значения из формы HTML и наборы `location.href` для веб-представление:

```javascript
location.href = "hybrid:" + elm.name + "?" + qs;
```

Это пытается перейти на веб-представление на URL-адрес с пользовательской схемы (например) `hybrid:`)

```
hybrid:UpdateLabel?textbox=SomeValue&UpdateLabel=Click
```

Когда собственного веб-представление обрабатывает этот запрос навигации, у нас есть возможность сможет его перехватить. Операций ввода-вывода это делается путем обработки события HandleShouldStartLoad UIWebView. В Android мы просто подкласс WebViewClient, используемых в форме и переопределить ShouldOverrideUrlLoading.

Внутренние компоненты, эти два перехватчики навигации по существу одинакова.

Во-первых, проверьте URL-адрес, который веб-представление пытается загрузить, и если он не начинается с пользовательской схемы (`hybrid:`), разрешить для перехода в обычном режиме.

Для нестандартной схемы URL-адрес, все данные в URL-АДРЕСЕ между схемой и «?» — Это имя метода для обработки (в данном случае «UpdateLabel»). Все, что в строке запроса будет рассматриваться как параметры для вызова метода:

```csharp
var resources = url.Substring(scheme.Length).Split('?');
var method = resources [0];
var parameters = System.Web.HttpUtility.ParseQueryString(resources[1]);
```

`UpdateLabel` в этом образце делает минимальный объем строками в параметре текстовое поле (добавляя «C# говорит» в строку), а затем выполняет обратный вызов веб-представление.

После обработки URL-адрес, метод прерывает навигации, чтобы веб-представление не будет пытаться Готово переход к пользовательский URL-адрес.

#### <a name="manipulating-the-template-from-c"></a>Обработка шаблона из C#

Взаимодействия готовый для просмотра веб-представление HTML из C# выполняется путем вызова Javascript в веб-представлении. На iOS, это делается путем вызова `EvaluateJavascript` на UIWebView:

```csharp
webView.EvaluateJavascript (js);
```

В Android Javascript может быть вызвана в режиме веб-загрузки Javascript как URL-адрес с помощью `"javascript:"` схема URL-адресов:

```csharp
webView.LoadUrl ("javascript:" + js);
```

## <a name="making-an-app-truly-hybrid"></a>Создание приложения истинно гибридную

Эти шаблоны не следует использовать в собственных элементов управления для каждой платформы-во весь экран, заполненный представление один веб-узел.

HTML могут быть отлично подходит для создания прототипов и отображение тех вещей веб-сайте лучше всего подходит для таких как форматированного текста и динамический макет. Однако не все задачи подходят для HTML и Javascript – прокрутке длинные списки данных, например, выполняет лучше собственные элементы управления пользовательского интерфейса, например (UITableView на iOS) или ListView на Android.

Веб-представления в шаблоне можно легко дополнить элементов управления от платформы — просто изменить **MainStoryboard.storyboard** в конструкторе операций ввода-вывода или **Resources/layout/Main.axml** на Android.

Оба образца приложения включают Twitter для управления доступом и преобразования текста в речь API из собственной платформы, демонстрации, гибридные приложения с помощью Xamarin по-прежнему иметь доступ к базовой функции из представлений на основе шаблона HTML Razor.
